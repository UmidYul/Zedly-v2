## 🧠 Инструкция по мышлению

Перед написанием ЛЮБОГО кода:

1. Прочитай весь промт целиком.
2. Составь список всех файлов которые нужно трогать — компоненты, страницы, стили, роуты.
3. Думай системно: у нас образовательная платформа, три роли (director / teacher / student), каждая видит разный интерфейс. Все дизайн-решения должны работать для всех трёх.
4. Не ломай существующую логику — это UI/UX overhaul, не рефакторинг бизнес-логики.
5. Если видишь место где UX можно улучшить сверх задания — сделай и опиши что улучшил.
6. Весь новый UI должен быть pixel-consistent: одна дизайн-система, одни токены, одни компоненты.

## 📦 Контекст проекта

Образовательная платформа: школы, директора, учителя, ученики. Тесты, сессии, аналитика.

**Стек:** React + Vite + PWA, TypeScript, Tailwind CSS

**Текущее состояние UI:**
- Auth screens — есть
- Profile, School Users, Class Invites — есть
- Tests Workbench (Teacher Builder + Student Screen + Results) — есть, в процессе доводки
- Admin Panel — **слабая или отсутствует, нужно строить с нуля**

**Роли:**
- `director` — управляет школой, пользователями, видит аналитику по всей школе
- `teacher` — создаёт тесты, назначает классам, смотрит результаты своих классов
- `student` — проходит тесты, видит свои результаты

**Существующие API (все рабочие, трогать не нужно):**
```
Auth:        POST /api/v1/auth/login|logout|logout-all|refresh
             POST /api/v1/auth/password/forgot|change
Users:       GET/PATCH /api/v1/users/me
             GET/PATCH /api/v1/schools/{id}/users/{uid}
Schools:     GET /api/v1/schools/{id}/users
Tests:       POST /api/v1/tests (create)
             POST /api/v1/tests/{id}/assign
             GET  /api/v1/tests/{id}/results?class_id=
Sessions:    POST /api/v1/sessions/{id}/start|finish
Analytics:   GET  /api/v1/analytics/teacher/dashboard
```

## 🎨 ЧАСТЬ 1: Дизайн-система — Academic Design

### Философия
Строгий академический стиль. Не корпоративный bootstrap, не playful edtech с градиентами. Ближе всего к: Notion + Linear + академический журнал. Чистота, иерархия, уважение к данным.

**Принципы:**
- Контент важнее декора — никаких декоративных иллюстраций, паттернов, blob-форм
- Данные читаются с первого взгляда — правильная типографика, контраст, whitespace
- Интерактивность очевидна — hover states, focus rings, transitions (но не анимации ради анимаций)
- Плотность информации — дашборды должны показывать много данных без ощущения перегруза

### Цветовая палитра — определить как CSS переменные и Tailwind tokens

```css
/* Базовые — нейтральный slate, не чистый чёрный */
--color-bg:          #F8F9FA;   /* фон страниц */
--color-surface:     #FFFFFF;   /* карточки, панели */
--color-surface-2:   #F1F3F5;   /* вложенные поверхности, zebra */
--color-border:      #E2E8F0;   /* все бордеры */
--color-border-strong: #CBD5E1; /* акцентные бордеры */

/* Текст */
--color-text-primary:   #0F172A; /* заголовки */
--color-text-secondary: #475569; /* вторичный текст */
--color-text-muted:     #94A3B8; /* подписи, плейсхолдеры */

/* Акцент — глубокий синий (академический, не tech-синий) */
--color-accent:         #1E40AF; /* primary actions */
--color-accent-hover:   #1D3280;
--color-accent-light:   #EFF6FF; /* backgrounds для accent элементов */
--color-accent-muted:   #BFDBFE; /* badges, теги */

/* Семантические */
--color-success:   #166534; /* --success-light: #F0FDF4 */
--color-warning:   #92400E; /* --warning-light: #FFFBEB */
--color-danger:    #991B1B; /* --danger-light:  #FEF2F2 */

```

### Компоненты дизайн-системы — создать в `src/components/ui/`

Каждый компонент — TypeScript + Tailwind, никаких inline styles.

**`Button`**
```tsx
variants: 'primary' | 'secondary' | 'ghost' | 'danger' | 'link'
sizes: 'sm' | 'md' | 'lg'
states: loading (spinner), disabled
// primary: bg-accent text-white, hover: bg-accent-hover
// secondary: bg-white border border-border, hover: bg-surface-2
// ghost: transparent, hover: bg-surface-2
// danger: bg-danger-light text-danger border border-danger/20
```

**`Badge`**
```tsx
variants: 'default' | 'success' | 'warning' | 'danger' | 'info' | 'outline'
sizes: 'sm' | 'md'
// Маленький, rounded-full, uppercase text-xs font-medium
// Никаких ярких fill-цветов — только light backgrounds с темным текстом
```

**`Card`**
```tsx
// bg-white border border-border rounded-lg
// variants: 'default' | 'flat' (no shadow) | 'elevated' (тень)
// Заголовок карточки: <Card.Header> с разделителем
// Тело: <Card.Body> с padding
```

**`Table`**
```tsx
// Строгая таблица: thead bg-surface-2, borders, hover на строках
// Поддержка: сортировка (иконки ↑↓), loading skeleton, empty state
// Sticky header для длинных таблиц
// Checkbox column для bulk actions
// Pagination встроенная
```

**`Input`, `Select`, `Textarea`, `Checkbox`, `RadioGroup`**
```tsx
// Единый стиль: border border-border rounded-md, focus: ring-2 ring-accent/30
// Label сверху, error message снизу красным
// Обязательно: aria-labels, описания ошибок связаны с полем
```

**`Modal`**
```tsx
// Radix Dialog под капотом
// Backdrop blur(4px) opacity-50
// Анимация: fade + scale (200ms)
// Header с заголовком и кнопкой закрыть
// Footer с action buttons
```

**`Tabs`**
```tsx
// Два варианта:
// 'line' — underline активного таба (для страниц)
// 'segment' — pill-группа (для переключателей внутри карточек)
```

**`Stat`** — карточка метрики
```tsx
// Заголовок метрики (text-sm text-muted)
// Значение (text-2xl font-bold)
// Дельта (↑12% зелёным / ↓3% красным)
// Опционально: sparkline (Recharts) или иконка
```

**`EmptyState`**
```tsx
// Иконка (outline, не заливка)
// Заголовок + подпись
// Опционально: CTA кнопка
// Никаких иллюстраций — только иконки из Lucide
```

**`Skeleton`** — loading placeholder
```tsx
// animate-pulse, bg-surface-2
// варианты: text, card, table-row, stat
```

**`Alert`**
```tsx
variants: 'info' | 'success' | 'warning' | 'danger'
// Левая цветная полоска, иконка, заголовок + текст
// Dismissible опционально
```

**`Avatar`**
```tsx
// Initials fallback если нет фото
// sizes: sm(24) | md(32) | lg(40) | xl(56)
// AvatarGroup для списка (до 5 + "+N")
```

**`Tooltip`**
```tsx
// Radix Tooltip, delay 400ms
// Тёмный bg, text-white, text-xs
```

**`ProgressBar`**
```tsx
// variants: 'default' | 'success' | 'warning' | 'danger'
// animated fill
// Labeled (% сверху или внутри)
```

## ⚠️ Важные ограничения

- **Не трогать backend** — все API endpoint'ы рабочие, менять нельзя
- **Не трогать auth логику** — cookie-first refresh, logout-all, OTP flow — всё работает, только визуал
- **Сохранить PWA** — `vite.config.ts` с PWA плагином не ломать
- **API envelope** — все запросы возвращают `{ok: true, data: ...}` или `{ok: false, error: ...}` — обрабатывать соответственно
- **TypeScript strict** — никаких `any`, все типы явные
- **Accessibility** — ARIA labels, keyboard navigation, focus management в modal/drawer
