# НТТ-Симулятор — Симулятор национального тестирования

**Файл:** `docs/03_features/ntt_simulator.md`  
**Приоритет:** MVP (killer feature для beachhead — частные школы Ташкента)  
**Статус:** Production Blueprint

---

## Цель фичи

Воспроизвести точный формат Национального тестового тестирования (НТТ) Узбекистана для подготовки учеников к поступлению в университет. Ученик проходит симулятор в тех же условиях, что и настоящий экзамен: 60 вопросов, 90 минут, строгий таймер, без паузы, без возврата к пройденным блокам.

**Почему это MVP:** НТТ — главный страх ученика и родителя в Узбекистане. Учителя, дающие доступ к симулятору, становятся незаменимыми. Это главная точка входа в freemium-воронку.

**Ключевое отличие от обычного теста:** НТТ-симулятор — не просто тест с таймером. Это полная эмуляция психологического и форматного давления реального экзамена: интерфейс, порядок предметов, штрафные баллы, разбор ошибок в формате официальных ключей НТТ.

---

## Кто использует

| Роль | Действие |
|------|----------|
| Ученик 10–11 класс (основной) | Проходит симулятор, изучает разбор ошибок, отслеживает прогресс |
| Ученик 8–9 класс | Тренируется заблаговременно |
| Учитель | Назначает симулятор классу, видит NTT Readiness Score класса |
| Директор | Видит NTT Readiness Score по 10–11 классам (платный план) |
| Родитель | Получает Telegram-уведомление о результате попытки |

## Формат НТТ (точная спецификация)

| Параметр | Значение |
|----------|----------|
| Количество вопросов | 60 |
| Время | 90 минут |
| Таймер | Строгий, на сервере. Нельзя поставить на паузу |
| Тип вопросов | Только MCQ (4 варианта, 1 правильный) |
| Порядок вопросов | Фиксированный по блокам предметов |
| Штрафные баллы | −0.25 за неправильный ответ (настраивается: вкл/выкл учителем) |
| Пропуск вопроса | Разрешён (0 баллов, без штрафа) |
| Возврат к вопросам | Разрешён в пределах текущего блока; между блоками — нет |
| Максимальный балл | 60 (без штрафов) / меньше при штрафах |
| Минимальный балл | 0 (не уходит в минус) |
| Язык | Узбекский / Русский (выбирается до старта) |

### Структура по предметам (стандарт НТТ Узбекистана)

| Блок | Предмет | Вопросов |
|------|---------|----------|
| 1 | Математика | 30 |
| 2 | Предмет по выбору (физика / химия / биология / история / география / и др.) | 30 |

Учитель задаёт «предмет по выбору» при создании симулятора для класса. Ученик при самостоятельном запуске выбирает профиль сам.

## User Flow

### Flow A: Ученик запускает симулятор (самостоятельно)

```
1. Ученик открывает раздел «НТТ-подготовка»
2. Видит карточки профилей:
   «Математика + Физика», «Математика + Химия», «Математика + Биология», и т.д.
3. Выбирает профиль → нажимает «Начать симулятор»
4. Экран-предупреждение (обязательное подтверждение):
   ┌────────────────────────────────────────────────────┐
   │ 60 вопросов · 90 минут · Таймер нельзя остановить │
   │ Штрафные баллы: −0.25 за неправильный ответ       │
   │ После старта нельзя вернуться в меню               │
   │ ☑ Я понимаю условия                                │
   │              [Начать симулятор]                    │
   └────────────────────────────────────────────────────┘
5. session.started_at фиксируется на сервере
6. Экран теста:
   - Таймер в правом верхнем углу (обратный отсчёт)
   - «Вопрос 1 из 60» + название блока «Математика»
   - Прогресс-бар: зелёный = отвечен, серый = пропущен, белый = не посещён
   - Навигация «← Предыдущий» / «Следующий →» внутри блока
   - При попытке перейти к блоку 2: «Вы переходите к следующему предмету. Вернуться к Математике будет нельзя»
7. Каждый ответ автосохраняется (≤ 300 мс, POST /ntt/sessions/{id}/answers)
8. Завершение:
   - Нажал «Завершить»: модальное окно «Вы ответили на N из 60. Завершить тест?»
   - Таймер истёк: сервер автоматически закрывает сессию
9. Экран результата → см. Flow B
```

### Flow B: Экран результата и разбор ошибок

```
Экран результата:
  ┌─────────────────────────────────────────┐
  │  Ваш результат: 43 / 60  (71.7%)        │
  │  Математика: 24/30  ·  Физика: 19/30   │
  │  Прогресс: ↑ +5 баллов с прошлой попытки│
  │  Вы лучше, чем 72% учеников на Zedly   │
  │  В Ташкенте: топ 18%                    │
  │                                         │
  │  [Разбор ошибок]  [Поделиться]  [Ещё раз]│
  └─────────────────────────────────────────┘

Экран разбора ошибок:
  - Все 60 вопросов: ✅ правильно / ❌ ошибка / ⏭ пропущен
  - Клик на ❌: правильный ответ + объяснение (если заполнено в пуле)
  - Группировка по темам:
    «Тригонометрия — 1/5 правильно ⚠»
    «Функции и графики — 5/5 правильно ✅»
  - Кнопка «Создать тест по слабым темам»
    → запускает AI-генерацию: 20 вопросов по топ-3 слабым темам
```

### Flow C: История попыток

```
Таблица:
  | Дата       | Профиль          | Балл  | Перцентиль | Время |
  |------------|------------------|-------|------------|-------|
  | 25.02.2026 | Матем. + Физика  | 43/60 | Топ 28%    | 87 мин|
  | 18.02.2026 | Матем. + Физика  | 38/60 | Топ 41%    | 90 мин|
  | 10.02.2026 | Матем. + Химия   | 31/60 | Топ 58%    | 90 мин|

График прогресса: линейный, по датам попыток
Breakdown по темам: динамика каждой темы по попыткам
Лучшая попытка: отмечена ★
```

### Flow D: Учитель назначает симулятор классу

```
1. Учитель → «Создать НТТ-симулятор для класса»
2. Выбирает профиль и устанавливает дедлайн
3. Опции: включить/отключить штрафные баллы, разрешить/запретить смотреть ответы сразу
4. Назначает классу → Telegram-уведомление ученикам
5. Дашборд учителя показывает:
   - Средний NTT Readiness Score класса
   - Гистограмма распределения баллов
   - Список не прошедших + кнопка «Напомнить» (Telegram-пуш)
```

## NTT Readiness Score

Прогнозный показатель готовности ученика / класса к реальному НТТ. Обновляется ежедневно через background job.

**Формула:**
```
NTT Readiness Score =
  (avg_score_last_3_attempts / 60 × 100) × 0.50
  + (score_delta_last_month / 10)         × 0.30
  + (topics_without_critical_errors / total_topics × 100) × 0.20
```

**Интерпретация:**

| Score | Статус | Цвет |
|-------|--------|------|
| 80–100 | Готов | 🟢 |
| 60–79 | Почти готов | 🟡 |
| 40–59 | Требует работы | 🟠 |
| 0–39 | Не готов | 🔴 |

Видят: ученик (свой), учитель (по классу), директор (по 10–11 классам школы).

## Рейтинг и перцентиль

### Внутри класса
- Анонимный: «3-е место из 24 учеников» (имена скрыты: «Ученик 1», «Ученик 2»)
- Учитель видит полный рейтинг с именами

### По платформе
- Национальный перцентиль: «Вы лучше, чем 68% учеников Zedly»
- Региональный перцентиль: «В Ташкенте вы в топ-15%»
- Порог отображения: минимум 100 попыток в пуле (иначе — «Рейтинг появится позже»)
- Обновляется ежедневно из `analytics_snapshots`

### Публичный рейтинг школ
- Средний NTT Readiness Score 11-х классов по школам (только агрегаты)
- Директор может скрыть школу из публичного рейтинга (opt-out)
- Имена учеников в публичный рейтинг не попадают никогда

| `analytics_snapshots` | `snapshot_type = ntt_readiness`, обновляется 1 раз в сутки |

> `ntt_question_pool` содержит **только вопросы уровня 3–5 по Bloom's Taxonomy**. Уровни 1–2 запрещены — согласовано с `ai_test_generation.md`.

> Таблица `ntt_answers` партиционирована по `(school_id, month)` аналогично `answers`.

## Граничные случаи

### GC-1: Таймер истёк — ученик не нажал «Завершить»
Background job каждые 60 сек проверяет `started_at + 90 min ≤ NOW()`. Принудительно закрывает сессию. Засчитываются все автосохранённые ответы. Незафиксированный в момент закрытия ответ — не засчитывается. Статус: `completed` (не `expired`).

### GC-2: Потеря соединения в середине теста
Таймер идёт на сервере. При reconnect в пределах оставшегося времени — сессия восстанавливается, ответы на месте, таймер продолжается. После истечения 90 минут — сессия уже закрыта, отображается результат.

### GC-3: Попытка начать вторую сессию одновременно
Если `status = in_progress` для этого `student_id` и профиля: новая сессия не создаётся. Редирект на активную сессию.

### GC-4: Попытка пройти НТТ offline
НТТ-режим недоступен offline — кнопка «Начать» заблокирована без интернета. Сообщение: «НТТ-симулятор требует подключения к интернету». Обычные тесты в offline не затрагиваются.

### GC-5: В пуле меньше 60 уникальных вопросов для профиля
Симулятор помечается как недоступный. UI: «Симулятор по этому профилю временно недоступен». Алерт команде Zedly. Минимальный рекомендуемый пул для качественной ротации: 200 вопросов на профиль.

### GC-6: Ученик видит повторяющиеся вопросы в разных попытках
При каждой новой попытке вопросы перемешиваются из пула случайно. Вопросы из последних 2 попыток ученика исключаются из следующей выборки (если пул достаточно большой).

### GC-7: Итоговый балл уходит в минус (штрафы)
Минимальный итог = 0. Расчёт: `max(0, correct × 1 − wrong × 0.25)`. В разборе каждый штраф виден явно.

### GC-8: Перцентиль при малом числе пользователей
Менее 100 попыток в пуле → перцентиль не отображается, показывается: «Рейтинг появится, когда больше учеников пройдут симулятор».

### GC-9: Ученик 8–9 класса
Симулятор доступен всем классам. Для 8–9 класса — дополнительный текст: «Тренируйтесь заблаговременно». NTT Readiness Score рассчитывается, но в дашборде директора приоритет — 10–11 класс.

### GC-10: Родитель получает уведомление о попытке
После завершения сессии — Telegram-пуш родителю: «Ваш ребёнок прошёл НТТ-симулятор: 43/60 (↑+5 с прошлой попытки)». Если Telegram не привязан — уведомление не отправляется (не блокирует завершение).

## Acceptance Criteria

### AC-1: Принудительное завершение по таймеру
```gherkin
Given: Ученик начал симулятор, ответил на 45 из 60 вопросов
When: На сервере фиксируется started_at + 90 мин
Then: Сессия автоматически закрывается со статусом completed
And: Засчитываются ответы на 45 вопросов; 15 неотвеченных = 0
And: Ученик видит результат и разбор ошибок
And: Принудительное завершение неотличимо от добровольного в интерфейсе
```

### AC-2: Штрафные баллы
```gherkin
Given: Штрафы включены; ученик ответил: 40 правильно, 15 неправильно, 5 пропустил
When: Сессия завершается
Then: Итоговый балл = max(0, 40 − 15 × 0.25) = 36.25
And: В разборе каждый неправильный ответ помечен «−0.25»
And: Пропущенные — 0 баллов без штрафа
```

### AC-3: Восстановление сессии после разрыва
```gherkin
Given: Ученик ответил на 30 вопросов и потерял интернет на 5 минут
When: Интернет восстанавливается, ученик открывает страницу
Then: Сессия восстанавливается с вопроса 31
And: Таймер показывает оставшееся время (уменьшенное на 5 минут)
And: Все 30 ответов на месте
```

### AC-4: Разбор ошибок по темам
```gherkin
Given: Ученик завершил симулятор с 15 ошибками
When: Открывает «Разбор ошибок»
Then: Ошибки сгруппированы по темам с процентом ошибок
And: Для каждой ошибки виден правильный ответ
And: Доступна кнопка «Создать тест по слабым темам»
And: Клик запускает AI-генерацию теста из топ-3 слабых тем
```

### AC-5: Перцентиль
```gherkin
Given: Ученик набрал 43/60, в пуле 500+ завершённых попыток
When: Просматривает результат
Then: Отображается национальный перцентиль
And: Региональный перцентиль отображается отдельно
And: Имена других учеников не раскрываются
```

### AC-6: NTT Readiness Score в дашборде учителя
```gherkin
Given: 10 учеников класса прошли симулятор хотя бы по 1 разу
When: Учитель открывает дашборд
Then: Отображается средний NTT Readiness Score класса с цветовым статусом
And: Гистограмма распределения баллов по классу
And: Список учеников, не прошедших симулятор + кнопка «Напомнить»
```

### AC-7: НТТ недоступен offline
```gherkin
Given: Ученик открывает НТТ-симулятор без интернета
When: Пытается нажать «Начать»
Then: Кнопка неактивна; сообщение «НТТ-симулятор требует подключения к интернету»
And: Раздел обычных offline-тестов работает нормально
```

### AC-8: Изоляция кастомного вопросного пула
```gherkin
Given: Учитель школы A добавил кастомные вопросы (school_id = A)
When: Ученик школы B проходит симулятор
Then: Вопросы школы A не включаются в выборку школы B
And: Обе школы используют общий пул (school_id IS NULL)
```

## KPI

| Метрика | Цель | Критический порог |
|---------|------|------------------|
| Completion rate (завершил / начал) | ≥ 80% | < 60% = проблема UX |
| Среднее число попыток на ученика в месяц | ≥ 3 | < 1.5 = нет retention |
| Рост балла между 1-й и 3-й попыткой | +5–10 баллов | 0 = нет обучающего эффекта |
| Учителя, назначившие симулятор 10–11 кл. | ≥ 50% активных | < 20% = не встроено в практику |
| Время загрузки экрана симулятора | ≤ 1.5 сек | > 3 сек |
| Родительские уведомления (доставлено / отправлено) | ≥ 90% | < 80% = проблема Telegram-интеграции |
