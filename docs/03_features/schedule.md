# Schedule — Расписание

**Файл:** `docs/03_features/schedule.md`  
**Приоритет:** Phase 1  
**Статус:** Production Blueprint

---

## 1. Цель

Управление уроками по дням недели с жёсткой связкой:
`teacher -> subject -> class`.

---

## 2. Таблицы

- `teacher_subjects`
- `teacher_class_assignments`
- `schedules`

---

## 3. Пример назначения

Teacher Иванов:
- Биология: 11А, 11Б, 9В, 10А
- Химия: 11В, 11Г

Это отражается через набор записей в `teacher_class_assignments`.

---

## 4. Интеграция

- Schedule является источником истины для `attendance_sessions`.
- На основе schedule создаются слоты журнала (урок/дата/предмет).

---

## 5. Ограничения

- Нельзя создать два урока в одно время для одного класса.
- Нельзя создать два урока в одно время для одного учителя.
- Все проверки выполняются в рамках `school_id`.
