# Analytics Dashboard — Единый дашборд аналитики

**Файл:** `docs/03_features/analytics_dashboard.md`  
**Приоритет:** MVP  
**Статус:** Production Blueprint

---

## Цель фичи

Единое окно аналитики для каждой роли — от ученика до Министерства. Каждая роль видит только свой уровень данных: учитель — свой класс, директор — школу, РОНО — район, Министерство — страну. Персональные данные не поднимаются выше директора.

**Ключевое архитектурное правило:** дашборд **всегда** читает из `analytics_snapshots`, обновляемых каждые 15 минут. Прямые запросы к `answers` или `test_results` из UI — запрещены. Это защищает производительность при 10,000+ одновременных сессий.

---

## Кто использует (роли и доступный уровень данных)

| Роль | Видит | Не видит |
|------|-------|----------|
| Учитель | Результаты своего класса по своим тестам | Результаты других классов / других учителей |
| Директор | Агрегированная аналитика всей школы | Персональные данные учеников (только агрегаты) |
| РОНО-инспектор | Агрегированная аналитика всех школ района | Данные отдельных учеников и классов |
| Министерство | Национальная и региональная аналитика | Данные ниже уровня школы |
| Родитель | Результаты только своего ребёнка | Результаты других учеников |
| Ученик | Собственные результаты + место в классе (анонимный рейтинг) | Имена других учеников в рейтинге |

## Дашборды по ролям

---

### 🧑‍🏫 Дашборд учителя

#### Виджеты (MVP)

**1. Средний балл класса по последнему тесту**
- Число от 0 до 100% + цветовая индикация: ≥75% зелёный, 50–74% жёлтый, <50% красный
- Сравнение с предыдущим тестом: стрелка вверх/вниз + Δ%

**2. Топ-5 тем с наибольшим количеством ошибок**
- Список тем в порядке убывания процента ошибок
- Пример: «Закон Ньютона II — 67% ошибок», «Скорость и ускорение — 54% ошибок»
- Кнопка «Создать тест по этой теме» (открывает AI-генерацию с предзаполненной темой)

**3. Прогресс учеников по времени (sparkline)**
- Мини-график средней успеваемости класса по последним 5 тестам
- При наведении: дата и балл каждого теста

**4. Таблица результатов класса**
- Строки: ученики (имя, фамилия)
- Колонки: последние 5 тестов + средний балл
- Ученики без результата: серая ячейка «Не прошёл»
- Сортировка по любой колонке

**5. Процент завершения последнего теста**
- «18 из 24 учеников прошли тест» + прогресс-бар
- Список не прошедших (с кнопкой «Напомнить» — Telegram-уведомление)

**6. Активные тесты**
- Список тестов с дедлайном + countdown до конца

#### Виджеты (Phase 2)

- **NTT Readiness Score**: прогнозная готовность класса к НТТ (0–100)
- **Heatmap слабых тем**: матрица «ученик × тема» с цветовой интенсивностью
- **Сравнение с другими классами школы** (если директор разрешил)

---

### 🏫 Дашборд директора (только платный план)

#### Виджеты (MVP)

**1. Сводка по школе**
- Средний балл по школе за текущий месяц / четверть
- Количество активных учителей / всего учителей
- Количество пройденных тестов за период

**2. Успеваемость по классам (heatmap)**
- Строки: классы (1А, 1Б, ..., 11В)
- Колонки: предметы
- Ячейки: средний балл, цвет от красного до зелёного
- Клик по ячейке → детальная аналитика класса по предмету

**3. Рейтинг предметов (слабые → сильные)**
- Горизонтальный бар-чарт: предмет → средний балл по школе
- Позволяет директору видеть, где нужно усиление

**4. Активность учителей**
- Таблица: учитель → количество тестов за месяц → последняя активность
- Учителя без активности > 14 дней: красная метка
- Это триггер для разговора директора с учителем

**5. Триггер конверсии (freemium → лицензия)**
- Если ≥ 5 учителей на freemium: «5 учителей уже активно используют Zedly. Перейдите на школьную лицензию и разблокируйте полную аналитику»
- Если ≥ 7 учителей: «92% ваших учителей на Zedly — пора объединить школу на одной платформе»
- Кнопка «Купить лицензию» → Payme/Click

**6. Экспорт отчёта для РОНО**
- Кнопка «Сформировать квартальный отчёт»
- Форматы: PDF (визуальный) + Excel (таблица для РОНО)
- Отчёт включает: сводку по классам, динамику за квартал, топ слабых тем

#### Виджеты (Phase 2)

- **Сравнение с прошлым кварталом**: Δ по каждому классу
- **Сравнение с районом**: анонимный бенчмарк (если РОНО подключён)
- **Прогноз результатов НТТ** по 11-м классам

---

### 🏢 Дашборд РОНО-инспектора

#### Виджеты (MVP)

**1. Карточки школ района**
- Каждая школа: название, средний балл, динамика (↑↓), количество активных учителей
- Сортировка: по баллу / по динамике / по активности

**2. Топ-3 отстающие школы**
- Автоматический расчёт по среднему баллу за последние 3 месяца
- Возможность отправить директору уведомление прямо из РОНО-панели

**3. Динамика по району за четверть/год**
- Линейный график: средний балл района по времени
- Возможность наложить несколько школ для сравнения

**4. Аналитика по предметам**
- Какие предметы системно западают по всему району
- Агрегированные данные — без привязки к конкретным учителям

**5. Экспорт в PDF/Excel**
- Шаблон отчёта в формате, принятом Министерством просвещения Узбекистана
- Автоматическая генерация с подписью «Сформировано платформой Zedly»
- Расписание: ежеквартальный автоматический отчёт (отправляется в Министерство напрямую — Phase 3)

---

### 🏛 Дашборд Министерства

#### Виджеты (Phase 3)

**1. Региональная карта успеваемости**
- Интерактивная карта Узбекистана (14 регионов)
- Цвет региона = средний балл
- Клик → детализация по РОНО региона

**2. Национальные тренды по предметам**
- За последний год: какие предметы улучшились / ухудшились по стране
- Детализация по регионам

**3. Подготовленность к НТТ (NTT Readiness Index)**
- Прогнозный показатель: % 11-классников, готовых пройти НТТ
- Детализация: регион → РОНО → школа (без имён учеников)

**4. Экспорт для государственных отчётов**
- Форматы: PDF с гербом / Excel / CSV
- Данные анонимизированы на уровне ниже школы

## Архитектура (критически важно)

### Принцип чтения из снапшотов

```
[Ученики / Учителя проходят тесты] 
        ↓ пишут в raw-таблицы
[answers, test_results, test_sessions]
        ↓ background job каждые 15 мин
[analytics_snapshots]
        ↓ читает дашборд
[UI дашборда]
```

**Запрещено:** `SELECT AVG(score) FROM answers WHERE school_id = X` из UI-запроса.  
**Разрешено:** `SELECT * FROM analytics_snapshots WHERE school_id = X AND snapshot_type = 'class_summary'`.

### Структура `analytics_snapshots`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | — |
| `school_id` | UUID | Изоляция по школе |
| `snapshot_type` | ENUM | `class_summary`, `school_summary`, `district_summary`, `subject_weakness`, `teacher_activity`, `ntt_readiness` |
| `entity_id` | UUID | class_id / school_id / district_id — в зависимости от типа |
| `entity_level` | ENUM | `class`, `school`, `district`, `national` |
| `data_json` | JSONB | Преагрегированные данные (avg_score, completion_rate, weak_topics, etc.) |
| `period_start` | DATE | Начало периода снапшота |
| `period_end` | DATE | Конец периода |
| `snapshot_at` | TIMESTAMP | Когда создан снапшот |
| `is_current` | BOOL | `true` для последнего снапшота каждого типа |

### Background Jobs для обновления снапшотов

| Job | Расписание | Обновляет |
|-----|-----------|-----------|
| `aggregate_class_snapshots` | Каждые 15 мин | `class_summary` для всех классов с активностью |
| `aggregate_school_snapshots` | Каждые 30 мин | `school_summary` на основе `class_summary` |
| `aggregate_district_snapshots` | Каждый час | `district_summary` на основе `school_summary` |
| `compute_ntt_readiness` | 1 раз в сутки | `ntt_readiness` для 10–11 классов |
| `compute_teacher_activity` | Каждые 30 мин | `teacher_activity` для триггера конверсии |

## Граничные случаи

### GC-1: Школа не провела ни одного теста (empty state)
- Учитель видит: иллюстрацию + «Создайте первый тест, чтобы увидеть аналитику»
- Директор видит: «Ни один учитель ещё не провёл тест. Пригласите учителей»
- РОНО видит: школа отображается в списке серым цветом с меткой «Нет данных»
- Нет пустых графиков, нет нулей — только дружелюбный empty state

### GC-2: Директор пытается получить данные другой школы
- API-запрос к `analytics_snapshots` фильтруется через RLS по `school_id` из JWT-токена
- Возвращает 403 Forbidden + запись в `audit_log`
- UI не показывает данные — вместо этого «Нет доступа»

### GC-3: Снапшот устарел (background job упал)
- Если `snapshot_at` > 30 минут назад для активной школы: предупреждение в UI «Данные могут быть устаревшими. Обновляем...»
- Алерт команде Zedly через мониторинг
- Fallback: показать последний доступный снапшот с пометкой времени

### GC-4: Учитель переключается между классами
- Дашборд перезагружает данные для выбранного класса
- Переключение ≤ 500 мс (из снапшотов, не из raw данных)

### GC-5: РОНО запрашивает аналитику по школе без лицензии Zedly
- Школа без лицензии не отправляет данные в РОНО-снапшот
- РОНО видит: «Школа [название] не подключена к Zedly» + кнопка «Пригласить директора»
- Это дополнительный триггер продаж: РОНО давит на директора подключиться

### GC-6: Открытые ответы ещё не проверены учителем
- Тест с открытыми ответами имеет статус `awaiting_review` для части учеников
- В снапшоте класса: балл считается только по проверенным ответам + предупреждение «N учеников ожидают проверки»
- Итоговый средний балл помечается `*` (предварительный)

### GC-7: Экспорт отчёта РОНО в пиковое время
- Генерация PDF для 50 школ может занять 10–30 секунд
- Реализация: async job → пользователь получает уведомление «Отчёт готов» вместо ожидания
- Готовый файл доступен в разделе «Мои отчёты» 30 дней

### GC-8: Данные за период без активности (каникулы)
- График показывает разрыв в линии за период без тестов
- Не интерполирует данные — честный пустой период
- Tooltip: «В этот период тесты не проводились»

## Acceptance Criteria

### AC-1: Время загрузки дашборда
```gherkin
Given: Учитель авторизован, в классе 30 учеников, 10 тестов за квартал
When: Открывает дашборд
Then: Все виджеты отображаются в течение 1 секунды
And: Данные актуальны (снапшот не старше 15 минут)
```

### AC-2: Изоляция данных директора
```gherkin
Given: Директор школы A авторизован
When: Делает запрос к аналитике школы B (через API или URL-манипуляцию)
Then: Система возвращает 403 Forbidden
And: Событие записывается в audit_log с user_id, school_id, timestamp
And: Директор не получает информации о существовании школы B
```

### AC-3: Триггер конверсии директора
```gherkin
Given: В школе 7 учителей активно используют Zedly на freemium
When: Директор входит в панель школы
Then: Отображается баннер «92% ваших учителей на Zedly — перейдите на школьную лицензию»
And: Кнопка «Купить лицензию» ведёт на страницу оплаты Payme/Click
And: Баннер нельзя скрыть, только закрыть (и он появится снова через 3 дня)
```

### AC-4: Топ слабых тем учителя
```gherkin
Given: Класс прошёл тест по физике, 60% ошибок в теме «Закон Ома»
When: Учитель открывает дашборд
Then: «Закон Ома» отображается первым в списке топ слабых тем
And: Рядом кнопка «Создать тест по этой теме» (открывает AI-генерацию с предзаполненной темой)
```

### AC-5: Экспорт отчёта РОНО
```gherkin
Given: РОНО-инспектор хочет квартальный отчёт по 20 школам района
When: Нажимает «Сформировать квартальный отчёт»
Then: Через ≤ 30 секунд получает уведомление «Отчёт готов»
And: PDF содержит: сводку по всем школам, динамику за квартал, топ слабых тем
And: Excel содержит сырые агрегаты для дальнейшей обработки
And: Отчёт доступен в «Мои отчёты» 30 дней
```

### AC-6: Empty state
```gherkin
Given: Учитель только зарегистрировался, тестов не создавал
When: Открывает дашборд
Then: Видит иллюстрацию и текст «Создайте первый тест, чтобы увидеть аналитику»
And: Нет пустых графиков, нулевых значений или сломанной вёрстки
```

### AC-7: Устаревший снапшот
```gherkin
Given: Background job не обновлял снапшоты 35 минут (сбой)
When: Учитель открывает дашборд
Then: Отображается предупреждение «Данные могут быть устаревшими. Последнее обновление: 35 минут назад»
And: Данные всё равно отображаются (последний доступный снапшот)
And: Команда Zedly получает алерт в мониторинге
```

### AC-8: Родитель видит только своего ребёнка
```gherkin
Given: Родитель авторизован через Telegram
When: Открывает дашборд
Then: Видит только результаты своего ребёнка
And: API возвращает 403 при запросе данных любого другого student_id
```

## KPI дашборда

| Метрика | Цель | Критический порог |
|---------|------|------------------|
| Время загрузки (P95) | ≤ 1 сек | > 3 сек = проблема |
| Актуальность данных | Снапшот ≤ 15 мин | > 30 мин = алерт |
| DAU дашборда (учителя) | ≥ 60% активных учителей заходят ежедневно | < 30% = фича не используется |
| Конверсия директор → лицензия | ≥ 25% директоров с 5+ учителями | < 10% = слабый триггер |
| Экспорт РОНО (время генерации) | ≤ 30 сек | > 2 мин = проблема |
