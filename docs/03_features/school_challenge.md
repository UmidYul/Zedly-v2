# Межшкольный челлендж — Публичное соревнование между школами

**Файл:** `docs/03_features/school_challenge.md`  
**Приоритет:** Phase 2 (вирусная механика роста)  
**Статус:** Production Blueprint

---

## Цель фичи

Директор школы A публично бросает вызов школе B: обе школы проходят один и тот же тест в течение 48 часов, результаты публичны на уровне школ. Родители следят в Telegram-чатах, ученики мотивированы играть за честь школы. Каждый челлендж — это новый виток регистраций из школы-соперника, которая ещё не в Zedly.

**Почему это вирусный двигатель:** Директор школы B получает сообщение «Школа [название] бросила вам вызов» — это социальное давление. Родители пишут в чаты: «Наша школа участвует?» Ученики делятся карточками результатов. Один челлендж = от 20 до 200 новых регистраций из обеих школ.

---

## Кто использует

| Роль | Действие |
|------|----------|
| Директор (инициатор) | Создаёт челлендж, выбирает соперника, тест, правила |
| Директор (вызванный) | Принимает или отклоняет вызов в течение 72 часов |
| Учитель | Назначает участие своего класса, следит за ходом |
| Ученик | Проходит тест, видит вклад в рейтинг школы |
| Родитель | Получает уведомления о ходе и итоге, делится карточкой |
| Публика (незарегистрированные) | Видит публичную страницу челленджа (read-only) |

## User Flow

### Flow A: Директор создаёт челлендж

```
1. Директор → «Челлендж» → «Бросить вызов»
2. Шаг 1 — Выбор соперника:
   - Поиск по названию / городу
   - «Популярные соперники» (близкие по размеру, тот же район)
   - Если школа B не в Zedly → «Пригласить школу»:
     email директору B: «Школа [A] вызывает вас на Zedly. Зарегистрируйтесь и примите вызов»
3. Шаг 2 — Выбор теста:
   - Из библиотеки учителей школы A
   - Из Marketplace
   - «Быстрый старт»: стандартный НТТ-подобный тест (30 вопросов, 45 мин)
4. Шаг 3 — Параметры:
   - Предмет и класс: «Физика, 10 класс»
   - Участники: весь класс / выбранные классы / добровольно
   - Продолжительность: 48 часов (MVP фиксировано)
   - Публичность: «Результаты видны всем» (по умолчанию)
5. Предпросмотр → «Отправить вызов»
6. Директор B получает Telegram + email:
   «⚔️ Лицей №1 бросил вам вызов по физике! Ответьте до [дата+72ч]»
```

### Flow B: Принятие / отклонение вызова

```
Принятие:
1. Директор B нажимает «Принять вызов»
2. started_at = NOW(), deadline_at = NOW() + 48ч фиксируются на сервере
3. Учителя обеих школ: «Ваша школа участвует в челлендже по физике! Дедлайн: [дата]»
4. Ученики обеих школ: «⚔️ Сыграй за честь школы! Тест по физике → [ссылка]»

Отклонение: статус declined, публичной записи нет, директор A уведомлён.

Таймаут 72ч: статус expired, вызов аннулируется, публичной записи нет.
```

### Flow C: Ученик проходит тест челленджа

```
1. Баннер в приложении:
   ┌────────────────────────────────────────────────────────┐
   │ ⚔️ ИДЁТ ЧЕЛЛЕНДЖ · Физика 10 класс                   │
   │ Лицей №1 vs Гимназия №7 · Осталось: 31 ч 22 мин      │
   │ Счёт: Лицей №1: 68%  ·  Гимназия №7: 71%             │
   │              [Пройти тест]                             │
   └────────────────────────────────────────────────────────┘
2. Ученик проходит тест стандартным образом (Test Engine, режим challenge)
3. После завершения: результат мгновенно учитывается в счёте школы
4. Экран результата:
   «Ваш результат: 85% ✅. Вы помогли школе! Лицей №1 теперь ведёт: 72% vs 71%»
5. Персональная карточка-результат с брендингом челленджа → кнопка «Поделиться»
```

### Flow D: Публичная страница челленджа (live)

```
URL: https://zedly.uz/challenge/{challenge_id}
Доступна без авторизации

  ┌─────────────────────────────────────────────────────────┐
  │  ⚔️  МЕЖШКОЛЬНЫЙ ЧЕЛЛЕНДЖ · ФИЗИКА 10 КЛАСС            │
  │  Лицей №1 (Ташкент)    vs    Гимназия №7 (Ташкент)    │
  │                                                         │
  │  [████████████░░░] 72%      vs    [██████████░░░] 68%  │
  │  24 / 30 участников                19 / 28 участников  │
  │                                                         │
  │  ⏱ Осталось: 14 ч 37 мин                              │
  │                                                         │
  │  [Вступить в Zedly и присоединиться к команде]         │
  └─────────────────────────────────────────────────────────┘

Данные обновляются каждые 60 сек из challenge_snapshots.
Имена учеников не отображаются — только агрегаты по школе.
Кнопка «Вступить» → страница регистрации.
```

### Flow E: Завершение и итоговая карточка

```
1. 48ч истекло → background job финализирует
2. Победитель: школа с более высоким avg_score участников
   Ничья: разница < 1%
3. Итоговая карточка обеим школам (Telegram + платформа):
   ┌─────────────────────────────────────────────────────────┐
   │  🏆 ПОБЕДИТЕЛЬ: Лицей №1                               │
   │  Лицей №1: 72% (24 участника)                         │
   │  Гимназия №7: 68% (19 участников)                     │
   │  Физика 10 класс · 27 февраля 2026                     │
   └─────────────────────────────────────────────────────────┘
4. Директора получают полный отчёт: распределение баллов + слабые темы
5. Победители-ученики → сертификат «Межшкольный чемпион»
6. Публичная страница хранится 30 дней после завершения
```

## Вирусная механика

| Момент | Действие | Охват |
|--------|----------|-------|
| Вызов отправлен | Директор A анонсирует в родительском чате | ~50 родителей |
| Вызов принят | Обе школы рассылают ученикам | ~60 учеников + их родители |
| Ученик проходит тест | Делится карточкой | 100+ человек в Telegram/WhatsApp |
| Итоговая карточка | Директор делится в соцсетях | Вся аудитория обеих школ |
| Незарегистрированные видят страницу | Кнопка «Вступить в Zedly» | Конверсия в новых пользователей |

**Приглашение незарегистрированной школы:** горячий лид — директор мотивирован конкурировать, а не просто изучать платформу. Конверсия ожидаемо выше обычной cold outreach в 3–5 раз.

## Зависимости от данных (таблицы)

| Таблица | Ключевые поля |
|---------|---------------|
| `challenges` | `id`, `initiator_school_id`, `opponent_school_id`, `test_id`, `status` (pending/active/completed/declined/expired), `started_at`, `deadline_at`, `is_public`, `winner_school_id` |
| `challenge_participants` | `id`, `challenge_id`, `student_id`, `school_id`, `test_session_id`, `score_percent`, `participated_at` |
| `challenge_snapshots` | `id`, `challenge_id`, `school_id`, `avg_score`, `participant_count`, `snapshot_at` — обновляется каждые 60 сек |
| `challenge_invites` | `id`, `challenge_id`, `invitee_email`, `invitee_school_name`, `sent_at`, `opened_at`, `registered_at` |

RLS: директор школы X видит челленджи, где `initiator_school_id = X OR opponent_school_id = X`. Публичная страница читает только из `challenge_snapshots` и не раскрывает персональные данные.

## Граничные случаи

### GC-1: Школа B не отвечает 72 часа
Статус `expired`. Директор A уведомлён. Публичной записи нет. Может создать новый вызов.

### GC-2: Директор A отзывает вызов до принятия
Статус `cancelled`. Директор B уведомлён. Не позорит отменившего публично.

### GC-3: В школе B нет учеников нужного класса на платформе
Предупреждение при принятии: «В вашей школе нет 10-классников в Zedly. Продолжить?» Директор решает сам.

### GC-4: Ученик пытается пройти тест дважды
Одна запись на (challenge_id, student_id). При повторной попытке: «Вы уже участвовали. Ваш результат учтён». Счёт школы не пересчитывается.

### GC-5: Школа теряет платный план в ходе челленджа
Текущий челлендж доводится до конца. Новые челленджи — только с платным планом.

### GC-6: Тест удалён во время активного челленджа
Удаление заблокировано: «Тест используется в активном челлендже». Разрешено после завершения.

### GC-7: Публичная страница под нагрузкой
Читает из `challenge_snapshots` (TTL 60 сек). CDN кэширует HTML на 60 сек. Один запрос к БД в минуту независимо от числа посетителей.

### GC-8: Ничья (разница < 1%)
Карточка: «🤝 Ничья! Обе школы показали выдающиеся результаты». `winner_school_id = NULL`. Публичный рейтинг не изменяется.

### GC-9: Поиск приводит к дублирующемуся приглашению
Если школа B уже в Zedly — вызов отправляется директору напрямую (не email, а Telegram/платформа). Повторное приглашение незарегистрированной школе (уже есть pending) блокируется.

### GC-10: Участник из неверной школы
`challenge_participants` принимает только `school_id ∈ {initiator_school_id, opponent_school_id}`. Ученик из третьей школы не может засчитать результат ни одной из команд.

## Acceptance Criteria

### AC-1: Создание и отправка вызова
```gherkin
Given: Директор школы A с платным планом
When: Создаёт челлендж — школа B, тест 30 MCQ, физика 10 класс
Then: challenges.status = pending
And: Директор B получает Telegram-уведомление в течение 60 сек
And: Вызов виден в интерфейсе A со статусом «Ожидает ответа»
```

### AC-2: Принятие и старт
```gherkin
Given: Вызов в статусе pending
When: Директор B нажимает «Принять»
Then: challenges.status = active
And: started_at = NOW(), deadline_at = NOW() + 48ч
And: Учителя и ученики обеих школ получают уведомления
```

### AC-3: Публичная страница в реальном времени
```gherkin
Given: 10 учеников школы A уже прошли тест
When: Незарегистрированный пользователь открывает страницу
Then: Отображается текущий avg_score обеих школ (данные ≤ 60 сек)
And: Имена учеников не отображаются
And: Кнопка «Вступить в Zedly» присутствует
```

### AC-4: Защита от двойного участия
```gherkin
Given: Ученик уже прошёл тест челленджа с результатом 80%
When: Пытается пройти снова
Then: Система показывает «Вы уже участвовали. Результат: 80%»
And: Новая запись в challenge_participants не создаётся
And: Счёт школы не изменяется
```

### AC-5: Завершение и победитель
```gherkin
Given: 48ч истекло; школа A avg 72%, школа B avg 68%
When: Background job финализирует
Then: challenges.status = completed, winner_school_id = школа A
And: Обе школы получают карточку в Telegram
And: Публичная страница: «🏆 Победитель: Лицей №1»
And: Победители-участники получают сертификат
```

### AC-6: Таймаут вызова
```gherkin
Given: Прошло 72 часа без ответа директора B
When: Background job проверяет
Then: challenges.status = expired
And: Директор A получает уведомление
And: Публичной страницы для этого челленджа нет
```

### AC-7: Ничья
```gherkin
Given: Школа A avg 71.3%, школа B avg 71.0%
When: Челлендж завершается
Then: winner_school_id = NULL
And: Карточка: «🤝 Ничья!»
And: Рейтинг обеих школ не изменяется
```

### AC-8: Приглашение незарегистрированной школы
```gherkin
Given: Директор A ищет школу B, которой нет в Zedly
When: Вводит email директора B и нажимает «Пригласить»
Then: Директор B получает email с персональной ссылкой
And: Ссылка ведёт на регистрацию с предзаполненными данными челленджа
And: challenge_invites фиксирует sent_at
```

## KPI

| Метрика | Цель | Критический порог |
|---------|------|------------------|
| Acceptance rate (принято / отправлено) | ≥ 60% | < 30% = школ мало или вызовы неинтересны |
| Completion rate учеников (прошли / записаны) | ≥ 70% | < 40% = нет мотивации |
| Новые регистрации с публичной страницы на 1 челлендж | ≥ 15 | < 5 = вирусность не работает |
| Шейринг карточки результата учениками | ≥ 40% участников | < 15% |
| Директора, создавшие ≥ 2 челленджа | ≥ 50% участвовавших | < 20% = не стало привычкой |
| Незарегистрированные школы → регистрация | ≥ 20% от приглашённых | < 5% = плохой конверт |
