# Journal — Электронный журнал

**Файл:** `docs/03_features/journal.md`  
**Приоритет:** Phase 1  
**Статус:** Production Blueprint

---

## 1. Назначение

Отдельный модуль для ведения:
- оценок по предметам,
- посещаемости,
- комментариев учителя,
- четвертей.

---

## 2. Ключевые таблицы

- `journal_terms`
- `journal_entries`
- `grading_rules`

Связанные:
- `attendance_records` (для присутствия),
- `tests`/`test_sessions` (авто-импорт оценок).

---

## 3. 5-балльная система

Поддерживается только 5-балльная шкала.
Храним:
- `raw_score_percent` (для аналитики и обратной совместимости),
- `grade_5` (основная оценка в журнале).

Преобразование выполняется через `grading_rules`.

---

## 4. Бизнес-правила

- Любая запись в журнале содержит `school_id`, `class_id`, `subject_id`, `teacher_id`.
- Teacher может ставить оценки только для своих assignments.
- Director может корректировать запись с обязательным audit reason.

---

## 5. Аналитика

Journal feed участвует в витринах:
- успеваемость по предметам,
- динамика по четвертям,
- корреляция с посещаемостью.
