# AI-Генерация тестов — Автоматическое создание вопросов из учебников

**Файл:** `docs/03_features/ai_test_generation.md`  
**Приоритет:** MVP (killer feature — барьер для конкурентов 12–18 месяцев)  
**Статус:** Production Blueprint

---

## Цель фичи

Учитель загружает учебник (PDF/текст) или вставляет фрагмент темы — AI за 30–60 секунд генерирует готовый тест. Учитель редактирует, принимает или отклоняет вопросы. Тест сохраняется в библиотеку и может быть опубликован в Marketplace.

**Почему это барьер для конкурентов:**
- Обучение модели на узбекских учебниках (узбекский + русский язык) = 6–9 месяцев данных
- Накопленная история «принято/отклонено» учителями = обучающая выборка, которой нет у других
- Интеграция с педагогической программой Узбекистана = не воспроизвести без глубокого domain knowledge
- Итого: 12–18 месяцев для любого конкурента, начинающего с нуля

---

## Кто использует

| Роль | Действие |
|------|----------|
| Учитель | Загружает материал, редактирует/принимает/отклоняет вопросы |
| Zedly-команда (curator) | Проверяет вопросы с флагом `needs_review`, улучшает качество |
| AI-модель | Генерирует черновик вопросов (не финальный автор) |

---

## Флоу генерации (пошагово)

### Шаг 1 — Загрузка материала

Учитель выбирает один из трёх методов ввода:

| Метод | Описание | Ограничения |
|-------|----------|-------------|
| **PDF-загрузка** | Учебник или глава целиком | Макс. 50 MB, текстовый PDF (не scan) |
| **Текстовая вставка** | Copy-paste параграфа или темы | 200–10,000 символов |
| **Ссылка на главу** | URL учебника с открытым доступом (Phase 2) | Только одобренные домены |

Язык определяется автоматически (узбекский / русский). Оба языка поддерживаются с первого дня.

Если загружен scan (изображение-PDF без текстового слоя): система выдаёт ошибку «Файл содержит только изображения. Загрузите PDF с текстовым слоем или вставьте текст вручную». OCR — не в MVP.

---

### Шаг 2 — Настройка генерации

Перед запуском учитель указывает параметры:

| Параметр | Варианты | По умолчанию |
|----------|----------|-------------|
| Количество вопросов | 5 / 10 / 15 / 20 / задать вручную (1–30) | 10 |
| Типы вопросов | MCQ, Множественный выбор, Matching, Открытый ответ | MCQ (70%), Matching (20%), Открытый (10%) |
| Уровень сложности | Лёгкий / Средний / Сложный / Смешанный | Средний |
| Класс | 1–11 | Берётся из профиля учителя |
| Предмет | Выбор из справочника | Берётся из профиля учителя |
| Язык вопросов | Узбекский / Русский / Как в источнике | Как в источнике |

---

### Шаг 3 — Генерация (AI обработка)

```
Учитель нажимает «Сгенерировать» →
Система показывает прогресс-бар («Анализируем текст... Создаём вопросы...») →
Через 20–60 секунд: черновик из N вопросов
```

**Что делает AI внутри:**
1. Парсит и чистит текст (убирает колонтитулы, номера страниц, формулы без контекста)
2. Извлекает ключевые концепции, термины, факты
3. Для каждой концепции генерирует вопрос нужного типа
4. Для MCQ — создаёт 3 правдоподобных дистрактора (неправильных варианта)
5. Присваивает сложность (1–5 звёзд) и тему из учебной программы
6. Ставит флаги качества (см. ниже)

**Целевое время генерации:**
- 10 вопросов из 2,000 слов текста: ≤ 30 секунд
- 20 вопросов из 10,000 слов PDF: ≤ 90 секунд

---

### Шаг 4 — Редактирование черновика

Учитель видит список сгенерированных вопросов. Для каждого вопроса доступны действия:

| Действие | Иконка | Результат |
|----------|--------|-----------|
| **Принять** | ✅ | Вопрос добавляется в тест |
| **Отклонить** | ❌ | Вопрос удаляется, событие логируется для обучения модели |
| **Редактировать** | ✏️ | Открывается inline-редактор текста вопроса и вариантов |
| **Перегенерировать** | 🔄 | AI создаёт новый вопрос по той же концепции |
| **Изменить сложность** | ⭐ | Учитель переопределяет оценку AI |

Кнопки «Принять все» и «Отклонить все» с подтверждением.

---

### Шаг 5 — Сохранение и публикация

```
Учитель нажимает «Сохранить тест» →
Тест попадает в «Мои тесты» (приватный) →
Опционально: «Опубликовать в Marketplace» →
Если опубликован: проходит модерацию (автоматическая + ручная для спорных) →
После одобрения: доступен другим учителям
```

## Контроль качества вопросов

Каждый сгенерированный вопрос получает автоматические флаги:

| Флаг | Условие | Действие системы |
|------|---------|-----------------|
| `low_confidence` | AI уверен < 70% в правильности ответа | Жёлтая метка «Проверьте ответ» |
| `needs_review` | Вопрос содержит цифры/даты/имена — высокий риск ошибки | Отправляется Zedly-куратору |
| `language_mismatch` | Язык вопроса ≠ языку источника | Предупреждение учителю |
| `duplicate` | Похожий вопрос уже есть в тестах этого учителя (косинусное сходство > 0.9) | «Похожий вопрос уже существует» |
| `off_topic` | Концепция из вопроса не найдена в загруженном тексте | Флаг «Возможно, не по теме» |

**Порог публикации в Marketplace:**
- Вопрос не имеет флага `needs_review` или `low_confidence`
- Или учитель явно подтвердил: «Я проверил, всё верно»

## Оценка сложности (1–5 звёзд)

| Уровень | Описание | Bloom's Taxonomy |
|---------|----------|-----------------|
| ⭐ (1) | Определения, простые факты | Remember |
| ⭐⭐ (2) | Понимание концепции | Understand |
| ⭐⭐⭐ (3) | Применение в стандартной задаче | Apply |
| ⭐⭐⭐⭐ (4) | Анализ, сравнение | Analyze |
| ⭐⭐⭐⭐⭐ (5) | Синтез, оценка, НТТ-уровень | Evaluate / Create |

НТТ-симулятор использует только вопросы уровня 3–5.

## Marketplace тестов

### Как работает сетевой эффект
```
Учитель A публикует тест → 
Учитель B использует и оценивает → 
Рейтинг теста растёт → 
Учитель A получает «звёзды» (геймификация) → 
Больше учителей публикуют тесты → 
Ценность платформы растёт для всех
```

### Структура Marketplace

| Атрибут | Описание |
|---------|----------|
| Предмет | Математика, Физика, Узбекский язык, История и т.д. |
| Класс | 1–11 |
| Язык | Узбекский / Русский |
| Рейтинг | 1–5 звёзд (голоса учителей, использовавших тест) |
| Использований | Сколько раз тест был взят в работу |
| Автор | Псевдоним (реальное имя скрыто по умолчанию) |
| Источник | «Создан учителем» / «AI-генерация» / «AI + редактирование» |

### Правила Marketplace
- Учитель может публиковать тест только в рамках своей школы по умолчанию (privacy)
- Опция «Поделиться со всей платформой» — явное действие
- Нельзя копировать тест другого учителя без клонирования (клон = новый объект в собственности)
- Тест с рейтингом < 2 звёзд и > 10 использований автоматически отправляется на ревью

## Граничные случаи

### GC-1: Загружен не учебник, а посторонний документ (договор, художественный текст)
- AI не может извлечь педагогические концепции
- Система возвращает результат с флагом `off_topic` на 80%+ вопросов
- Пользователю показывается: «Текст не похож на учебный материал. Проверьте загруженный файл или попробуйте другой фрагмент»
- Генерация не блокируется — учитель сам решает, использовать или нет

### GC-2: AI генерирует фактически неверный вопрос
- Флаг `needs_review` на вопросах с датами, числами, именами
- Учитель обязан явно подтвердить вопросы с этим флагом перед публикацией
- После отклонения: данные уходят в `ai_feedback_log` для дообучения
- SLA куратора: проверить `needs_review` вопросы из Marketplace в течение 48 часов

### GC-3: Учебник на узбекском с кириллицей vs латиницей
- Узбекский язык имеет два стандарта: кириллица (школы) и латиница (официальный с 1993)
- Система принимает оба варианта, детектирует автоматически
- Вопросы генерируются в том же скрипте, что и источник
- При смешанном тексте: спрашивает учителя «Какой скрипт использовать для вопросов?»

### GC-4: PDF с формулами (математика, физика, химия)
- Формулы в PDF часто закодированы как изображения или специальные символы
- MVP: формулы в текстовом виде (LaTeX) обрабатываются; изображения формул — пропускаются
- Учителю показывается: «N формул в тексте не были обработаны. Добавьте вопросы по ним вручную»
- Phase 2: интеграция MathPix или аналогичного парсера формул

### GC-5: Слишком короткий текст (< 200 символов)
- Клиентская валидация до запуска генерации: «Текст слишком короткий. Добавьте минимум 200 символов»
- Генерация не запускается

### GC-6: Слишком длинный PDF (> 50 MB или > 100,000 слов)
- Клиентская валидация: отказ с сообщением «Файл слишком большой. Загрузите отдельную главу (до 50 MB)»
- Если прошёл client-side: сервер возвращает 413 Payload Too Large

### GC-7: AI-сервис недоступен (timeout / downtime)
- Фронт показывает spinner максимум 120 секунд
- При таймауте: «Генерация занимает дольше обычного. Попробуйте снова или создайте вопросы вручную»
- Задача помечается как `failed` в `ai_generation_jobs`, retry через 5 мин (3 попытки)
- Критический downtime: алерт команде через мониторинг

### GC-8: Учитель закрыл страницу во время редактирования черновика
- Черновики сохраняются в `question_drafts` с привязкой к `job_id`
- При повторном входе: «У вас есть незавершённая генерация от [дата]. Продолжить?»
- Черновик хранится 7 дней, затем автоудаляется

### GC-9: Дублирование — тест с идентичными вопросами уже в Marketplace
- Косинусное сходство вопросов > 0.9 → флаг `duplicate`
- Публикация в Marketplace блокируется: «Похожий тест уже существует. Просмотреть оригинал?»
- В личной библиотеке дубликат разрешён (учитель может адаптировать)

### GC-10: Freemium-ограничения на AI-генерацию
- Freemium учителям: 10 генераций в месяц бесплатно
- При достижении лимита: «Вы использовали 10/10 AI-генераций этого месяца. Обновите план или дождитесь следующего месяца»
- Платный план (школьная лицензия): неограниченные генерации

## Acceptance Criteria

### AC-1: Успешная генерация из PDF
```gherkin
Given: Учитель загрузил PDF главы учебника (физика, 8 класс, 3,000 слов)
And: Указал параметры: 10 вопросов, MCQ, средняя сложность
When: Нажимает «Сгенерировать»
Then: Через 60 секунд отображается 10 вопросов-черновиков
And: Каждый вопрос имеет 4 варианта ответа, 1 правильный отмечен
And: Каждый вопрос имеет оценку сложности (1–5 звёзд)
And: Вопросы на том же языке, что и PDF
```

### AC-2: Принятие и сохранение вопросов
```gherkin
Given: Учитель видит 10 черновиков вопросов
When: Он принимает 7 вопросов и отклоняет 3
And: Нажимает «Сохранить как тест»
Then: Тест из 7 вопросов появляется в «Мои тесты»
And: Отклонённые вопросы логируются в ai_feedback_log с решением rejected
And: Принятые вопросы имеют source_type = 'ai' (значения: 'manual' / 'ai' / 'ai_edited')
```

### AC-3: Редактирование вопроса
```gherkin
Given: Учитель видит черновик с фактической ошибкой в ответе
When: Он нажимает «Редактировать», исправляет текст правильного ответа
And: Принимает вопрос
Then: Вопрос сохраняется с source_type = 'ai_edited' (значения: 'manual' / 'ai' / 'ai_edited')
And: Исходный и отредактированный текст сохранены в ai_feedback_log (для дообучения)
```

### AC-4: Публикация в Marketplace
```gherkin
Given: Учитель создал тест из 10 AI-сгенерированных вопросов без флагов needs_review
When: Нажимает «Опубликовать в Marketplace»
Then: Тест проходит автоматическую проверку (< 5 секунд)
And: Появляется в Marketplace с атрибутами: предмет, класс, рейтинг 0, источник «AI-генерация»
And: Другие учителя могут найти и использовать тест
```

### AC-5: Флаг needs_review блокирует публикацию
```gherkin
Given: AI сгенерировал вопрос с датой «В каком году...» (флаг needs_review)
When: Учитель пытается опубликовать тест в Marketplace без проверки
Then: Система показывает «1 вопрос требует проверки перед публикацией»
And: Публикация заблокирована до подтверждения учителем
```

### AC-6: Лимит freemium
```gherkin
Given: Учитель на freemium использовал 10 AI-генераций в текущем месяце
When: Пытается запустить 11-ю генерацию
Then: Система показывает «Лимит исчерпан. 10/10 генераций использовано»
And: Кнопка «Сгенерировать» недоступна
And: Показывается предложение обновить план
```

### AC-7: Восстановление черновика
```gherkin
Given: Учитель начал редактировать черновик из 10 вопросов
When: Он закрывает страницу (случайно или намеренно)
And: Возвращается в раздел AI-генерации в течение 7 дней
Then: Система показывает «Продолжить незавершённую генерацию от [дата]?»
And: При подтверждении — открывается тот же черновик с сохранёнными решениями
```

### AC-8: Изоляция данных
```gherkin
Given: Учитель школы A загрузил учебник и сгенерировал тест
When: Учитель школы B делает запрос к API генерации школы A
Then: Система возвращает 403 Forbidden
And: Материалы школы A недоступны другим школам
```

## Технические требования к AI-модели

| Требование | Описание |
|------------|----------|
| **Языки** | Узбекский (кириллица + латиница), Русский |
| **Типы вопросов** | MCQ, Множественный выбор, Matching, Открытый ответ |
| **Точность дистракторов** | Неправильные варианты должны быть правдоподобными (не очевидными) |
| **Latency** | P95 ≤ 60 сек для 10 вопросов |
| **Доступность API** | SLA ≥ 99.5% (алерт при downtime > 5 мин) |
| **Конфиденциальность** | Загруженные материалы не используются для обучения внешних моделей без согласия |

**MVP-стратегия выбора модели:**
- Использовать облачный LLM API (OpenAI / Anthropic / Google) с prompt engineering
- Self-hosted модель — Phase 3 (при 100+ школах, для снижения затрат и data sovereignty)
- Стоимость генерации 10 вопросов: ≤ $0.05 (контролируется через rate limits)

## KPI

| Метрика | Цель | Критический порог |
|---------|------|------------------|
| Среднее время создания теста (10 вопросов через AI) | ≤ 2 мин | > 5 мин = проблема adoption |
| Среднее время без AI (ручной ввод, 10 вопросов) | ~15 мин | — (baseline) |
| Процент принятых вопросов без редактирования | ≥ 60% | < 40% = плохое качество модели |
| Процент вопросов с флагом needs_review | ≤ 15% | > 30% = модель генерирует слишком много рискованных вопросов |
| Retention: учителя, использующие AI-генерацию повторно | ≥ 70% через 30 дней | < 50% = фича не даёт ценности |
| Тесты из Marketplace (использованы хотя бы 1 раз) | ≥ 40% от опубликованных | < 20% = Marketplace не работает |
