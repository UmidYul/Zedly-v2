# Telegram Integration — Нативная интеграция с Telegram

**Файл:** `docs/03_features/telegram_integration.md`  
**Приоритет:** MVP  
**Статус:** Production Blueprint

---

## Цель фичи

Сделать Telegram основным каналом взаимодействия с платформой: учителя назначают тесты прямо из бота, ученики проходят тесты не выходя из Telegram, родители получают мгновенные уведомления о результатах ребёнка. Все результаты автоматически синхронизируются с основной платформой.

**Почему это критично:** Узбекские учителя уже работают в Telegram (групповые чаты классов — стандарт). Добавление тестирования в уже используемый канал снижает барьер входа до нуля. Ученику не нужно устанавливать приложение — Telegram уже есть на каждом телефоне.

---

## Кто использует

| Роль | Канал | Действие |
|------|-------|----------|
| Учитель | Bot (команды) / Telegram Mini App | Назначает тесты, смотрит быстрый итог класса |
| Ученик | Bot (inline-кнопки) / Telegram Mini App | Проходит тест прямо в Telegram |
| Родитель | Bot (пассивный получатель) | Получает уведомления о результатах ребёнка |
| Директор | Telegram Mini App | Просматривает сводку школы (Phase 2) |

---

## Компоненты интеграции

### 1. Telegram Bot (`@ZedlyBot`)
Основной канал. Работает через Telegram Bot API. Команды и inline-кнопки.

### 2. Telegram Login Widget
Авторизация на веб-платформе через Telegram-аккаунт (OAuth-подобный flow). Не требует пароля.

### 3. Telegram Mini App (TMA)
Веб-приложение внутри Telegram (WebApp API). Полный интерфейс платформы без выхода из мессенджера. MVP: базовое прохождение теста. Phase 2: полный дашборд.

### 4. Webhook → Platform API
Все действия через бота записываются в основную платформу через внутренний API. Единый источник правды — PostgreSQL, не Telegram.

## User Flow

### Flow A: Учитель назначает тест через бота

```
1. Учитель пишет боту /assign или нажимает кнопку «Назначить тест»
2. Бот отвечает:
   «Выберите тест:»
   [Физика 8Б — Закон Ньютона] [Алгебра 9А — Квадратные уравнения] [+ Создать новый]
3. Учитель выбирает тест
4. Бот спрашивает: «Выберите класс:»
   [8А] [8Б] [9А] [Все мои классы]
5. Учитель выбирает класс
6. Бот спрашивает: «Дедлайн?»
   [Сегодня 23:59] [Завтра 23:59] [Через 3 дня] [Указать вручную]
7. После подтверждения:
   — Тест назначен в платформе (API-вызов)
   — Бот рассылает ссылку всем ученикам класса (у кого есть Telegram)
   — Учитель получает: «✅ Тест назначен. Ссылка отправлена 24 ученикам»
```

### Flow B: Ученик проходит тест через бота (inline-кнопки)

```
1. Ученик получает сообщение от бота:
   ┌────────────────────────────────────────────────┐
   │ 📝 Новый тест от Алиевой Д.Н.                 │
   │ «Физика: Закон Ньютона»                        │
   │ 10 вопросов · Дедлайн: сегодня 23:59          │
   │ [Начать тест]  [Открыть в браузере]            │
   └────────────────────────────────────────────────┘
2. Нажимает «Начать тест»
3. Бот открывает Telegram Mini App (TMA) или переходит в inline-режим
4. Режим inline (если TMA недоступен или ученик предпочитает):
   — Бот присылает вопросы по одному сообщению
   — MCQ: inline-кнопки A / B / C / D
   — После выбора: бот подтверждает «✅ Ответ принят» и присылает следующий вопрос
   — Открытый ответ: текстовое сообщение ученика
   — Matching: бот присылает таблицу + просит ответить в формате «1-B, 2-A, 3-C»
5. После последнего вопроса:
   «Вы ответили на все 10 вопросов. Завершить тест?»
   [Да, завершить] [Вернуться и проверить]
6. После завершения:
   — Результат сохранён в платформе
   — Бот присылает итог: «Ваш результат: 8/10 (80%) ✅»
   — Карточка-результат (изображение) для шейринга
   — Кнопка «Посмотреть разбор ошибок» → открывает веб-версию
```

### Flow C: Ученик проходит тест через Telegram Mini App

```
1. Ученик нажимает «Начать тест» → открывается TMA (WebApp.open)
2. Внутри TMA — полный интерфейс теста (как веб, но внутри Telegram):
   — Все типы вопросов работают нативно
   — Таймер, прогресс-бар, навигация
   — Изображения к вопросам отображаются корректно
3. После завершения: TMA передаёт результат через WebApp.sendData
4. Платформа сохраняет результат, бот присылает карточку в чат
```

### Flow D: Telegram Login Widget (авторизация на веб-платформе)

```
1. Пользователь открывает zedly.uz → нажимает «Войти через Telegram»
2. Страница показывает Telegram Login Widget (официальный виджет Telegram)
3. Пользователь нажимает «Войти» → Telegram открывается на телефоне (deep link)
4. В Telegram: «ZedlyBot хочет получить доступ к вашему аккаунту» → [Разрешить]
5. Telegram возвращает auth_data (hash, id, first_name, username, photo_url)
6. Zedly backend верифицирует HMAC-подпись auth_data (ключ = Bot Token)
7. Если пользователь новый: создаётся аккаунт, запрашивается роль (ученик / учитель)
8. Если существующий: выдаётся JWT-токен (15 мин) + Refresh Token (30 дней)
9. Пользователь авторизован — попадает на дашборд
```

**Верификация Telegram auth_data (обязательно):**
```python
import hashlib, hmac, time

def verify_telegram_auth(auth_data: dict, bot_token: str) -> bool:
    check_hash = auth_data.pop('hash')
    data_string = '\n'.join(f'{k}={v}' for k, v in sorted(auth_data.items()))
    secret_key = hashlib.sha256(bot_token.encode()).digest()
    computed_hash = hmac.new(secret_key, data_string.encode(), hashlib.sha256).hexdigest()
    # Проверка подписи + проверка свежести: auth_date не старше 86400 сек
    return hmac.compare_digest(computed_hash, check_hash) and \
           (time.time() - int(auth_data['auth_date'])) < 86400
```

### Flow E: Уведомления родителям

```
Родитель получает Telegram-сообщение от бота при:

Событие 1 — Завершение теста учеником:
  «📊 Ваш ребёнок Алишер прошёл тест «Физика: Закон Ньютона»
   Результат: 8/10 (80%) ✅
   [Посмотреть детали]»

Событие 2 — НТТ-симулятор завершён:
  «🎯 Алишер прошёл НТТ-симулятор
   Результат: 43/60 ↑ +5 с прошлой попытки
   NTT Readiness: 🟡 Почти готов (72/100)»

Событие 3 — Тест просрочен (ребёнок не прошёл до дедлайна):
  «⚠️ Алишер не прошёл тест «Алгебра» (дедлайн истёк)
   Учитель: Каримова Н.Б.»

Событие 4 — Результат в топ (геймификация):
  «🏆 Алишер в топ-10% класса по последнему тесту!
   Поздравьте его 👏»

Правила:
  - Уведомление отправляется только если родитель привязал Telegram (opt-in)
  - Родитель может настроить: какие события получать (все / только плохие / только хорошие)
  - Нельзя отключить уведомление о просроченном тесте (важно для вовлечённости)
  - Максимум 3 уведомления в день на одного ребёнка (анти-спам)
```

## Зависимости от данных (таблицы)

| Таблица | Ключевые поля |
|---------|---------------|
| `telegram_accounts` | `id`, `user_id`, `school_id`, `telegram_id`, `username`, `first_name`, `linked_at`, `is_active` |
| `telegram_sessions` | `id`, `telegram_id`, `test_session_id`, `channel` (bot_inline / tma), `started_at`, `completed_at` |
| `notification_settings` | `user_id`, `role`, `events_json` (какие уведомления включены), `max_per_day` |
| `notification_log` | `id`, `recipient_telegram_id`, `event_type`, `sent_at`, `status` (delivered / failed) |
| `parent_child_links` | `parent_user_id`, `child_user_id`, `school_id`, `linked_at` |

> Все записи содержат `school_id` для RLS-изоляции. `telegram_id` — уникальный идентификатор Telegram-пользователя (не изменяется при смене username).

## Граничные случаи

### GC-1: У ученика нет Telegram-аккаунта
Ссылка на тест через Telegram недоступна. Учитель видит в списке класса: «Ученик [имя] — Telegram не привязан». Ученик получает ссылку на веб-версию через другой канал (email / прямой URL). Результат теста сохраняется в платформу в любом случае.

### GC-2: Ученик начал тест в боте, потерял соединение на середине
Ответы, принятые ботом, сохранены через API в `test_sessions`. При повторном открытии теста через бота: «У вас есть незавершённый тест. Продолжить?» → восстанавливается с последнего сохранённого вопроса.

### GC-3: Telegram Bot API недоступен (downtime Telegram)
Критические действия (прохождение теста, сохранение ответов) не зависят от доступности бота — они проходят через основное веб-API. Бот используется только как канал доставки и уведомлений. При недоступности бота: уведомления ставятся в очередь (Redis Queue), отправляются при восстановлении. Таймаут очереди: 24 часа.

### GC-4: Родитель пытается привязать Telegram другого ребёнка
`parent_child_links` создаётся только через верифицированный процесс: учитель или директор подтверждает связь родитель–ученик в платформе. Самопривязка через бота — только по инвайт-коду, выданному учителем.

### GC-5: Telegram-аккаунт учителя использован несколькими пользователями
`telegram_id` уникально привязывается к одному `user_id` в таблице `telegram_accounts`. Попытка привязать тот же `telegram_id` к другому аккаунту: «Этот Telegram-аккаунт уже привязан к другой учётной записи Zedly».

### GC-6: Авторизация через Telegram — auth_date устарел (> 24 часа)
Backend отклоняет запрос: 401 Unauthorized. Пользователь видит: «Ссылка авторизации устарела. Попробуйте снова». Новый виджет сгенерирует свежий auth_date.

### GC-7: Matching-вопрос в inline-режиме бота — неправильный формат ответа
Ученик ввёл «1B, 2A» вместо «1-B, 2-A». Бот распознаёт оба формата через regex: `(\d+)[-\s]?([A-Za-z])`. Если не распознан совсем: «Пожалуйста, ответьте в формате: 1-B, 2-A, 3-C». Максимум 3 попытки ввода, после чего вопрос засчитывается как пропущенный.

### GC-8: Уведомление для родителя в период «ночного тихого часа» (23:00–07:00)
Уведомления о результатах тестов откладываются до 07:00. Уведомления о безопасности (если бы были) — немедленно. Настройка тихого часа: по умолчанию включена, можно отключить в профиле родителя.

### GC-9: Ученик заблокировал бота в Telegram
`notification_log.status = failed` с кодом `bot_blocked`. Статус `telegram_accounts.is_active = false`. Последующие уведомления не отправляются. Учитель видит в списке класса: «Telegram-уведомления недоступны».

### GC-10: Сессия Telegram Mini App истекла (TMA закрылась)
TMA может быть закрыта пользователем или Telegram. При повторном открытии: восстановление сессии аналогично веб-версии. Если тест был с таймером и время истекло: стандартный flow принудительного завершения.

## Acceptance Criteria

### AC-1: Назначение теста через бота
```gherkin
Given: Учитель авторизован в боте, у него есть тест «Физика 8Б»
When: Отправляет команду /assign и проходит диалог выбора теста и класса
Then: Тест назначается в платформе (появляется в «Активных тестах»)
And: Все ученики класса с привязанным Telegram получают уведомление через бота
And: Учитель получает подтверждение с числом уведомленных учеников
```

### AC-2: Прохождение MCQ-теста в inline-режиме
```gherkin
Given: Ученик получил сообщение от бота с тестом на 5 MCQ-вопросов
When: Нажимает «Начать тест» и выбирает ответы через inline-кнопки
Then: Каждый ответ сохраняется в платформе (≤ 1 сек)
And: После последнего вопроса бот показывает итог
And: Результат виден учителю в дашборде в течение 15 сек
```

### AC-3: Telegram Login Widget
```gherkin
Given: Новый пользователь открывает zedly.uz и нажимает «Войти через Telegram»
When: Авторизуется в Telegram и возвращается на сайт
Then: Backend верифицирует HMAC-подпись auth_data
And: Создаётся новый аккаунт (если первая авторизация)
And: Пользователю выдаётся JWT + Refresh Token
And: auth_date проверяется: не старше 86400 сек
```

### AC-4: Уведомление родителю о результате
```gherkin
Given: Родитель привязал Telegram и включил уведомления о результатах
When: Его ребёнок завершает тест с результатом 8/10
Then: Родитель получает Telegram-сообщение в течение 30 секунд
And: Сообщение содержит: имя ребёнка, название теста, балл, сравнение с прошлым
And: Если событие произошло между 23:00 и 07:00 — уведомление отложено до 07:00
```

### AC-5: Бот заблокирован — учитель видит статус
```gherkin
Given: Ученик заблокировал @ZedlyBot
When: Учитель открывает список класса в дашборде
Then: У ученика статус «Telegram-уведомления недоступны»
And: Уведомление этому ученику через бота не отправляется
And: Ученик по-прежнему может проходить тесты через веб-версию
```

### AC-6: Восстановление теста в боте после разрыва
```gherkin
Given: Ученик ответил на 3 из 5 вопросов в боте и потерял соединение
When: Открывает бота снова
Then: Бот спрашивает «Продолжить незавершённый тест?»
And: При подтверждении — тест возобновляется с вопроса 4
And: Первые 3 ответа сохранены
```

### AC-7: Очередь уведомлений при Telegram downtime
```gherkin
Given: Telegram Bot API недоступен 30 минут
And: За это время 50 учеников завершили тесты
When: Telegram Bot API восстанавливается
Then: Все 50 уведомлений родителям отправляются в течение 5 минут
And: Уведомления с временем отправки > 24 часов назад отбрасываются
```

### AC-8: Anti-spam для родителей
```gherkin
Given: Ребёнок прошёл 5 тестов за один день
When: Все 5 результатов обработаны платформой
Then: Родитель получает максимум 3 уведомления за день
And: Четвёртое и пятое уведомления объединяются в дайджест или откладываются
```

## KPI

| Метрика | Цель | Критический порог |
|---------|------|------------------|
| % учеников с привязанным Telegram | ≥ 85% | < 60% = barьер для Telegram-first стратегии |
| Completion rate тестов через бот | ≥ 75% | < 50% = UX inline-режима неудобен |
| Доставляемость уведомлений родителям | ≥ 90% | < 80% = много заблокированных ботов |
| Время между завершением теста и уведомлением родителю | ≤ 30 сек | > 2 мин |
| Учителя, назначающие тесты через бота (vs только веб) | ≥ 40% | < 15% = бот не используется |
| Авторизаций через Telegram Login Widget | ≥ 70% от всех новых регистраций | < 40% = нужен push на Telegram-вход |
