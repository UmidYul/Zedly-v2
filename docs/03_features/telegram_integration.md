## 1. Назначение

Telegram в Zedly используется для:
- быстрого входа после привязки аккаунта;
- доставки уведомлений (назначение теста, напоминания, статусы).

Telegram не используется как самостоятельная регистрация.

---

## 2. Принципы UX

- Пользователь не вводит технические идентификаторы Telegram вручную.
- Подключение выполняется кнопкой `Подключить Telegram` в разделе `Способы входа`.
- После подключения пользователь может входить через `Войти через Telegram`.
- Если Telegram не установлен, показывается подсказка:  
  `Установите приложение Telegram, затем вернитесь сюда`.

---

## 3. Пользовательские сценарии

### Flow A: Подключение Telegram в профиле

1. Пользователь авторизован в Zedly.
2. Открывает `Профиль -> Способы входа`.
3. Нажимает `Подключить Telegram`.
4. Открывается Telegram с подготовленным действием для бота.
5. Пользователь подтверждает действие (`Start` / `Отправить`).
6. В Zedly Telegram отмечается как `подключено`.

### Flow B: Вход через Telegram

1. Пользователь нажимает `Войти через Telegram` на странице входа.
2. Система проверяет, что Telegram уже привязан к аккаунту.
3. Если привязан: выдаются access/refresh токены и выполняется вход.
4. Если не привязан: система возвращает статус `telegram_not_connected` и просит войти по login+паролю.

---

## 4. Backend/API точки

- `POST /api/v1/auth/telegram`
- `GET /api/v1/users/me/login-methods`
- `POST /api/v1/users/me/login-methods/telegram/connect`

Источник контракта:
- `docs/07_api/openapi.yaml`

---

## 5. Acceptance

Минимальные критерии:
- После подключения Telegram статус в профиле меняется на `подключено`.
- После подключения Telegram вход кнопкой `Войти через Telegram` проходит без ввода login/пароля.
- Если Telegram не подключён, пользователь получает понятное сообщение и fallback на login+пароль.

---

## 6. Ограничения

- Telegram-подключение не заменяет обязательную смену OTP при первом входе.
- Без первичной активации аккаунта (login + OTP + смена пароля) Telegram вход недоступен.
