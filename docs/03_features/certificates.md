# Сертификаты — Верифицируемые цифровые сертификаты с QR-кодом

**Файл:** `docs/03_features/certificates.md`  
**Приоритет:** Phase 2 (MVP: базовая генерация; полная верификация API — Phase 2)  
**Статус:** Production Blueprint

---

## Цель фичи

Выдавать ученикам верифицируемые цифровые сертификаты о прохождении тестов и курсов. Каждый сертификат содержит QR-код, по которому работодатель или вуз может мгновенно проверить подлинность — без звонков в школу и без бумаги. Личное портфолио ученика накапливает сертификаты с 1 по 11 класс и становится его цифровым академическим досье.

**Почему это lock-in:** Сертификаты Zedly начинают принимать вузы и работодатели → ученик и родитель никогда не уйдут с платформы, потому что портфолио нельзя перенести → директор не может отказаться от Zedly, не лишив учеников их истории.

---

## Кто использует

| Роль | Действие |
|------|----------|
| Ученик | Получает сертификат, хранит в портфолио, делится с вузами/работодателями |
| Учитель | Настраивает условия выдачи сертификата для теста/курса |
| Директор | Видит статистику выданных сертификатов по школе |
| Вуз / Работодатель | Сканирует QR-код или вводит ID — верифицирует подлинность |
| Zedly API-клиент | Программная верификация через REST API |

## Типы сертификатов

| Тип | Условие выдачи | Пример |
|-----|---------------|--------|
| **Тест пройден** | Результат ≥ порогового балла (учитель задаёт: обычно ≥ 70%) | «Сдал тест по физике на 85%» |
| **НТТ-симулятор** | Завершена сессия симулятора (без порога) | «Прошёл НТТ-симулятор, балл: 43/60» |
| **Курс завершён** | Пройдены все тесты курса с результатом ≥ порога | «Завершил курс "Алгебра 9 класс"» |
| **Учитель года** | Присваивается учителю директором (Phase 2) | «Лучший учитель физики 2025» |
| **Межшкольный чемпион** | Победа в межшкольном челлендже | «Победитель межшкольного турнира по химии» |

## User Flow

### Flow A: Автоматическая выдача сертификата после теста

```
1. Учитель создаёт тест и включает опцию «Выдать сертификат»:
   - Пороговый балл для получения: [70%] (ввод вручную)
   - Название сертификата: [Курс "Законы Ньютона". Физика 8 класс]
   - Подписант: [Директор / Учитель / Автоматически]
2. Ученик проходит тест и набирает ≥ порогового балла
3. Background job запускает генерацию сертификата:
   - Заполняет шаблон данными (имя, школа, балл, дата, ID)
   - Генерирует уникальный certificate_id (UUID)
   - Генерирует QR-код (ссылка: https://verify.zedly.uz/{certificate_id})
   - Рендерит PDF (A4, высокое разрешение)
   - Сохраняет в Cloudflare R2 / локальное хранилище
4. Ученик получает уведомление:
   - В платформе: «🎓 Вы получили сертификат!»
   - В Telegram: «Поздравляем! Ваш сертификат по физике готов. [Скачать]»
5. Сертификат добавляется в портфолио ученика
```

### Flow B: Ученик делится сертификатом

```
1. Ученик открывает портфолио → выбирает сертификат
2. Доступные действия:
   [Скачать PDF]  [Копировать ссылку]  [Поделиться в Telegram]  [LinkedIn (Phase 2)]
3. При «Поделиться в Telegram»:
   — Бот отправляет красивую карточку-превью + ссылку для верификации
4. Получатель (вуз, работодатель) открывает ссылку и видит страницу верификации:
   ┌─────────────────────────────────────────────────┐
   │ ✅ Сертификат подлинный                         │
   │ Ученик: Алишер Каримов                          │
   │ Школа: Лицей №1, Ташкент                        │
   │ Тест: «Законы Ньютона. Физика 8 класс»          │
   │ Результат: 85% (51/60 баллов)                   │
   │ Дата выдачи: 15 февраля 2026                    │
   │ Выдан платформой Zedly · ID: ZDL-2026-A8F3C     │
   └─────────────────────────────────────────────────┘
```

### Flow C: Портфолио ученика (1–11 класс)

```
Портфолио — хронологическая лента сертификатов и достижений:

Структура записи в портфолио:
  [Год] [Класс] [Предмет] [Тип] [Балл] [Сертификат PDF]

Пример:
  2026 · 11 класс · НТТ-симулятор «Математика + Физика»     · 52/60 · [PDF]
  2025 · 10 класс · «Алгебра: полный курс»                  · 94%   · [PDF]
  2025 · 10 класс · Межшкольный чемпион по химии             · 🏆    · [PDF]
  2024 · 9 класс  · «История Узбекистана: итоговый тест»     · 78%   · [PDF]

Правила портфолио:
  - Нельзя удалить запись из портфолио (защита от фальсификации)
  - Можно скрыть запись из публичного просмотра (остаётся в приватном портфолио)
  - При смене школы: сертификаты остаются привязаны к user_id, не к school_id
  - Экспорт всего портфолио: PDF-документ со всеми сертификатами
```

## Верификация сертификатов (API)

### Публичная страница верификации

URL: `https://verify.zedly.uz/{certificate_id}`

Доступна без авторизации. Возвращает:
- Статус: ✅ Подлинный / ❌ Недействительный / ⚠️ Отозван
- Данные ученика (имя, школа — без персональных данных типа даты рождения)
- Данные теста (название, балл, дата)
- Статус: `valid` / `revoked` / `not_found`

Если `revoked`: «Этот сертификат был аннулирован. Свяжитесь с учебным заведением».

### REST API для вузов и работодателей (Phase 2)

```
GET /api/v1/certificates/verify/{certificate_id}
Authorization: Bearer {api_key}   // API-ключ вуза/работодателя

Response 200:
{
  "status": "valid",
  "certificate_id": "ZDL-2026-A8F3C",
  "student_name": "Алишер Каримов",
  "school_name": "Лицей №1, Ташкент",
  "test_name": "Законы Ньютона. Физика 8 класс",
  "score_percent": 85,
  "issued_at": "2026-02-15T14:32:00Z",
  "issuer": "Zedly Educational Platform"
}

Response 404: { "status": "not_found" }
Response 200: { "status": "revoked", "revoked_at": "...", "reason": "..." }
```

**Монетизация API верификации:**
- До 100 верификаций в месяц: бесплатно (для вузов при регистрации)
- 100–1,000: $20/мес
- 1,000+: enterprise-тариф, согласуется отдельно

## Дизайн сертификата

Сертификат рендерится в PDF (A4, 300 DPI) с элементами:

| Элемент | Описание |
|---------|----------|
| Логотип Zedly + логотип школы | Верхняя часть |
| Заголовок | «СЕРТИФИКАТ» / «GUVOHNOMA» (узбекский) |
| Имя ученика | Крупным шрифтом (по центру) |
| Текст | «Настоящим удостоверяется, что [имя] успешно прошёл(а) [название теста]» |
| Балл | «Результат: 85% (51 из 60 вопросов)» |
| Дата | «15 февраля 2026 года» |
| Подпись | Директора / учителя (факсимиле из профиля) |
| QR-код | Правый нижний угол; ссылка на verify.zedly.uz |
| Уникальный ID | «ZDL-2026-A8F3C» под QR-кодом |
| Защитный паттерн | Фоновый watermark «ZEDLY» |

Языки: узбекский (кириллица), русский — оба на одном сертификате (билингвальный).

## Зависимости от данных (таблицы)

| Таблица | Ключевые поля |
|---------|---------------|
| `certificates` | `id` (UUID = certificate_id), `student_id`, `school_id`, `test_id`, `session_id`, `type`, `score_percent`, `issued_at`, `status` (valid/revoked), `pdf_url`, `revoked_at`, `revoked_reason` |
| `certificate_verifications` | `id`, `certificate_id`, `verifier_ip`, `verifier_api_key` (null = публичный QR), `verified_at` |
| `portfolio_entries` | `id`, `student_id`, `certificate_id`, `school_id`, `class_year`, `is_public`, `created_at` |
| `certificate_templates` | `id`, `school_id` (null = дефолтный), `template_html`, `logo_url`, `signature_url` |
| `api_clients` | `id`, `name`, `type` (university/employer), `api_key_hash`, `monthly_limit`, `usage_count` |

> `certificates.id` — это и есть публичный `certificate_id`. UUID v4, не предсказуем. Никаких инкрементных ID для сертификатов.

> `school_id` в таблице `certificates` нужен для RLS: директор видит сертификаты своей школы. Но сами сертификаты привязаны к `student_id` и не теряются при смене школы.

## Граничные случаи

### GC-1: Ученик набрал балл ниже порогового
Сертификат не выдаётся. В портфолио появляется запись «Тест пройден: 65% (ниже порога 70%)» без сертификата. Ученик видит: «Для получения сертификата необходимо набрать ≥ 70%. Попробуйте ещё раз». Если тест допускает несколько попыток — новая попытка с результатом ≥ порога выдаёт сертификат.

### GC-2: Директор или учитель хочет отозвать сертификат
Причины: ошибка в данных, нарушение правил. Действие: `certificates.status = revoked`, `revoked_at`, `revoked_reason` заполняются. Страница верификации показывает: «Сертификат аннулирован». PDF из хранилища не удаляется (аудиторская дорожка), но ссылка возвращает статус `revoked`. Ученик получает уведомление с причиной.

### GC-3: Ученик переходит в другую школу
`certificates` привязаны к `student_id`, не к `school_id`. При смене школы все сертификаты остаются в портфолио ученика. В записи сертификата `school_name` отображает исходную школу (на момент выдачи).

### GC-4: Генерация PDF зависла или упала
Background job использует retry: 3 попытки с интервалом 5 мин. При окончательном провале: `certificates.status = generation_failed`, алерт команде. Ученику: «Ваш сертификат готовится. Придёт уведомление». Повторная генерация запускается вручную или по крону.

### GC-5: Верификация несуществующего certificate_id
Страница: «Сертификат с таким ID не найден. Проверьте QR-код». Логируется: `certificate_verifications` с `status = not_found` и IP-адресом — для мониторинга попыток подбора ID.

### GC-6: Шаблон сертификата отсутствует для школы
Используется дефолтный шаблон Zedly (school_id IS NULL). Школа может загрузить свой логотип и факсимиле директора — тогда используется кастомный шаблон.

### GC-7: Массовая выдача сертификатов (конец учебного года)
При завершении курса всем учеником класса одновременно: генерация сертификатов ставится в очередь (Bull/Celery). Каждый сертификат — отдельная задача. Лимит: 100 одновременных задач генерации. Ожидаемое время для 30 учеников: ≤ 5 минут.

### GC-8: Ученик скачал PDF и изменил его вручную
PDF не является юридически значимым документом без верификации через QR-код. Работодатель/вуз обязан сканировать QR — изменённый PDF при сканировании покажет оригинальные данные из БД. Подделка обнаруживается мгновенно.

### GC-9: API-клиент превысил лимит верификаций
`api_clients.usage_count >= monthly_limit`: ответ `429 Too Many Requests`. Тело: `{"error": "monthly_limit_exceeded", "limit": 100, "reset_at": "2026-03-01T00:00:00Z"}`.

### GC-10: Портфолио — ученик хочет удалить «плохой» сертификат
Удаление записей из портфолио запрещено (защита целостности). Ученик может скрыть запись из публичного профиля (`portfolio_entries.is_public = false`). Запись остаётся в приватном портфолио и в БД навсегда. Это намеренное ограничение — портфолио должно быть честным.

## Acceptance Criteria

### AC-1: Автоматическая выдача сертификата
```gherkin
Given: Учитель настроил порог сертификата 70%; ученик набрал 85%
When: Тест завершён и результат сохранён
Then: В течение 60 секунд создаётся запись в certificates со статусом valid
And: PDF-сертификат сгенерирован и доступен по ссылке
And: Ученик получает уведомление в Telegram
And: Сертификат добавлен в портфолио
```

### AC-2: Верификация через QR-код
```gherkin
Given: Сертификат с certificate_id = "ZDL-2026-A8F3C" имеет статус valid
When: Работодатель сканирует QR-код и открывает страницу верификации
Then: Страница отображает: статус ✅ Подлинный, имя ученика, школу, тест, балл, дату
And: Факт верификации логируется в certificate_verifications
And: Страница загружается без авторизации
```

### AC-3: Верификация отозванного сертификата
```gherkin
Given: Директор отозвал сертификат с причиной «Ошибка в данных»
When: Кто-либо открывает страницу верификации этого сертификата
Then: Страница показывает «⚠️ Сертификат аннулирован»
And: Причина отзыва не раскрывается публично (только статус)
And: Ученик ранее получил уведомление о причине отзыва
```

### AC-4: Портфолио — несколько лет
```gherkin
Given: Ученик учится с 1 по 11 класс и получил 15 сертификатов за это время
When: Открывает своё портфолио
Then: Все 15 сертификатов отображаются в хронологическом порядке
And: При смене школы (переезд) все сертификаты остались в портфолио
And: Каждый сертификат содержит название исходной школы
```

### AC-5: Сертификат не выдаётся ниже порога
```gherkin
Given: Учитель установил порог 70%; ученик набрал 65%
When: Тест завершён
Then: Запись в certificates НЕ создаётся
And: В портфолио появляется запись «Тест пройден: 65%» без иконки сертификата
And: Ученик видит «Для сертификата необходимо ≥ 70%»
```

### AC-6: REST API верификации
```gherkin
Given: Вуз имеет валидный api_key с лимитом 100 запросов
When: Делает GET /api/v1/certificates/verify/{id} с Bearer-токеном
Then: Получает JSON с полем status = valid и данными сертификата
And: certificate_verifications логирует api_key и время
And: usage_count инкрементируется
```

### AC-7: Массовая выдача не блокирует платформу
```gherkin
Given: 30 учеников класса одновременно завершили тест выше порога
When: Задачи генерации сертификатов поставлены в очередь
Then: Все 30 сертификатов сгенерированы в течение 5 минут
And: Платформа не испытывает деградации производительности
And: Учителю доступны результаты немедленно (не ждёт генерации PDF)
```

### AC-8: Изоляция данных директора
```gherkin
Given: Директор школы A смотрит статистику выданных сертификатов
When: Делает запрос к API
Then: Видит только сертификаты своей школы
And: Сертификаты школы B недоступны (RLS по school_id)
```

## KPI

| Метрика | Цель | Критический порог |
|---------|------|------------------|
| Время генерации одного PDF-сертификата | ≤ 10 сек | > 60 сек = проблема |
| Учителя, включившие выдачу сертификатов | ≥ 50% активных учителей | < 20% = фича не используется |
| Верификаций сертификатов в месяц | Растёт на 20%/мес | Падение = сертификаты перестали принимать |
| Доля учеников с ≥ 1 сертификатом в портфолио | ≥ 60% активных учеников | < 30% = пороги слишком высокие |
| API-клиенты (вузы/работодатели) | ≥ 5 к концу Phase 2 | < 2 = нет внешнего признания |
| Retention: учителя используют сертификаты повторно | ≥ 70% через 60 дней | < 40% = нет ценности для учителя |
