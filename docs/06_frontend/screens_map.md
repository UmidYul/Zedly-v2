# Карта экранов Zedly

> Документ покрывает все роли: Ученик, Учитель, Директор, Родитель, РОНО-инспектор.
> Для каждого экрана: название, цель, ключевые элементы UI, переходы.

---

## Условные обозначения

- 🔒 — требует авторизации
- 📲 — доступен в Telegram Mini App
- 📴 — поддерживает offline-режим (PWA)
- 🤖 — использует AI-функции

---

## 1. Общие экраны (все роли)

---

### 1.1 Login

**Цель:** Аутентификация пользователя в системе. Единая точка входа для всех ролей.

**Ключевые элементы UI:**
- Логотип Zedly + слоган
- Поле «Логин» (сгенерированный системой идентификатор)
- Поле «Пароль» + иконка показа/скрытия
- Кнопка «Войти»
- Ссылка «Забыли пароль?»
- Кнопка «Войти через Telegram»
- Переключатель языка: O'zbek / Русский

**Логика:**
- После успешного входа система определяет роль по JWT-payload (`role`) и редиректит на соответствующий Home/Dashboard
- Ошибки: «Неверный логин или пароль» (общий текст, без указания что именно неверно — security)
- После 5 неудачных попыток — капча (reCAPTCHA v3) + блокировка на 15 минут

**Переходы:**
- → `First Password Setup` (только после первого входа по OTP)
- → `Forgot Password`
- → `Student / Home` (role = student)
- → `Teacher / Dashboard` (role = teacher)
- → `Director / School Dashboard` (role = director)
- → `Parent / Child Overview` (role = parent)
- → `ROНО / District Dashboard` (role = inspector)

---

### 1.2 First Password Setup

**Цель:** Принудительно заменить одноразовый OTP-пароль на постоянный при первом входе.

**Ключевые элементы UI:**
- Поле `Новый пароль`
- Поле `Повторите пароль`
- Подсказка: минимум 8 символов и минимум 1 цифра
- Кнопка `Сохранить и продолжить`

**Логика:**
- Пока OTP не заменён, доступ к платформе не открывается.
- После успешной смены показывается экран подключения быстрых способов входа.

**Переходы:**
- → `Quick Login Setup` (Google/Telegram/Пропустить)
- → `Dashboard` (если пользователь выбирает «Пропустить — сделаю позже»)

---

### 1.3 Forgot Password

**Цель:** Запрос восстановления пароля по логину пользователя.

**Ключевые элементы UI:**
- Поле логина
- Кнопка «Отправить ссылку»
- После отправки: баннер «Если аккаунт существует, инструкции отправлены»

**Переходы:**
- → `Login`
- → `Reset Password` (по ссылке из письма / кода из Telegram)

## 2. Ученик (role = student)

---

### 2.1 Home 🔒 📲 📴

**Цель:** Главная страница ученика. Список назначенных тестов, быстрый старт, статус выполнения.

**Ключевые элементы UI:**
- Приветствие: «Привет, [Имя]! Сегодня [дата]»
- Раздел **«Активные тесты»** — карточки теста:
  - Название теста
  - Предмет + иконка
  - Дедлайн (countdown: «Осталось 2ч 15м» красным если < 3 часов)
  - Статус: `Не начат` / `В процессе` / `Завершён` / `Просрочен`
  - Кнопка «Начать» / «Продолжить» / «Посмотреть результат»
- Раздел **«Скоро»** — тесты с дедлайном > 24 часов (сжатый список)
- Раздел **«Завершённые»** — последние 3 теста со ссылкой «Показать все» → Portfolio
- Баннер **«НТТ-симулятор»** — постоянный виджет в правом углу / bottom sheet
- Прогресс-бар «Выполнено сегодня: 2 из 3 тестов»
- Bottom Navigation: `Главная` | `Тесты` | `Портфолио` | `Профиль`

**Логика:**
- Тесты загружаются при первом открытии и кэшируются в IndexedDB (offline PWA)
- Порядок: по дедлайну по возрастанию, просроченные — внизу с серым фоном
- Уведомление (push/Telegram): за 2 часа и 30 минут до дедлайна

**Переходы:**
- → `Test Screen` (по кнопке «Начать» / «Продолжить»)
- → `Result Screen` (по кнопке «Посмотреть результат»)
- → `NTT Simulator` (по баннеру)
- → `Portfolio` (по «Показать все» или bottom nav)

---

### 2.2 Test Screen 🔒 📲 📴

**Цель:** Прохождение теста. Основной экран взаимодействия ученика с платформой.

**Ключевые элементы UI:**
- **Header:** Название теста | Таймер обратного отсчёта (MM:SS, красный при < 5 мин) | Прогресс «Вопрос 7 из 30»
- **Тело вопроса:**
  - Текст вопроса (поддержка LaTeX для математики, формул физики/химии)
  - Изображение (если есть, fullscreen по тапу)
  - Варианты ответа (A / B / C / D) — radio-кнопки, крупные, touch-friendly (min 44px)
- **Навигация по вопросам:** горизонтальный скролл нумерации — зелёный (отвечено), белый (пропущено), текущий (синяя рамка)
- **Кнопки:** `← Назад` | `Пропустить` | `Далее →`
- **Footer:** «Ответить позже» (флаг на вопрос) | Кнопка «Завершить тест» (доступна после ответа на 80%+ вопросов или по истечении времени)
- **Диалог «Завершить?»:** «Вы ответили на X из Y вопросов. Завершить?» + «Отмена» / «Да, завершить»

**Логика:**
- Каждый ответ мгновенно сохраняется в localStorage (offline) и синхронизируется с сервером (online)
- При потере сети: баннер «Offline — ответы сохраняются локально»
- По истечении таймера: тест автоматически завершается, финальный ответ отправляется
- Смена вкладки / минимизация: фиксируется событие `tab_switch` в лог (для НТТ — запрет)
- Для НТТ-симулятора: полноэкранный режим, кнопка «Назад» заблокирована

**Переходы:**
- → `Result Screen` (после завершения теста)
- → `Home` (только через системную кнопку «Назад», с подтверждением — «Выйти? Прогресс сохранится»)

---

### 2.3 Result Screen 🔒 📲

**Цель:** Карточка результата теста. Вирусная механика (как Spotify Wrapped) + разбор ошибок.

**Ключевые элементы UI:**
- **Result Card (шейрабельная карточка):**
  - Имя ученика + название теста
  - Большой процент результата (например, **83%**) с цветовой индикацией (красный / жёлтый / зелёный)
  - Правильных / Всего: «25 / 30»
  - Время прохождения: «Затрачено: 18 мин 42 сек»
  - Место в классе: «🏆 3 место из 24» (если тест уже завершили > 50% класса)
  - Кнопки: «📤 Поделиться» (генерируется PNG-карточка) | «📱 Telegram»
- **Разбор ошибок:**
  - Аккордеон по темам: «Алгебра (2 ошибки)» / «Геометрия (1 ошибка)»
  - Для каждой ошибки: вопрос → Ваш ответ (красный) → Правильный ответ (зелёный) → Объяснение
- **Рекомендации AI:** 🤖 «Повторите тему: Квадратные уравнения — 3 ошибки из 5 вопросов по этой теме»
- **Кнопка «Пройти снова»** (если учитель разрешил повторное прохождение)
- **Кнопка «Скачать сертификат»** — если результат ≥ порогового значения (устанавливает учитель, по умолчанию 70%)

**Логика:**
- Карточка генерируется на сервере как PNG (html2canvas или Puppeteer) и кэшируется в R2
- Место в классе обновляется в реальном времени через WebSocket (или polling каждые 30 сек)
- Сертификат: PDF с QR-кодом, верифицируемый через `zedly.uz/verify/{uuid}`

**Переходы:**
- → `Portfolio` (кнопка «Мои результаты»)
- → `Test Screen` (кнопка «Пройти снова», если доступно)
- → `Home` (кнопка «На главную»)
- → External: Telegram, Instagram (шеринг карточки)

---

### 2.4 NTT Simulator 🔒 📴

**Цель:** Симулятор НТТ (национальный тест для поступления в университет). Максимально приближен к реальному экзамену по формату, времени и стрессу.

**Ключевые элементы UI:**
- **Экран запуска:**
  - Инструкция: «60 вопросов, 90 минут, 3 блока (Математика 30 / Физика 15 / Химия 15). Выход из теста невозможен»
  - Выбор варианта (1–100, реальные архивы НТТ за 2019–2024)
  - Кнопка «Начать экзамен»
- **В процессе теста:** идентично `Test Screen` + усиленные ограничения:
  - Полноэкранный режим (Fullscreen API), предупреждение при попытке выйти
  - Вкладка «Навигатор блоков» — переход между блоками с подтверждением (блок нельзя вернуть)
  - Таймер каждого блока отдельно + общий таймер
- **Результаты симулятора:**
  - Балл по системе НТТ (из 189)
  - Сравнение с порогом вузов (таблица: «ТГТУ — 140 баллов — ✅ / ТИИМ — 160 — ❌»)
  - Исторический прогресс: график «Мои баллы по 5 последним симуляциям»
  - Тепловая карта слабых тем по предметам

**Логика:**
- Вопросы скачиваются заранее (при открытии экрана), доступны offline
- По завершении таймера — принудительная отправка, без возможности доделать
- Статистика хранится отдельно от обычных тестов (таблица `ntt_attempts`)

**Переходы:**
- → `Home` (только до старта теста)
- → `NTT Result Screen` (после завершения)
- → `Portfolio` (через результат — «К истории симуляций»)

---

### 2.5 Portfolio 🔒

**Цель:** История всех результатов ученика. Визуализация прогресса. Хранилище сертификатов.

**Ключевые элементы UI:**
- **Шапка:** Аватар | ФИО | Класс | Школа | Общий рейтинг в классе
- **Вкладки:** `Результаты` | `НТТ-симуляции` | `Сертификаты`
- **Вкладка «Результаты»:**
  - Фильтры: по предмету, по дате, по учителю
  - Список карточек: дата, название теста, предмет, результат %, место в классе
  - График «Динамика по предметам» (линейный, последние 10 результатов по каждому)
- **Вкладка «НТТ-симуляции»:**
  - Список попыток с баллом по системе НТТ (из 189)
  - График динамики баллов
- **Вкладка «Сертификаты»:**
  - Карточки сертификатов: тест, дата, результат, QR-код
  - Кнопка «Скачать PDF» | «Поделиться»
- **Кнопка «Экспорт портфолио»** — PDF-сводка (для передачи в вуз)

**Переходы:**
- → `Result Screen` (по клику на конкретный результат)
- → `NTT Simulator` (кнопка «Запустить новую симуляцию»)
- → External: скачать PDF

## 3. Учитель (role = teacher)

---

### 3.1 Teacher Dashboard 🔒

**Цель:** Главная страница учителя. Обзор всех классов, активных тестов, ключевых метрик.

**Ключевые элементы UI:**
- **Header:** Имя учителя | Предмет | Школа | Тарифный план (Freemium: «28/30 учеников» с кнопкой «Upgrade»)
- **KPI-плашки (4 штуки):**
  - Активных тестов сейчас
  - Учеников прошли сегодня
  - Средний результат за последние 7 дней
  - Тестов завершено за месяц
- **Раздел «Активные тесты»:**
  - Карточки: название, класс, прогресс прохождения (17/24 учеников), дедлайн countdown, кнопка «Смотреть результаты»
- **Раздел «Мои классы»:**
  - Список классов с аватарками (max 3 в ряд) и числом учеников
  - Кнопка «+ Добавить класс»
- **Быстрые действия:** «➕ Создать тест» | «📋 Назначить тест» | «📊 Открыть аналитику»
- **Лента событий (правая колонка на desktop):** «Акбаров А. завершил тест — 92%», «Юсупова Д. просрочила тест»
- **Sidebar Nav:** `Главная` | `Создать тест` | `Библиотека` | `Результаты` | `Аналитика`

**Переходы:**
- → `Test Builder` (кнопка «Создать тест»)
- → `Test Library` (sidebar)
- → `Class Results` (по кнопке «Смотреть результаты» у конкретного теста)
- → `Analytics` (sidebar)
- → `Class Management` (по клику на класс)

### 3.2 Test Builder 🔒 🤖

**Цель:** Создание теста вручную или с помощью AI. Основной инструмент учителя, главное конкурентное преимущество Zedly.

**Ключевые элементы UI:**
- **Шаг 1 — Настройки теста:**
  - Название теста (обязательно)
  - Предмет (dropdown из каталога предметов)
  - Класс(ы) — мультиселект
  - Тип: `Контрольная` / `Домашняя работа` / `Олимпиада` / `НТТ-формат`
  - Лимит времени (переключатель вкл/выкл + поле минут)
  - Порог сертификата (%) + переключатель «Выдавать сертификат»
  - Настройки: «Показывать правильные ответы после» (сразу / после дедлайна / никогда)
  - Кнопка «Далее»
- **Шаг 2 — Добавление вопросов (два пути):**
  - **Путь A — Вручную:**
    - Кнопка «+ Добавить вопрос»
    - Редактор вопроса: текст (rich-text, LaTeX-поддержка), 4 варианта ответа, radio «Правильный», поле «Объяснение», тег темы (из таксономии предмета)
    - Drag-and-drop для изменения порядка вопросов
    - Bulk-import: «Загрузить из .docx / .xlsx» (шаблон скачивается)
  - **Путь B — AI-генерация:** 🤖
    - Выбор источника: «Из учебника» (загрузить PDF/фото страницы) | «По теме» (текстовый запрос)
    - Параметры: количество вопросов (5–60), сложность (лёгкие / средние / сложные / микс), тип (MCQ / true-false)
    - Превью: AI генерирует вопросы, каждый можно отредактировать / удалить / одобрить
    - Индикатор генерации: прогресс-бар «Генерирую вопросы... 12/30»
- **Шаг 3 — Предпросмотр и публикация:**
  - Превью теста в режиме ученика
  - Кнопки: «Сохранить черновик» | «Опубликовать» | «Добавить в библиотеку» (Marketplace)
  - При публикации: выбор классов + дедлайн (date-time picker)

**Логика:**
- Автосохранение черновика каждые 30 секунд
- AI-генерация: очередь задач (Bull/Celery), статус polling каждые 3 секунды
- Лимит для Freemium: не более 30 вопросов в тесте; более 30 — paywall

**Переходы:**
- → `Test Library` (кнопка «Сохранить в библиотеку»)
- → `Class Results` (после публикации — предложение «Перейти к результатам»)
- → `Teacher Dashboard` (кнопка «Отмена» / после публикации)

### 3.3 Test Library 🔒

**Цель:** Управление собственными тестами и доступ к Marketplace чужих тестов.

**Ключевые элементы UI:**
- **Вкладки:** `Мои тесты` | `Marketplace`
- **Вкладка «Мои тесты»:**
  - Поиск + Фильтры: предмет, класс, статус (черновик / опубликован / архив)
  - Карточки тестов: название, предмет, класс, дата создания, число вопросов, кол-во прохождений
  - Действия на карточке: «✏️ Редактировать» | «📋 Назначить» | «📊 Результаты» | «🗑️ Удалить» | «📤 Поделиться в Marketplace»
- **Вкладка «Marketplace»:**
  - Поиск по ключевому слову
  - Фильтры: предмет, класс, язык, рейтинг (★★★★+), автор (верифицированный / любой)
  - Карточки чужих тестов: название, автор, предмет, класс, число вопросов, рейтинг, число использований
  - Кнопка «Добавить в мои тесты» (копирует тест в библиотеку)
  - Кнопка «Предпросмотр» (просмотр вопросов без ответов)

**Переходы:**
- → `Test Builder` (кнопка «+ Создать тест»)
- → `Test Builder` (редактирование существующего теста)
- → `Class Results` (результаты конкретного теста)

---

### 3.4 Class Results 🔒

**Цель:** Результаты конкретного теста по конкретному классу. Детальный разбор после проведения теста.

**Ключевые элементы UI:**
- **Заголовок:** Название теста | Класс | Дата | Статус («Идёт» / «Завершён»)
- **Сводка (Summary bar):**
  - Прошли: X из Y учеников
  - Средний результат: **78%**
  - Лучший результат: 98% (Акбаров А.)
  - Самый слабый: 34% (Каримов Б.)
  - Среднее время: 14 мин 22 сек
- **Таблица результатов:**
  - Колонки: Место | ФИО | Результат % | Правильных | Время | Статус | Действия
  - Сортировка по любой колонке
  - Строка «Просрочили» (accordion): ученики, не прошедшие в срок
  - Экспорт: «Скачать Excel» | «Скачать PDF»
- **Тепловая карта вопросов:**
  - Матрица: строки = вопросы, столбцы = ученики, цвет = правильно/неправильно
  - Красные вопросы = проблемные для класса (правильных < 50%)
- **Раздел «Слабые темы»:**
  - «Тема: Производная. Ошибок: 45%. Рекомендация: провести разбор» с кнопкой «Создать тест по теме» → `Test Builder` с предзаполненной темой

**Переходы:**
- → `Student Result Screen` (по клику на ученика — просмотр его разбора ошибок)
- → `Test Builder` (кнопка «Создать тест по слабым темам»)
- → `Analytics` (ссылка «Полная аналитика класса»)

---

### 3.5 Analytics 🔒

**Цель:** Углублённая аналитика по классам, предметам, динамике прогресса. Инструмент для принятия педагогических решений.

**Ключевые элементы UI:**
- **Фильтры (верхний ряд):** Класс | Предмет | Период (неделя / месяц / четверть / год)
- **Раздел «Прогресс класса»:**
  - Линейный график «Средний балл класса по времени»
  - Распределение: гистограмма баллов (0–100%) — сколько учеников в каждом диапазоне
- **Раздел «Слабые темы»:**
  - Горизонтальный bar-chart: темы, отсортированные по % ошибок
  - Клик на тему → список вопросов с ошибками + кнопка «Создать практику»
- **Раздел «Рейтинг учеников»:**
  - Таблица: ФИО | Средний балл | Тенденция (↑↓) | Пройдено тестов | Последняя активность
  - Выделение «Требуют внимания» (красным): балл упал на > 15% за месяц
- **Раздел «Активность»:**
  - Heatmap по дням: «сколько тестов прошли» (как GitHub Activity)

**Переходы:**
- → `Class Results` (по клику на конкретный тест в аналитике)
- → `Test Builder` (кнопка «Создать тест по слабой теме»)

## 4. Директор (role = director)

---

### 4.1 School Dashboard 🔒

**Цель:** Аналитика всей школы в одном экране. Инструмент для директора и завучей.

**Ключевые элементы UI:**
- **KPI-плашки (6 штук):**
  - Активных учителей за месяц
  - Тестов проведено за месяц
  - Учеников прошли тесты за месяц
  - Средний балл по школе
  - Процент учеников с результатом ≥ 70%
  - Активных НТТ-симуляций
- **Раздел «Топ классов»:**
  - Bar-chart: классы по среднему баллу (все предметы вместе)
  - Фильтр по предмету
- **Раздел «Активность учителей»:**
  - Мини-таблица: ФИО | Тестов создано | Тестов проведено | Учеников охвачено
  - «Неактивные учителя» (не создавали тесты > 30 дней) — выделены
- **Раздел «Динамика школы»:**
  - Линейный график среднего балла по школе за учебный год (по месяцам)
- **Раздел «Готовность к НТТ»:**
  - Процент 11-классников с > 3 симуляциями
  - Средний балл по симуляциям в 11 классе vs порог ведущих вузов

**Переходы:**
- → `Teacher Activity` (по клику на учителя или по sidebar)
- → `Report Generator` (sidebar)
- → `License Management` (sidebar или баннер если лицензия истекает)

### 4.2 Teacher Activity 🔒

- **Список учителей** (таблица + поиск):

- **Детальная карточка учителя** (по клику):
  - Список тестов с датами и результатами классов

  - Средний балл учеников этого учителя vs среднее по школе
- **Фильтры:** по предмету, по активности (активен / неактивен / никогда не использовал)

**Переходы:**

- **Раздел «Сертификаты»:** список с QR-кодами

### 5.2 Notifications Settings 🔒 📲

### 6.1 District Dashboard 🔒

**Цель:** Агрегированная аналитика по всем школам района. Мониторинг образовательных результатов на уровне РОНО.

**Ключевые элементы UI:**
- **Выбор района:** dropdown (если инспектор курирует несколько районов)
- **KPI-плашки:**
  - Школ в районе / подключённых к Zedly
  - Учителей активных за месяц
  - Учеников охвачено тестированием
  - Средний балл по району
  - Динамика: +/- % vs прошлый месяц
- **Раздел «Рейтинг школ»:**
  - Таблица: Школа | Средний балл | Тестов проведено | Охват учеников % | Тренд | Детали
  - Клик → детальная аналитика школы (read-only, аналог School Dashboard)
- **Раздел «Предметная аналитика»:**
  - Heatmap: строки = школы, столбцы = предметы, цвет = средний балл

  - По всем школам: % 11-классников с > 3 симуляциями + средний балл

### 6.2 School Comparison 🔒

**Цель:** Сравнение двух и более школ района по ключевым метрикам.

**Ключевые элементы UI:**
- **Выбор школ для сравнения:** мультиселект (2–6 школ), кнопка «Сравнить»
- **Режимы сравнения:**
  - `По предметам` — grouped bar chart для каждого предмета
  - `По динамике` — линейный график среднего балла за выбранный период
  - `По охвату` — % учеников, прошедших хотя бы 1 тест за месяц
  - `По активности учителей` — среднее число тестов на учителя
- **Таблица сравнения:**
  - Строки = метрики, столбцы = школы
  - Лучшее значение выделено зелёным, худшее — красным
- **Кнопка «Экспорт»**: PDF с графиками для отчёта

**Переходы:**
- → `District Dashboard` (breadcrumb)
- → `Quarterly Report` (кнопка «Включить в квартальный отчёт»)
- → External: скачать PDF
