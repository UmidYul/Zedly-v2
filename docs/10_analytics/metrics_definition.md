## Метрики ученика

### `average_score`
**Средний балл по всем тестам за период.**

- **Формула:** `sum(scores) / count(completed_tests)` за выбранный период
- **Единица:** баллы (0–100)
- **Период по умолчанию:** текущий учебный месяц
- **Применение:** дашборд ученика, отчёт учителя, прогноз НТТ

---

### `weak_topics`
**Темы с уровнем ошибочности выше 40%.**

- **Формула:** темы, где `(wrong_answers / total_answers) > 0.40` за последние 3 теста
- **Единица:** список тем (строки)
- **Применение:** персональные рекомендации ученику, уведомления учителю
- **Связанный алгоритм:** см. раздел [Weak Topic Detection](#алгоритм-weak-topic-detection)

---

### `improvement_rate`
**Динамика среднего балла за последние 4 недели.**

- **Формула:** `average_score(неделя 4) - average_score(неделя 1)` — скользящее сравнение
- **Единица:** Δ баллы (положительное = рост, отрицательное = снижение)
- **Применение:** трендовый график на дашборде ученика

---

### `ntt_readiness_score`
**Прогнозируемый балл на Национальном Тестировании (НТТ) на основе истории активности.**

- **Формула:** ML-модель (см. раздел [Предсказательная аналитика](#предсказательная-аналитика-v2))
- **Единица:** баллы в шкале НТТ
- **Точность модели:** обновляется с каждым новым батчем данных
- **Применение:** прогноз-виджет на дашборде ученика и родителя

---

### `completion_rate`
**Процент начатых тестов, которые были завершены.**

- **Формула:** `completed_tests / started_tests × 100`
- **Единица:** % (0–100)
- **Применение:** индикатор вовлечённости ученика

---

## Метрики класса

### `class_average`
**Средний балл класса по конкретному предмету.**

- **Формула:** `sum(student_scores) / count(students_in_class)` по предмету за период
- **Единица:** баллы (0–100)
- **Применение:** отчёт учителя, сравнение классов внутри школы

---

### `class_weak_topics`
**Топ-5 тем с наибольшим процентом ошибок по классу.**

- **Формула:** агрегация `weak_topics` по всем ученикам класса; ранжирование по среднему `error_rate`
- **Единица:** список из 5 тем с % ошибок
- **Применение:** приоритизация тем для повторения, уведомления учителю

---

### `participation_rate`
**Процент учеников класса, прошедших конкретный тест.**

- **Формула:** `students_completed_test / total_students_in_class × 100`
- **Единица:** % (0–100)
- **Применение:** контроль охвата тестирования учителем

---

### `teacher_engagement`
**Количество активных тестов, созданных или назначенных учителем за месяц.**

- **Формула:** `count(tests_created_or_assigned)` за календарный месяц
- **Единица:** количество тестов
- **Применение:** метрика активности учителя для школьной администрации

---

## Метрики школы

### `school_average_by_subject`
**Матрица средних баллов: класс × предмет.**

- **Формула:** `class_average` для каждой пары (класс, предмет) → сведено в матрицу
- **Единица:** баллы (0–100) по каждой ячейке матрицы
- **Применение:** дашборд директора, выявление слабых классов/предметов

---

### `teacher_activity_rate`
**Процент учителей школы с минимум 2 активными тестами в месяц.**

- **Формула:** `count(teachers_with_teacher_engagement ≥ 2) / count(total_teachers) × 100`
- **Единица:** % (0–100)
- **Применение:** KPI администрации по вовлечённости педагогов

---

### `improvement_vs_last_quarter`
**Динамика среднего балла школы по сравнению с предыдущим учебным кварталом.**

- **Формула:** `school_avg_current_quarter - school_avg_previous_quarter`
- **Единица:** Δ баллы
- **Применение:** квартальный отчёт для директора и РОНО

---

### `license_utilization`
**Процент оплаченных лицензионных мест, которые используются.**

- **Формула:** `active_licensed_users / total_purchased_licenses × 100`
- **Единица:** % (0–100)
- **Применение:** мониторинг эффективности использования лицензии, биллинговая аналитика

---

## Метрики района (РОНО)

### `district_ranking`
**Рейтинг школ района по среднему баллу.**

- **Формула:** ранжирование школ по `school_average_by_subject` → агрегированный общий балл
- **Единица:** позиция в рейтинге (1 = лучшая школа)
- **Применение:** отчёт РОНО, сравнительный анализ школ

---

### `coverage_rate`
**Процент школ района, подключённых к платформе Zedly.**

- **Формула:** `schools_on_platform / total_schools_in_district × 100`
- **Единица:** % (0–100)
- **Применение:** KPI по проникновению платформы в регионе

---

### `quarterly_progress`
**Динамика среднего балла по району за учебный год (поквартально).**

- **Формула:** `district_avg_score` по каждому кварталу → временной ряд Q1–Q4
- **Единица:** баллы (0–100) × 4 точки
- **Применение:** годовой отчёт РОНО, оценка системного прогресса

---

## Алгоритм Weak Topic Detection

**Цель:** автоматически выявлять темы, требующие дополнительного внимания, и уведомлять учителя.

**Триггер:** если по теме `X` в классе `Y` за последние 3 теста средний процент ошибок превышает 40%, тема помечается как `weak_topic`.

**Псевдокод:**
```python
for topic in class_topics:
    recent_tests = get_last_n_tests(topic, class_id, n=3)
    avg_error_rate = mean([test.error_rate for test in recent_tests])
    
    if avg_error_rate > 0.40:
        mark_as_weak_topic(topic, class_id)
        send_telegram_notification(teacher_id, topic, avg_error_rate)
```

**Действие:** учитель получает автоматическое уведомление в Telegram с указанием темы и процента ошибок.

**Периодичность пересчёта:** после каждого завершённого теста по классу.

## Предсказательная аналитика (V2)

### NTT Readiness Score

**Описание:** ML-модель для прогнозирования итогового балла ученика на Национальном Тестировании (НТТ) на основе исторических данных активности в Zedly.

**Входные признаки модели:**
- `average_score` — динамика за последние 3 месяца
- `improvement_rate` — траектория роста
- `completion_rate` — регулярность работы с платформой
- `weak_topics` — количество и глубина слабых тем
- Исторические данные: результаты НТТ выпускников Zedly (размеченный датасет)

**Обучение модели:**
- Данные: реальные результаты НТТ учеников, ранее использовавших Zedly (с согласия школ)
- Валидация: сравнение прогноза `ntt_readiness_score` с фактическими результатами НТТ
- Переобучение: по мере накопления новых данных (ежегодно или при значимом смещении)

**Ограничения V2:**
- Точность модели зависит от объёма исторических данных по конкретному предмету и региону
- Прогноз носит ориентировочный характер и не гарантирует итоговый результат НТТ

**Дорожная карта:** расширение модели на прогноз по отдельным предметам (V3+).
