## Критические сценарии MVP

---

### 1. Создание теста учителем

```
Given: учитель авторизован и находится в конструкторе тестов

When: учитель добавляет 5 вопросов с вариантами ответов
  And: указывает предмет, тему и время прохождения
  And: нажимает «Опубликовать»

Then: тест сохраняется в БД со статусом published
  And: запись теста содержит school_id учителя
  And: тест доступен для назначения любому классу школы
  And: тест НЕ отображается в интерфейсе учителей других школ
  And: запрос к API теста с токеном другой школы возвращает 403 Forbidden
```

**Граничные условия:**
- Тест с 0 вопросами не может быть опубликован; кнопка «Опубликовать» неактивна.
- Максимум 100 вопросов на тест; при превышении показывается ошибка валидации.

---

### 2. Прохождение теста учеником

```
Given: учитель назначил тест классу с дедлайном
  And: ученик является членом этого класса и авторизован

When: ученик открывает ссылку на тест
  And: отвечает на все вопросы
  And: нажимает «Завершить»

Then: все ответы сохраняются в таблице answers с уникальным session_id
  And: итоговый балл рассчитывается немедленно и отображается ученику
  And: ученик получает карточку-результат с разбивкой по темам
  And: аналитика класса (class_average, class_weak_topics) обновляется в течение 15 минут
  And: учитель видит нового участника в статистике теста
```

**Граничные условия:**
- Повторное прохождение одного теста недоступно; попытка открыть тест повторно показывает карточку результата.
- Если дедлайн истёк, тест недоступен для старта; ученик видит сообщение о завершении приёма.

---

### 3. НТТ-симулятор

```
Given: ученик авторизован и запускает НТТ-симулятор
  And: таймер установлен на 90 минут
  And: ученик ответил на часть вопросов

When: истекают 90 минут (независимо от действий ученика)

Then: система автоматически завершает сессию с текущими ответами
  And: неотвеченные вопросы засчитываются как пропущенные (no_answer)
  And: ученик перенаправляется на экран результата
  And: результат содержит итоговый балл и разбор ошибок по темам
  And: сессия получает статус completed (не expired)
  And: ntt_readiness_score пересчитывается с учётом новой сессии
```

**Граничные условия:**
- Если ученик нажал «Завершить» до истечения времени — поведение идентично; сессия получает статус completed.
- Разрыв соединения в момент автозавершения: ответы, сохранённые до разрыва, принимаются; после восстановления соединения ученик видит результат.

---

### 4. Изоляция школ

```
Given: пользователь (учитель или ученик) авторизован в школе A
  And: существует ресурс (тест, класс, отчёт), принадлежащий школе B

When: пользователь выполняет запрос к ресурсу школы B
     (прямой вызов API, подбор ID, манипуляция параметрами)

Then: система возвращает HTTP 403 Forbidden
  And: тело ответа не содержит никакой информации о ресурсе школы B
  And: ответ неотличим от ответа на запрос несуществующего ресурса (security through obscurity)
  And: попытка записывается в audit_log с полями: user_id, school_id_requester, resource_id, timestamp
  And: администратор безопасности получает Telegram-алерт в течение 60 секунд
```

**Граничные условия:**
- Попытка перебора ID через последовательные запросы (> 20 запросов к чужим ресурсам за 5 минут) вызывает блокировку токена и алерт с пометкой enumeration_attack.

---

### 5. Offline-тест

```
Given: ученик авторизован и открыл тест при активном интернет-соединении
  And: Service Worker закэшировал данные теста локально

When: ученик теряет интернет-соединение
  And: продолжает отвечать на вопросы
  And: нажимает «Завершить»

Then: ответы сохраняются в локальном хранилище (IndexedDB) с меткой pending_sync
  And: интерфейс показывает индикатор «Нет сети — ответы сохранены локально»

When: интернет-соединение восстанавливается

Then: ответы автоматически синхронизируются с сервером без действий со стороны ученика
  And: метка pending_sync снимается
  And: если время теста истекло в период offline — ответы всё равно принимаются
  And: сессия получает дополнительный флаг late_submission: true
  And: учитель видит пометку «Сдано офлайн» в статистике теста
```

**Граничные условия:**
- Если тот же тест был сдан online с другого устройства в период offline на первом устройстве — принимается первая по timestamp запись; дублирующая помечается sync_conflict для ручной проверки.

### 6. Аналитика директора — триггер конверсии

```
Given: школа использует Zedly на freemium-плане
  And: 7 из 8 учителей школы активно использовали платформу за последние 30 дней
     (т.е. доля активных учителей ≥ 85%)

When: директор (пользователь с ролью school_admin) входит в панель школы

Then: система показывает уведомление-баннер:
     «92% ваших учителей используют Zedly — перейдите на школьную лицензию»
  And: баннер содержит кнопку «Купить лицензию»
  And: кнопка ведёт на страницу оплаты с предзаполненными данными школы (Payme / Click)
  And: событие conversion_trigger_shown фиксируется в аналитике с параметрами:
       school_id, active_teacher_rate, timestamp
```

**Граничные условия:**
- Если директор закрыл баннер — он не показывается повторно в течение 7 дней.
- Баннер не показывается, если школа уже на платном плане.

## Расширенные сценарии

---

### 7. Регистрация через Telegram — новый учитель

```
Given: новый учитель переходит по реферальной ссылке или находит бота Zedly в Telegram
  And: учитель ещё не зарегистрирован в системе

When: учитель нажимает /start в боте
  And: проходит шаги онбординга: вводит имя, выбирает школу, предмет и класс
  And: подтверждает регистрацию

Then: создаётся аккаунт учителя со статусом pending_approval
  And: администратор школы получает уведомление в Telegram с запросом на подтверждение
  And: после подтверждения статус меняется на active
  And: учитель получает сообщение «Добро пожаловать» с кнопкой «Открыть Zedly»
  And: учитель может войти в Telegram Mini App без отдельного пароля (auth через Telegram ID)
```

**Граничные условия:**
- Если школа не найдена в системе — учитель получает инструкцию по добавлению школы и контакт поддержки.
- Повторный /start от уже зарегистрированного Telegram ID открывает существующий аккаунт без создания дубликата.

---

### 8. Регистрация через Telegram — ученик по ссылке

```
Given: учитель сгенерировал пригласительную ссылку для класса
  And: ученик переходит по ссылке в Telegram

When: ученик нажимает «Вступить в класс» в Mini App
  And: вводит своё имя (псевдоним допускается)

Then: ученик привязывается к классу через telegram_id без email и пароля
  And: учитель видит нового ученика в списке класса
  And: ученик немедленно получает доступ к назначенным тестам
  And: повторный переход по той же ссылке с того же Telegram ID не создаёт дубликат
```

---

### 9. Генерация отчёта РОНО — сводный по району

```
Given: пользователь авторизован с ролью district_admin (РОНО)
  And: выбирает период (учебный квартал) и предмет

When: нажимает «Сформировать отчёт»

Then: система ставит задачу в очередь и возвращает task_id
  And: в течение 30 секунд формируется PDF-отчёт, содержащий:
       — district_ranking (рейтинг школ по среднему баллу)
       — coverage_rate (% школ на платформе)
       — quarterly_progress (динамика по кварталам)
       — топ-3 школы с наибольшим ростом
       — топ-3 школы с наибольшим снижением
  And: администратор РОНО получает уведомление в Telegram со ссылкой на скачивание
  And: ссылка действительна 7 дней
  And: отчёт содержит данные только тех школ, которые входят в район пользователя
```

---

### 10. Генерация отчёта РОНО — выгрузка в Excel

```
Given: пользователь с ролью district_admin просматривает сводный отчёт района

When: нажимает «Экспорт в Excel»

Then: формируется .xlsx-файл с листами:
       — «Сводная таблица» (матрица школа × предмет × средний балл)
       — «Динамика» (поквартальные данные по каждой школе)
       — «Рейтинг» (district_ranking за выбранный период)
  And: числовые значения в ячейках (не текст) — пригодны для дальнейшей обработки
  And: файл скачивается напрямую без очереди (< 10 секунд для района до 200 школ)
  And: имя файла содержит название района и период: `district_report_tashkent_Q1_2026.xlsx`
```

### 11. Межшкольный челлендж — создание

```
Given: пользователь с ролью district_admin или school_admin авторизован

When: создаёт межшкольный челлендж, указывая:
       — название, предмет, класс (параллель)
       — список приглашённых школ (минимум 2)
       — даты проведения и дедлайн

Then: челлендж сохраняется со статусом draft
  And: администраторы приглашённых школ получают уведомление в Telegram с запросом на участие
  And: после подтверждения всеми школами статус меняется на active
  And: учителя участвующих школ видят челлендж в своей панели
  And: тест для челленджа изолирован — участники не видят вопросы до старта
```

---

### 12. Межшкольный челлендж — участие и лидерборд

```
Given: межшкольный челлендж активен
  And: ученик является членом класса школы-участника

When: ученик проходит тест челленджа в установленные сроки

Then: результат засчитывается в общий зачёт челленджа
  And: лидерборд обновляется в реальном времени (задержка ≤ 60 секунд)
  And: лидерборд показывает агрегированный рейтинг школ (не индивидуальные баллы учеников из других школ)
  And: индивидуальные результаты ученика видны только его учителю и ему самому
  And: по завершении челленджа все участвующие школы получают итоговый отчёт в Telegram
```

---

### 13. Генерация сертификата ученику

```
Given: ученик завершил тест или челлендж с баллом выше порогового значения
     (порог задаётся учителем при создании теста; по умолчанию 80%)

When: система определяет, что условие для сертификата выполнено

Then: автоматически генерируется PDF-сертификат, содержащий:
       — имя ученика
       — название теста / челленджа
       — итоговый балл
       — дату выдачи
       — имя учителя и школу
       — уникальный верификационный QR-код
  And: сертификат доступен ученику в его профиле для скачивания
  And: ученик получает уведомление в Telegram со ссылкой на сертификат
  And: QR-код ведёт на публичную страницу верификации без авторизации
```

---

### 14. Генерация сертификата — массовая выдача классу

```
Given: учитель завершил тест для класса
  And: несколько учеников набрали балл выше порога

When: учитель нажимает «Выдать сертификаты классу»

Then: система ставит задачу генерации в очередь
  And: в течение 60 секунд генерируются индивидуальные PDF для каждого прошедшего порог ученика
  And: учитель получает ZIP-архив со всеми сертификатами
  And: каждый ученик получает уведомление в Telegram индивидуально
  And: ученики, не достигшие порога, не получают сертификат и не уведомляются об этом в контексте массовой выдачи
```

---

### 15. AI-генерация теста по теме

```
Given: учитель авторизован и открывает конструктор тестов
  And: выбирает режим «Создать с помощью AI»

When: учитель вводит тему, предмет, класс (параллель) и желаемое количество вопросов (5–30)
  And: нажимает «Сгенерировать»

Then: система обращается к AI-модели и в течение 15 секунд возвращает черновик теста
  And: черновик содержит указанное количество вопросов с вариантами ответов и отмеченным правильным ответом
  And: учитель попадает в режим редактирования, где может изменить, удалить или добавить вопросы
  And: тест не публикуется автоматически — требуется явное действие учителя «Опубликовать»
  And: в метаданных теста фиксируется флаг ai_generated: true для аналитики качества
```

**Граничные условия:**
- Если AI-сервис недоступен — учитель получает сообщение об ошибке и предложение создать тест вручную; функциональность конструктора не нарушается.
- Сгенерированные вопросы проходят автоматическую проверку на минимальное качество (наличие правильного ответа, длина вопроса > 10 символов); вопросы, не прошедшие проверку, помечаются для ревью учителем.

---

### 16. AI-генерация теста по загруженному тексту

```
Given: учитель авторизован и открывает конструктор тестов в режиме «AI по тексту»

When: учитель загружает документ (PDF или .docx, до 10 MB) или вставляет текст вручную
  And: указывает количество вопросов и нажимает «Сгенерировать»

Then: система извлекает текст из документа
  And: передаёт его в AI-модель вместе с инструкцией генерировать вопросы строго по содержанию текста
  And: в течение 20 секунд возвращается черновик теста
  And: каждый вопрос содержит опциональную ссылку на фрагмент исходного текста (цитату-подсказку)
  And: учитель попадает в режим редактирования перед публикацией
  And: исходный документ не сохраняется на сервере после генерации (удаляется по завершении задачи)
```

**Граничные условия:**
- Файлы с защитой паролем или нечитаемым сканом возвращают ошибку с описанием причины.
- Текст на языках, отличных от узбекского и русского, принимается, но качество генерации не гарантируется; учитель получает предупреждение.
