## 2. Барьеры входа для конкурентов

### Накопленные данные школ
- Zedly аккумулирует длинные исторические ряды по `school_id`, классам, предметам и темам, что повышает точность сравнений по периодам.
- При смене платформы школа теряет сопоставимость «год к году», а значит теряет основу для управленческих выводов.
- Исторические данные формируют обучающий контур для улучшения рекомендаций, который новому игроку нужно строить заново.

### Сетевой эффект marketplace тестов
- Учителя и методисты публикуют и переиспользуют проверенные тесты внутри экосистемы Zedly.
- С ростом числа школ растет качество и разнообразие контента: больше предметов, уровней сложности и локальных формулировок.
- Новому конкуренту трудно быстро собрать сопоставимый каталог валидированного контента без доступа к действующей сети пользователей.

### Интеграция с госструктурами (переключиться = потерять историю)
- Zedly встраивается в контур отчетности школы, РОНО и Минпроса с едиными метриками и регламентами.
- Переход на другую платформу означает миграцию данных, переутверждение форм отчетности и риск потери непрерывности мониторинга.
- Для государственного уровня ключевой риск переключения: временная «слепая зона» в аналитике во время миграции.

### Lock-in через сертификаты (вузы и работодатели проверяют Zedly-сертификаты)
- Сертификаты Zedly содержат верифицируемые данные попыток, результатов и периодов обучения.
- При признании сертификатов внешними стейкхолдерами (вузы, работодатели) ценность платформы выходит за пределы школы.
- Экосистемный эффект: чем больше организаций проверяют Zedly-сертификаты, тем выше стоимость отказа от платформы для пользователей.

## 3. Что усиливает ценность для всех ролей одновременно
- Единый контур данных без ручной «склейки» между школой, районом и министерством.
- Telegram-нативный вход для ежедневных операций учителя и ученика без дополнительного порога внедрения.
- Offline-first сценарии для регионов, где стабильный интернет нельзя считать гарантией.
- Мультитенантность через `school_id` и RLS: масштабируемость без компромисса по изоляции данных.
