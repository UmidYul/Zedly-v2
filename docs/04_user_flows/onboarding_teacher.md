# docs/04_user_flows/onboarding_teacher.md

> **Проект:** Zedly — SaaS-платформа онлайн-тестирования и аналитики для школ Узбекистана  
> **Файл:** Онбординг нового учителя  
> **Цель флоу:** Учитель проходит регистрацию и получает первые результаты класса за **≤ 15 минут**  
> **Приоритет:** MVP

## Шаг 1 — Регистрация

### Что видит учитель

Страница `/onboarding` с двумя вариантами входа:

```
┌─────────────────────────────────────────────┐
│           Добро пожаловать в Zedly          │
│                                             │
│  [  Войти через Telegram  ]  ← рекомендован │
│                                             │
│  или зарегистрируйтесь по email:            │
│  Email:    [___________________]            │
│  Пароль:   [___________________]            │
│  [        Создать аккаунт       ]           │
└─────────────────────────────────────────────┘
```

**UI-подсказка:** «Большинство учителей входят через Telegram — быстрее и безопаснее»

### Telegram Login Widget (основной путь)

**Actor → System → Response:**

| Шаг | Actor | Action | System | Response |
|-----|-------|--------|--------|----------|
| 1.1 | Учитель | Нажимает «Войти через Telegram» | Открывает Telegram OAuth popup | Telegram запрашивает разрешение |
| 1.2 | Учитель | Подтверждает в Telegram | Backend получает hash + user data | HMAC-SHA256 верификация подписи Telegram |
| 1.3 | System | — | Создаёт `users` запись (или находит существующую) | Генерирует JWT (15 мин) + Refresh Token (30 дней) в Redis |
| 1.4 | System | — | Устанавливает `onboarding_step = 1` в `user_metadata` | Редирект на Шаг 2 |

### Email-регистрация (резервный путь)

| Шаг | Actor | Action | System | Response |
|-----|-------|--------|--------|----------|
| 1.1 | Учитель | Вводит email + пароль | Валидирует: email-формат, пароль ≥ 8 символов | Показывает ошибки инлайн |
| 1.2 | Учитель | Нажимает «Создать аккаунт» | Проверяет уникальность email | Если занят — «Этот email уже зарегистрирован. Войти?» |
| 1.3 | System | — | Хэширует пароль (bcrypt, cost 12) | Отправляет письмо верификации |
| 1.4 | Учитель | Открывает письмо, кликает ссылку | Верифицирует токен (TTL 24 часа) | Активирует аккаунт, редирект на Шаг 2 |

### Ошибки и обработка (Шаг 1)

| Ошибка | Отображение | Действие системы |
|--------|-------------|-----------------|
| Telegram отказал в доступе | «Войти через Telegram не удалось. Попробуйте снова или используйте email» | Ничего не создаётся |
| Email занят | «Этот email уже зарегистрирован» + ссылка «Войти» | Не создаёт дублирующий аккаунт |
| Письмо не пришло | «Не получили письмо? Отправить повторно» (cooldown 60 сек) | Генерирует новый токен, инвалидирует старый |
| Токен верификации просрочен | «Ссылка устарела. Запросите новую» | Страница с кнопкой повторной отправки |

### Трекинг онбординга

```json
{ "event": "onboarding_step_completed", "step": 1, "method": "telegram|email", "user_id": "...", "timestamp": "..." }
```

## Шаг 2 — Выбор школы

### Что видит учитель

```
┌─────────────────────────────────────────────┐
│       В какой школе вы преподаёте?          │
│                                             │
│  Поиск:  [  Введите название школы...  ]    │
│                                             │
│  Или введите код школы от директора:        │
│  [  __  __  __  __  __  __  ]              │
│                                             │
│  Нет кода? Директор может прислать его      │
│  из раздела «Управление школой»             │
└─────────────────────────────────────────────┘
```

### Actor → System → Response

| Шаг | Actor | Action | System | Response |
|-----|-------|--------|--------|----------|
| 2.1 | Учитель | Вводит 3+ символа в поиск | Запрос к `schools` с debounce 300ms | Список до 10 совпадений (название + район + город) |
| 2.2 | Учитель | Выбирает школу из списка | Проверяет: школа активна, не заблокирована | Записывает `school_id` в `teacher_profile` |
| 2.2a | Учитель | Вводит код школы | Ищет в `school_invite_codes` | Если код найден — автоматически присоединяет к школе; если нет — «Код не найден» |
| 2.3 | System | — | Устанавливает `school_id` в JWT при следующем обновлении | `onboarding_step = 2`, редирект на Шаг 3 |

### Сценарий: школа не зарегистрирована в Zedly

**Триггер:** учитель не находит свою школу в поиске.

**Отображение:**
```
Не нашли школу? 
[  Сообщить директору о Zedly  ]  ← копирует pre-filled Telegram-сообщение
[  Продолжить без школы (демо-режим)  ]  ← ограниченный функционал
```

**Демо-режим:** учитель может создать тест и пройти его самостоятельно, но не может назначить реальным ученикам до привязки к школе.

### Ошибки и обработка (Шаг 2)

| Ошибка | Отображение | Действие |
|--------|-------------|----------|
| Школа заблокирована | «Эта школа временно недоступна. Свяжитесь с поддержкой» | Не присоединяет |
| Учитель уже привязан к другой школе | «Вы уже в [Школа X]. Хотите перейти в [Школа Y]?» | Подтверждение с предупреждением о потере доступа к данным |
| Код школы истёк (TTL 72 часа) | «Код устарел. Попросите директора выдать новый» | Не присоединяет |

### Трекинг

```json
{ "event": "onboarding_step_completed", "step": 2, "school_id": "...", "join_method": "search|code|demo", "timestamp": "..." }
```

## Шаг 3 — Настройка профиля (предмет и классы)

### Что видит учитель

```
┌─────────────────────────────────────────────┐
│          Расскажите о себе                  │
│                                             │
│  Ваш предмет:                               │
│  [  Математика ▼  ]                         │
│                                             │
│  Классы (можно выбрать несколько):          │
│  [ ] 5А    [ ] 5Б    [ ] 6А                 │
│  [x] 7А    [x] 7Б    [ ] 8А                 │
│  [ ] 9А                                     │
│                                             │
│  [       Продолжить →       ]               │
└─────────────────────────────────────────────┘
```

**Примечание:** Список классов берётся из `school_classes` (классы, созданные директором). Если директор не создал классы — учитель видит текстовое поле «Введите название класса» + кнопку «+Добавить».

### Actor → System → Response

| Шаг | Actor | Action | System | Response |
|-----|-------|--------|--------|----------|
| 3.1 | Учитель | Выбирает предмет | Загружает список предметов из `subjects` (фиксированный справочник + кастомные школы) | — |
| 3.2 | Учитель | Отмечает классы | Проверяет: классы принадлежат `school_id` учителя | — |
| 3.3 | Учитель | Нажимает «Продолжить» | Создаёт записи в `teacher_classes` (связка teacher_id ↔ class_id) | `onboarding_step = 3`, редирект на Шаг 4 |

### Ошибки и обработка (Шаг 3)

| Ошибка | Отображение | Действие |
|--------|-------------|----------|
| Не выбран предмет | «Выберите хотя бы один предмет» — инлайн | Блокирует переход |
| Не выбраны классы | «Добавьте хотя бы один класс» | Блокирует переход |
| В школе нет классов (директор не создал) | «Классы ещё не созданы. Введите название вашего класса» | Создаёт `school_class` с флагом `created_by_teacher: true`; директор увидит в уведомлении |

### Трекинг

```json
{ "event": "onboarding_step_completed", "step": 3, "subject_id": "...", "classes_count": 2, "timestamp": "..." }
```

### Путь A — Marketplace

| Шаг | Actor | Action | System | Response |
|-----|-------|--------|--------|----------|
| 4A.1 | Учитель | Нажимает «Выбрать из Marketplace» | Открывает Marketplace, предфильтрован по предмету+классу учителя | Список топ-10 тестов |
| 4A.2 | Учитель | Выбирает тест | Показывает превью (количество вопросов, средний балл в других школах, автор) | — |
| 4A.3 | Учитель | Нажимает «Использовать» | Копирует тест в библиотеку учителя (`test_copies`) | Тест появляется в «Мои тесты», редирект на Шаг 5 |

### Путь B — AI-генерация

| Шаг | Actor | Action | System | Response |
|-----|-------|--------|--------|----------|
| 4B.1 | Учитель | Нажимает «Сгенерировать тест» | Открывает форму загрузки | — |
| 4B.2 | Учитель | Загружает PDF/текст (глава учебника) | Валидирует: формат PDF/TXT, размер ≤ 20MB | Показывает progress bar «Анализирую текст...» |
| 4B.3 | System | — | LLM-пайплайн: парсит текст → извлекает концепции → генерирует MCQ + open-ended вопросы | «Готово! Сгенерировано 15 вопросов» |
| 4B.4 | Учитель | Просматривает вопросы | Может принять / редактировать / удалить каждый вопрос | Tooltip: «Не нравится вопрос? Нажмите ✏️ для правки» |
| 4B.5 | Учитель | Нажимает «Сохранить тест» | Сохраняет в `tests`, вопросы в `questions` с `source_type = 'ai'` | Редирект на Шаг 5. Проверяет freemium-лимит (10 AI-генераций/месяц) |

### Путь C — Ручное создание

| Шаг | Actor | Action | System | Response |
|-----|-------|--------|--------|----------|
| 4C.1 | Учитель | Нажимает «Создать вручную» | Открывает Test Builder | — |
| 4C.2 | Учитель | Добавляет вопросы (минимум 3) | Валидирует: тип вопроса, текст непустой, для MCQ ≥ 2 варианта ответа, один правильный | — |
| 4C.3 | Учитель | Устанавливает параметры: время (мин), попытки, дедлайн | Валидирует: время 1–240 мин | — |
| 4C.4 | Учитель | Нажимает «Сохранить» | Сохраняет в `tests` со статусом `draft` | Редирект на Шаг 5 |

### Ошибки и обработка (Шаг 4)

| Ошибка | Отображение | Действие |
|--------|-------------|----------|
| PDF не читается (сканированный) | «Не удалось прочитать файл. Попробуйте текстовый PDF или вставьте текст вручную» | Предлагает textarea для вставки текста |
| AI вернул < 3 вопросов | «Текст слишком короткий. Загрузите больше страниц» | Не сохраняет тест |
| Превышен freemium-лимит (10 генераций/мес) | Баннер: «Вы использовали 10/10 AI-генераций. Обновитесь для безлимитного доступа» + кнопка «Купить» | Блокирует генерацию, не блокирует ручное создание |
| Файл > 20MB | «Файл слишком большой. Максимум 20MB» | Инлайн-ошибка, не загружает |

### Actor → System → Response

| Шаг | Actor | Action | System | Response |
|-----|-------|--------|--------|----------|
| 5.1 | Учитель | Отмечает классы | Показывает количество учеников в каждом классе | — |
| 5.2 | Учитель | Устанавливает дедлайн | Валидирует: дедлайн в будущем | — |

### Freemium-проверка

Перед назначением система проверяет: количество уникальных учеников в `test_assignments` учителя за текущий период.

- **≤ 30 учеников:** назначение разрешено.  
- **> 30 учеников (freemium):** блокирующий баннер:

```
⚠️ У вас 24 бесплатных ученика, назначение в 7А добавит ещё 18.
Лимит freemium: 30 учеников.

[  Обновить план — назначить неограниченно  ]
[  Назначить только 6 учеников из 7А  ]  ← ручной выбор
```

| Ошибка | Отображение | Действие |

## Aha-moment — Первые результаты класса

> **Цель:** Учитель видит реальные результаты своего класса в течение **15 минут** после начала онбординга.

### Триггер Aha-moment

Условие срабатывания: хотя бы **1 ученик** завершил тест (`test_submissions.status = 'completed'`).

### Что видит учитель

```
┌─────────────────────────────────────────────────────────────┐
│  ✅ Первый ученик прошёл тест!                              │
│                                                             │
│  📊 Результаты 7А — Тема: Дроби                            │
│  ─────────────────────────────────────────────────────────  │
│  Прошли: 3 из 24  |  Средний балл: 74%                     │
│                                                             │
│  🔴 Слабые темы:                                            │
│  · Сложение дробей с разными знаменателями  — 33% правильно │
│  · Смешанные числа                          — 45% правильно │
│                                                             │
│  🏆 Лучший результат: Алишер А. — 95%                      │
│                                                             │
│  [  Открыть полный дашборд  ]                               │
│  [  Поделиться результатами с классом  ]                    │
└─────────────────────────────────────────────────────────────┘
```

**Читается из:** `analytics_snapshots` (обновляется каждые 15 мин + при `test_completed`-событии).  
**Не читается из raw-данных — критично для производительности.**

### Что происходит в системе при Aha-moment

1. `test_completed` event → Bull/Celery очередь
2. Worker пересчитывает `analytics_snapshots` для `(teacher_id, class_id, test_id)`
3. WebSocket push или SSE → клиент обновляет дашборд без перезагрузки
4. Система отмечает `onboarding_completed = true` в `user_metadata`
5. Фоновая задача: если учитель в школе уже 5-й активный → отправить уведомление директору «92% учителей вашей школы на Zedly»

### Трекинг Aha-moment

```json
{ "event": "aha_moment_reached", "user_id": "...", "school_id": "...", "time_from_registration_sec": 720, "students_completed": 3, "timestamp": "..." }
```

## Полная таблица Actor → System → Response (сводная)

| Шаг | Actor | Action | System | Response |
|-----|-------|--------|--------|----------|
| 1 | Учитель | Регистрация через Telegram / email | HMAC-верификация / bcrypt + email verify | JWT + Refresh Token |
| 2 | Учитель | Выбор школы (поиск / код) | Поиск в `schools` / `school_invite_codes` | `school_id` записан в профиль |
| 3 | Учитель | Выбор предмета и классов | Создание `teacher_classes` | Профиль настроен |
| 4 | Учитель | Создание теста (AI / marketplace / ручное) | LLM-пайплайн / копирование / сохранение в `tests` | Тест сохранён |
| 5 | Учитель | Назначение классу + дедлайн | Создание `test_assignments`, freemium-проверка | Уведомления ученикам |
| 6 | Учитель | Отправка ссылки в Telegram | Генерация deep-link | Ученики получают тест |
| 6+ | System | Ученик завершает тест | `test_completed` event → analytics pipeline | Aha-moment для учителя |

## Состояния онбординга

Хранятся в `user_metadata.onboarding`:

```json
{
  "current_step": 4,
  "completed_steps": [1, 2, 3],
  "registration_method": "telegram",
  "school_id": "uuid",
  "first_test_id": "uuid",
  "aha_moment_reached": false,
  "aha_moment_timestamp": null,
  "onboarding_completed": false
}
```

**Прерывание и возобновление:** если учитель закрыл браузер на любом шаге, при следующем входе система восстанавливает его с `current_step`. Прогресс-бар вверху экрана показывает, на каком шаге из 6 находится учитель.

## Граничные случаи

| Сценарий | Поведение системы |
|----------|-------------------|
| Учитель закрыл браузер на шаге 3 | При следующем входе — восстанавливает шаг 3, предмет/классы не сброшены |
| Школа удалена из системы после Шага 2 | При Шаге 3 — «Ваша школа удалена. Выберите другую» |
| Ученик не авторизован в Telegram | Ссылка на тест открывается в браузере; Telegram-уведомление не отправляется |
| Нет интернета при назначении теста | Offline-индикатор: «Нет соединения. Назначение сохранено и будет отправлено при подключении» — операция ставится в sync-очередь |
| Учитель пытается сгенерировать 11-й AI-тест на freemium | Блокирующий баннер с предложением апгрейда; предыдущие тесты работают |
| Два браузера одновременно на одном аккаунте | Последнее действие побеждает (last-write-wins); шаги онбординга синхронизируются через `user_metadata` |
| Учитель уже работал в другой школе через Zedly | При выборе новой школы — предупреждение: «Вы покидаете [Школа A]. Данные за этот период останутся, но станут недоступны» |
| AI сгенерировал фактически неверный вопрос | Учитель обязательно проходит шаг ревью перед сохранением; вопрос можно отредактировать или удалить |

## Acceptance Criteria

### AC-01: Успешная регистрация через Telegram

```gherkin
Given: Учитель открывает страницу /onboarding
When: Нажимает «Войти через Telegram» и подтверждает в приложении
Then: Аккаунт создан, JWT выдан, учитель перенаправлен на Шаг 2
And: В БД создана запись users с telegram_id и onboarding_step = 1
```

### AC-02: Выбор школы по коду

```gherkin
Given: Учитель на Шаге 2, директор выдал код "ABC123"
When: Учитель вводит "ABC123" и нажимает «Присоединиться»
Then: school_id записан в профиль учителя
And: Директор получает уведомление «[Имя учителя] присоединился к школе»
And: Код помечается как использованный (многоразовый, но логируется)
```

### AC-03: AI-генерация теста в лимитах freemium

```gherkin
Given: Учитель на freemium с 9 использованными AI-генерациями в текущем месяце
When: Загружает PDF и запускает генерацию
Then: Генерация выполняется, создаётся 10-я запись в ai_generation_log
And: После сохранения показывается предупреждение «Вы использовали последнюю бесплатную генерацию»
```

### AC-04: Блокировка 11-й AI-генерации на freemium

```gherkin
Given: Учитель на freemium с 10 использованными AI-генерациями
When: Нажимает «Сгенерировать тест»
Then: Система показывает блокирующий баннер с предложением апгрейда
And: Генерация не запускается, файл не загружается
```

### AC-05: Freemium-лимит при назначении

```gherkin
Given: Учитель на freemium уже назначил тесты 30 уникальным ученикам
When: Пытается назначить тест новому классу (5 новых учеников)
Then: Система показывает предупреждение с количеством лимита
And: Предлагает купить платный план или выбрать учеников вручную
```

### AC-06: Aha-moment достигнут

```gherkin
Given: Учитель назначил тест классу, ученик прошёл тест
When: Ученик отправляет последний ответ (test_completed event)
Then: За ≤ 30 секунд учитель видит обновлённый дашборд с результатами
And: В user_metadata устанавливается aha_moment_reached = true
And: Логируется событие aha_moment_reached с time_from_registration_sec
```

### AC-07: Восстановление онбординга после разрыва

```gherkin
Given: Учитель завершил Шаг 2, закрыл браузер
When: Открывает приложение снова
Then: Восстанавливается на Шаге 3 с уже выбранной школой
And: Прогресс-бар показывает 2/6
```

### AC-08: Нет интернета при назначении

```gherkin
Given: Учитель на Шаге 5, соединение потеряно
When: Нажимает «Назначить»
Then: Операция сохраняется в sync-очередь локально
And: Показывается offline-индикатор «Будет отправлено при подключении»
And: При восстановлении интернета — назначение выполняется автоматически
```

## KPIs онбординга

| Метрика | Цель | Измерение |
|---------|------|-----------|
| Время регистрации → Aha-moment | ≤ 15 минут | `aha_moment_timestamp - registration_timestamp` |
| Completion rate онбординга (Шаг 1 → Шаг 6) | ≥ 60% | `onboarding_completed / registrations` |
| Drop-off на Шаге 2 (выбор школы) | ≤ 25% | Воронка по `onboarding_step` |
| Drop-off на Шаге 4 (первый тест) | ≤ 30% | Воронка по `onboarding_step` |
| % учителей, достигших Aha-moment | ≥ 50% в течение 1 дня | `aha_moment_reached / onboarding_completed` |
| Среднее время до Aha-moment | ≤ 15 мин | Среднее по `aha_moment_reached` событиям |
