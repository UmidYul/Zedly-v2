# docs/04_user_flows/onboarding_teacher.md

> **Проект:** Zedly — SaaS-платформа онлайн-тестирования и аналитики для школ Узбекистана  
> **Файл:** Первый вход учителя (актуальный flow)  
> **Цель флоу:** Учитель получает рабочий доступ без самостоятельной регистрации  
> **Источник истины по API:** `docs/07_api/openapi.yaml`

---

## Обзор флоу

```
Директор создаёт аккаунт → Учитель получает login + OTP вне платформы
        ↓
  Вход на zedly.uz (login + OTP)
        ↓
  Обязательная смена OTP-пароля
        ↓
  Экран «Подключить быстрый вход» (Google / Telegram / Пропустить)
        ↓
  Дашборд учителя
```

---

## Шаг 0 — Создание аккаунта (выполняет директор)

| Шаг | Actor | Action | System | Response |
|---|---|---|---|---|
| 0.1 | Директор | `POST /api/v1/users/provision` role=teacher | Проверка school-scope и RBAC | Создаёт user c auto-login и OTP |
| 0.2 | Система | — | Генерирует `login` + `otp_password` | Возвращает данные директору |
| 0.3 | Директор | Передаёт данные учителю | Вне платформы (список/распечатка/SMS) | Учитель готов к первому входу |

Ключевой принцип:
- Учитель **не** проходит самостоятельную регистрацию.

---

## Шаг 1 — Первый вход на zedly.uz

### Что видит учитель

Экран входа с полями:
- `Логин`
- `Пароль`
- кнопка `Войти`

### Actor → System → Response

| Шаг | Actor | Action | System | Response |
|---|---|---|---|---|
| 1.1 | Учитель | Вводит login + OTP | `POST /api/v1/auth/login` | Возвращает `status=password_change_required` |
| 1.2 | Система | — | Создаёт `challenge_token` (TTL 15 мин) | Переводит на обязательный экран смены пароля |

---

## Шаг 2 — Обязательная смена OTP-пароля

### Что видит учитель

Экран:
- `Новый пароль`
- `Повторите пароль`
- кнопка `Сохранить и продолжить`

Правило пароля:
- минимум 8 символов
- минимум 1 цифра

### Actor → System → Response

| Шаг | Actor | Action | System | Response |
|---|---|---|---|---|
| 2.1 | Учитель | Вводит новый пароль | `POST /api/v1/auth/password/change-first` | Проверка challenge + валидация |
| 2.2 | Система | — | Снимает `password_temporary`, сохраняет новый hash | Выдаёт access/refresh token |
| 2.3 | Система | — | Возвращает `status=login_methods_prompt` | Показывает экран быстрых способов входа |

Гарантия:
- Пока OTP не заменён, доступ к платформе не выдаётся.

---

## Шаг 3 — Быстрые способы входа

### Что видит учитель

Экран:
- кнопка `Подключить Google`
- кнопка `Подключить Telegram`
- ссылка `Пропустить — сделаю позже`

### Telegram подключение

Без ввода технических данных вручную:
1. Пользователь нажимает `Подключить Telegram`.
2. Открывается Telegram с подготовленным действием для бота.
3. Пользователь подтверждает действие (`Start` / `Отправить`).
4. Zedly фиксирует подключение и показывает статус `подключено`.

Fallback:
- если Telegram недоступен, показывается подсказка:  
  `Установите приложение Telegram, затем вернитесь сюда`.

---

## Шаг 4 — Переход в рабочий режим

После выбора одного из вариантов на шаге 3:
- пользователь попадает в `Dashboard`.
- в профиле всегда доступен раздел `Способы входа`, где можно позже:
  - подключить Google;
  - подключить Telegram.

---

## Ошибки и обработка

| Ошибка | Отображение | Поведение |
|---|---|---|
| Неверный login/пароль | `Invalid credentials` | Доступ не выдаётся |
| Истёк `challenge_token` | `Password change challenge is invalid or expired` | Повторный вход с OTP |
| Пароль не соответствует требованиям | `Password must be at least 8 characters and contain at least one digit` | Блок сохранения |
| Telegram не подключён для Telegram Login | `Telegram пока не подключен...` | Предложение войти через login+пароль |

## Важно

Этот документ описывает только актуальный auth/onboarding flow.

Исторический сценарий самостоятельной регистрации через onboarding-token:
- выведен из продукта;
- не используется в текущем API/frontend.
