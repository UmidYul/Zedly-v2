# docs/04_user_flows/teacher_test_creation_assignment.md

> **Проект:** Zedly — SaaS-платформа онлайн-тестирования для школ Узбекистана  
> **Файл:** User-flow учителя: от создания теста до назначения и мониторинга  
> **Цель флоу:** путь учителя от клика «Создать тест» до статуса «Назначено» за **≤ 7 минут**  
> **Приоритет:** Core / MVP

---

## 1. Обзор пользовательского сценария

```mermaid
flowchart LR
    A[Teacher Dashboard] --> B[Создать тест]
    B --> C{Источник теста}
    C -->|Вручную| D[Test Builder: шаг 1-2]
    C -->|AI-генерация| E[Генерация и модерация]
    C -->|Marketplace| F[Копия в Мои тесты]
    D --> G[Test Builder: предпросмотр]
    E --> G
    F --> G
    G --> H[Назначение: AssignModal]
    H --> I[POST /tests]
    I --> J[POST /tests/{id}/assign]
    J --> K[Уведомления ученикам]
    K --> L[Class Results: мониторинг]
```

Ключевые UX-принципы:
- Mobile-first (база 375px, без потери функционала).
- Все критичные действия доступны за 1-2 клика.
- Данные читаются с первого взгляда: KPI, активные тесты, статус назначения.
- Минимум блокирующих шагов, максимум inline-валидаций.

---

## 2. Предусловия и доступы

Предусловия:
- Пользователь авторизован и имеет `role = teacher`.
- У учителя есть минимум один класс в `teacher_classes`.
- В выбранных классах есть активные ученики.

RBAC (минимум):
- `create_test`
- `assign_test`
- `view_own_results`

Если доступов нет:
- Система возвращает `403` и показывает понятное сообщение об отсутствии прав.

## 3. Вход в сценарий (Dashboard)

### 3.1 Что видит учитель

Главная страница `Teacher Dashboard`:
- Topbar + приветствие.
- Быстрые действия: `Создать тест`, `Назначить тест`, `Аналитика`.
- 4 KPI-карточки.
- Список активных тестов.
- Карточки классов.
- Лента событий.

### 3.2 Что делает система

Запрашивает:
- `GET /api/v1/users/me`
- `GET /api/v1/analytics/teacher/dashboard`

Режим обновления:
- Активные тесты: polling каждые 60 секунд.
- Лента событий: polling каждые 30 секунд.

Состояния:
- Loading: skeleton.
- Error: banner «Не удалось загрузить данные» + retry.
- Empty: если нет тестов/классов — onboarding CTA.

## 4. Шаг 1 — Выбор источника теста

После клика `Создать тест` учитель выбирает один из путей:
1. Создать вручную.
2. Сгенерировать с AI.
3. Взять из Marketplace.

UX-правило:
- Для нового учителя AI и Marketplace визуально выделены как быстрые пути.

---

## 5. Шаг 2A — Создание вручную (Test Builder)

### 5.1 Шаг «Настройки»

Поля:
- Название теста (3-120 символов).
- Предмет.
- Классы (`1+`).
- Лимит времени (5-180 минут, optional).
- Порог сертификата (1-100%, optional).
- Показ ответов: `immediately | after_deadline | never`.
- `shuffle_questions`, `shuffle_answers`.

Inline-валидация:
- Ошибки на blur и при клике `Далее`.

### 5.2 Шаг «Вопросы»

Функции:
- Добавление/удаление/редактирование вопроса.
- Reorder вопросов.
- Типы: `mcq_single`, `mcq_multiple`, `open_text`.
- Для MCQ: 2-6 вариантов.

Правила валидации вопроса:
- Пустой текст — предупреждение.
- MCQ без правильного ответа — предупреждение.
- MCQ < 2 вариантов — предупреждение.
- `mcq_single`: если выбран второй правильный, первый автоматически снимается.

Системное поведение:
- Изменения сохраняются в Zustand store немедленно.
- Автосохранение черновика в `sessionStorage` каждые 30 секунд.
- При возврате на `/tests/new` после закрытия вкладки система предлагает продолжить черновик.

### 5.3 Шаг «Публикация»

Предпросмотр:
- Название, предмет, число вопросов, ключевые темы.
- Первые вопросы для быстрого контроля.

Назначение в шаге публикации:
- Выбор классов.
- `start_at` (optional), `deadline_at` (optional).
- Попытки: 1 или без ограничений.

Блокирующие проверки перед публикацией:
- Нет классов.
- `deadline_at` в прошлом.
- `start_at >= deadline_at`.
- Нет валидных вопросов.

## 6. Шаг 2B — AI-генерация

| Шаг | Action | System Response |
|-----|--------|-----------------|
| 1 | Учитель открывает AI-генерацию | Форма входных данных (текст/PDF) |
| 2 | Загружает файл | Проверка формата и размера |
| 3 | Генерация вопросов | Показ 10-20 предложений |
| 4 | Модерация | Учитель принимает/редактирует/удаляет |
| 5 | Сохранение | Создаётся валидный draft |

Особый кейс:
- При низком качестве вопросов доступна `Перегенерировать проблемные вопросы` без потери принятых.

---

## 7. Шаг 2C — Marketplace

Сценарий:
1. Учитель открывает Marketplace.
2. Применяет фильтры (предмет, класс, рейтинг, поиск).
3. Открывает карточку и предпросмотр.
4. Нажимает `Добавить в мои тесты`.

Системный результат:
- Создаётся копия в библиотеке учителя.
- Учитель может сразу назначить или открыть на редактирование.

Примечание для MVP:
- Marketplace может работать на статичных mock-данных до появления полного API.

## 8. Назначение теста (AssignModal)

`AssignModal` открывается из:
- Dashboard (карточка класса).
- Test Library (действие `Назначить`).
- Test Builder (шаг публикации).

Поля назначения:
- `class_ids[]`
- `start_at` (optional)
- `deadline_at` (optional)
- `attempt_limit`
- `shuffle_questions`
- `shuffle_answers`

Конфликт назначения:
- Если тот же тест уже назначен в пересекающийся интервал, показывается предупреждение (не блокирует публикацию).

Submit-flow:
1. `POST /api/v1/tests`
2. `POST /api/v1/tests/{id}/assign`
3. toast успеха с числом уведомлённых учеников
4. redirect на `/results/{testId}?class_id={firstClassId}`

Idempotency:
- При сетевой ошибке используется повтор с тем же `request_id`.
- Повторный клик по кнопке блокируется на время запроса.

## 9. Доставка уведомлений

После успешного назначения:
1. Формируются события уведомлений для всех учеников выбранных классов.
2. Отправляется in-app/push.
3. При интеграции Telegram отправляется сообщение с deep-link.
4. Карточка задания показывает воронку: «Открыли / Начали / Завершили».

---

## 10. Мониторинг выполнения (Class Results)

URL:
- `/results/{testId}?class_id={classId}`

Если `class_id` не передан:
- Система показывает селектор класса.

Содержимое экрана:
- Summary bar: завершили, средний, лучший, худший, среднее время.
- Таблица учеников (`ResultsTable`) со статусами:
  - `not_started`
  - `in_progress`
  - `completed`
  - `overdue`
- `QuestionHeatmap` (правильно/неправильно/нет ответа).
- `WeakTopicsPanel` с CTA `Создать тест по теме`.

Правила таблицы:
- Сортировка по score по умолчанию: `desc`.
- Клиентская сортировка по заголовкам.
- Действие `Напомнить` для `not_started` (MVP: toast).

## 11. Edge-cases

| Ситуация | Поведение UI | Поведение системы |
|----------|--------------|-------------------|
| В классе нет учеников | Сообщение об ошибке выбора класса | Назначение блокируется |
| Дедлайн в прошлом | Инлайн-ошибка поля | Submit недоступен |
| Потеря сети при публикации | Toast с retry | Идемпотентный повтор |
| Нет `class_id` в результатах | Экран выбора класса | Запрос результатов после выбора |
| Никто не завершил тест | Empty state в таблице/heatmap | Флоу не падает |
| Аналитика недоступна | Error-banner на Dashboard | Сохранение доступности остальных блоков |

## 13. Нефункциональные требования

- Все данные API обрабатываются через envelope:
  - `{ ok: true, data: T }`
  - `{ ok: false, error: { code, message } }`
- Loading состояния обязательны для экранов и мутаций.
- Empty states обязательны для списков.
- Инпуты блокируются во время submit.
- Доступность AssignModal: Tab navigation, `Enter=submit`, `Esc=close`.

---

## 14. Критерии успешности

Продуктовые:
- P50 time-to-assign ≤ 7 минут.
- Конверсия `create_started -> assigned` ≥ 65%.
- Доля `assign_failed` < 1%.
- ≥ 80% учителей завершают сценарий без обращения в поддержку.

Технические:
- Повторный клик `Опубликовать` не создаёт дубль.
- Class Results сортируется по score desc по умолчанию.
- Heatmap работает на мобильном (`375px`) через горизонтальный скролл.
- Inline-валидации видны до submit (на blur).

---

## 15. Область реализации (MVP)

MVP включает:
- Teacher Dashboard.
- Test Builder (wizard, draft, publish).
- Test Library (мои тесты + Marketplace mock).
- Class Results (summary + таблица + heatmap + weak topics).
- Переиспользуемый AssignModal.

Вне MVP / v2:
- Реальная обработка импорта из файла (`xlsx/csv/docx`).
- Полноценный backend Marketplace.
- Продвинутая отправка напоминаний и экспортов (Excel/PDF) без имитации.
