# Роль: Ученик (Student)

**Платформа:** Zedly — онлайн-тестирование и аналитика для школ Узбекистана  
**Версия документа:** 1.0  
**Статус:** Production-ready spec

---

## 1. Описание роли

Ученик — конечный потребитель образовательного контента на платформе Zedly. Это школьник общеобразовательной школы Узбекистана в возрасте 7–18 лет (1–11 класс), который проходит тесты, назначенные учителем, готовится к НТТ, накапливает портфолио и получает сертификаты.

Ученик — ключевой носитель вирусной механики платформы: карточка-результат (аналог Spotify Wrapped), которую ученик получает после каждого теста, шерится в Telegram и Instagram, привлекая новых пользователей органически.

**Язык интерфейса:** узбекский или русский (выбирается при первом входе, можно менять).  
**Основной канал входа:** ссылка-приглашение от учителя через Telegram Bot (не требует пароля — просто нажать кнопку).  
**Устройство:** преимущественно смартфон (Android 8+), реже десктоп.  
**Технический уровень:** минимальный — интерфейс должен работать без объяснений.

## 2. Цели

| # | Цель | Как Zedly помогает |
|---|------|--------------------|
| 1 | Пройти тест быстро и без технических проблем | Вход по выданному login/паролю или через приглашение от учителя, интерфейс в Telegram |
| 2 | Понять, где допустил ошибки | Разбор каждого вопроса после завершения теста: правильный ответ + объяснение |
| 3 | Подготовиться к НТТ | Симулятор в формате реального нацтеста: 60 вопросов, 90 минут, разбор ошибок |
| 4 | Видеть свой прогресс | История тестов, динамика баллов, слабые темы |
| 5 | Получить подтверждение знаний | Сертификаты с QR-кодом, проверяемые вузами и работодателями |
| 6 | Сравнить себя с одноклассниками | Рейтинг класса (опционально, если учитель включил) |

## 3. Боли

### 3.1 Боли до Zedly

| Боль | Конкретика | Цена |
|------|------------|------|
| Нет обратной связи после теста | Учитель раздаёт оценки, но не объясняет, где именно ошибка и почему | Ученик повторяет одну и ту же ошибку на следующем тесте |
| НТТ — чёрный ящик | Ученики не знают формата экзамена, количества вопросов, тайминга | Паника и потеря времени на экзамене из-за незнания формата |
| Нет истории успеваемости | Результаты в бумажном журнале, к которому ученик не имеет доступа | Невозможно отследить прогресс, слабые темы неизвестны |
| Неудобный формат тестов | Распечатанные листы, заполнение ручкой, ожидание проверки 2–3 дня | Потеря мотивации из-за долгого ожидания результата |
| Нет доказательства знаний | Аттестат отражает только итоговые оценки, не промежуточный рост | Вуз и работодатель не видят реального уровня подготовки |
| Нет доступа без интернета (регионы) | В отдалённых районах нестабильный интернет, тест недоступен | Ученики регионов в неравных условиях |

### 3.2 Чего Zedly НЕ делает для ученика

- Не объясняет новый материал (Zedly — инструмент проверки, не обучения)
- Не заменяет репетитора по НТТ — только симулирует формат и показывает пробелы
- Не гарантирует результат НТТ — только повышает готовность

### 4.1 Что ученик МОЖЕТ делать

```
ТЕСТЫ

✅ view_own_test_result           — видеть свой результат после завершения теста
✅ view_answer_breakdown          — видеть разбор каждого вопроса (правильный ответ + объяснение)
✅ retake_test                    — пройти тест повторно (только если учитель разрешил повтор)
✅ download_test_offline          — скачать тест для прохождения без интернета (если режим включён)
✅ sync_offline_results           — синхронизировать ответы при восстановлении интернета

ПОРТФОЛИО И СЕРТИФИКАТЫ
✅ view_own_portfolio             — история всех тестов с 1 по 11 класс
✅ view_own_progress_chart        — график динамики баллов по предметам
✅ view_own_weak_topics           — персональные слабые темы
✅ receive_certificate            — получить сертификат после прохождения теста/курса
✅ share_certificate              — поделиться сертификатом (QR-ссылка)
✅ view_ntt_readiness_score       — персональный прогноз готовности к НТТ

СОЦИАЛЬНЫЕ ФУНКЦИИ
✅ share_result_card              — поделиться карточкой-результатом в Telegram/Instagram
✅ view_class_leaderboard         — рейтинг класса (если учитель включил)
✅ view_own_rank_in_class         — своё место в рейтинге класса

ПРОФИЛЬ
✅ edit_own_profile               — имя, фото профиля, язык интерфейса
✅ connect_telegram               — привязать Telegram аккаунт
✅ switch_language                — узбекский / русский
```

### 4.2 Что ученик НЕ МОЖЕТ делать

```
❌ view_other_students_answers    — ответы других учеников (только свои)
❌ view_full_class_analytics      — агрегированная аналитика класса (только учитель)
❌ create_test                    — создавать тесты
❌ assign_test                    — назначать тесты кому-либо
❌ view_correct_answers_before_submission — правильные ответы до завершения теста
❌ edit_submitted_answers         — изменить ответы после сабмита
❌ access_data_of_other_schools   — любые данные других school_id (RLS на уровне БД)
❌ view_other_students_scores     — баллы одноклассников (кроме анонимного рейтинга, если учитель включил)
❌ delete_own_portfolio           — удалить историю тестов (портфолио накапливается, защита от манипуляций)
❌ access_teacher_dashboard       — панель аналитики учителя
```

### 4.3 Анонимный рейтинг класса

Если учитель включил опцию «Показать рейтинг класса»:
- Ученик видит своё место (например: «Ты #7 из 28»)
- Имена других учеников в рейтинге **скрыты** (только «Ученик 1», «Ученик 2»)
- Исключение: учитель может включить режим «Открытый рейтинг» — тогда имена видны всему классу

## 5. Ограничения

| Ограничение | Значение | Причина |
|-------------|----------|---------|
| Изоляция по school_id | Ученик видит только данные своей школы | Мультитенантность, RLS в PostgreSQL |
| Однократное прохождение теста | Нельзя пройти тест дважды, если учитель не разрешил повтор | Целостность результатов |
| Просмотр правильных ответов | Только после завершения теста (не во время) | Честность тестирования |
| Редактирование ответов | Невозможно после нажатия «Завершить тест» | Целостность результатов |
| Удаление портфолио | Портфолио нельзя удалить (только деактивировать аккаунт через директора) | Долгосрочная накопительная ценность |
| НТТ-симулятор (экзаменационный режим) | Нельзя вернуться к предыдущему вопросу, таймер не останавливается | Точная симуляция реального НТТ |
| Сертификаты | Выдаются только за тесты с минимальным порогом баллов (устанавливает учитель или Zedly) | Защита ценности сертификата |

## 6. Видимая аналитика

### 6.1 Личный дашборд ученика (главный экран)

**Виджет 1: Назначенные тесты**
- Список активных тестов с дедлайнами
- Для каждого: предмет, название, сколько вопросов, лимит времени, статус (не начат / в процессе / завершён)
- Срочные тесты (дедлайн < 24 часа) выделены красным

**Виджет 2: Мой прогресс**
- График среднего балла по последним 10 тестам (по каждому предмету отдельно)
- Динамика: «+12% по математике за последний месяц»
- Цветовая индикация: 🔴 < 50%, 🟡 50–70%, 🟢 > 70%

**Виджет 3: Мои слабые темы**
- Топ-3 темы с наибольшим % ошибок за последние 30 дней
- Пример: «Квадратные уравнения — 65% ошибок в 4 тестах»
- Кнопка: «Найти тест по этой теме» → поиск в назначенных тестах или НТТ-симуляторе

**Виджет 4: НТТ Readiness Score** (только для 9–11 класса)
- Круговой индикатор: процент готовности к НТТ (0–100%)
- Разбивка по разделам НТТ: «Математика 72%, Физика 48%, Узбекский язык 81%»
- Последний НТТ-симулятор: дата, балл, место в рейтинге класса

**Виджет 5: Мои достижения**
- Полученные сертификаты (последние 3)
- Количество пройденных тестов всего
- Серия: «Ты прошёл тест 7 недель подряд 🔥»

### 6.2 Экран результата теста (после завершения)

Открывается сразу после нажатия «Завершить»:

- **Карточка результата:** большой % балла, анимация (confetti при > 80%)
- **Сравнение:** «Выше среднего по классу на 8%» (анонимно)
- **Время прохождения:** «Ты завершил за 18 мин из 30 отведённых»
- **Разбор по вопросам:**
  - Каждый вопрос: ✅ правильно / ❌ неправильно
  - Для неправильных: что выбрал ученик, что было правильным, краткое объяснение
  - Группировка по темам: «3 ошибки в теме "Логарифмы"»
- **Кнопка «Поделиться»:** генерирует карточку-результат (шейрабельная в Telegram/Instagram)
- **Рекомендация:** «Поработай над темой "Логарифмы" — там 3 из 5 ошибок»

### 6.3 Портфолио (история 1–11 класс)

- Хронологический список всех пройденных тестов
- Фильтры: по предмету, классу, периоду, типу теста
- Для каждого теста: дата, предмет, балл, средний балл класса (анонимно), сертификат (если есть)
- Экспорт портфолио: PDF с QR-кодом для предъявления при поступлении в вуз
- Портфолио видит ученик, его родитель и (агрегированно) учитель

### 6.4 Что ученик НЕ видит

- Ответы других учеников
- Имена в рейтинге (если учитель не включил открытый рейтинг)
- Агрегированную аналитику класса/школы
- Данные других школ

## 7. Пользовательские флоу

### Flow 1: Первое прохождение теста (новый ученик, нет аккаунта)

```
Актор: Ученик, у которого ещё нет аккаунта Zedly
Предусловие: Учитель назначил тест и отправил ссылку через Telegram Bot

Шаг 1: Ученик получает сообщение от Zedly Bot в Telegram:
        «Учитель Азиза назначила тест "Алгебра: квадратные уравнения"
         Вопросов: 10 | Время: 20 минут | Дедлайн: 15 марта 23:59
         [Пройти тест]»

Шаг 2: Ученик нажимает «Пройти тест»
        → Открывается веб-страница теста (Telegram WebApp или браузер)
        → Если аккаунт ещё не активирован — вход выполняется по выданному login + OTP
        → При первом входе OTP обязательно меняется на постоянный пароль
        → После активации ученик автоматически связан с классом учителя

Шаг 3: Ученик видит стартовый экран теста:
        → Название, предмет, количество вопросов, лимит времени
        → Кнопка «Начать тест»
        → Предупреждение: «После начала таймер запустится. Тест нельзя приостановить»

Шаг 4: Ученик нажимает «Начать тест»
        → Таймер запускается (обратный отсчёт)
        → Отображается первый вопрос

Шаг 5: Ученик отвечает на вопросы
        → MCQ: тап по варианту → подсветка выбранного ответа → переход к следующему
        → Открытый ответ: поле ввода текста
        → Навигация: «Вопрос 3 из 10», кнопки «Назад» / «Далее»
        → Пропущенные вопросы помечаются: «2 вопроса без ответа»

Шаг 6: Ученик нажимает «Завершить тест»
        → Если есть пропущенные вопросы: «У вас 2 вопроса без ответа. Завершить?» [Да] [Вернуться]
        → Подтверждение → ответы отправляются на сервер

Шаг 7: Экран результата (< 2 секунды после сабмита):
        → Балл: «78%» с анимацией
        → «Выше среднего по классу на 5%»
        → Разбор вопросов
        → Карточка-результат (шейрабельная)

Постусловие:
✅ Аккаунт ученика создан и привязан к классу
✅ Результат сохранён в test_sessions
✅ Аналитика класса обновится в течение 15 минут
✅ Родитель получает уведомление (если подключён)
```

### Flow 2: НТТ-симулятор (полный экзаменационный режим)

```
Актор: Ученик 11 класса
Предусловие: Ученик авторизован в Zedly

Шаг 1: Ученик открывает раздел «НТТ-подготовка»
        → Видит список симуляторов по предметам
        → Каждый симулятор: название, год, вариант, количество вопросов, средний балл сдавших

Шаг 2: Ученик выбирает «Математика НТТ 2025, вариант 4»
        → Видит подробное описание:
          - 60 вопросов
          - 90 минут строго (таймер не останавливается)
          - Нельзя вернуться к предыдущему вопросу (экзаменационный режим)
          - Разбор ошибок — после завершения
        → Кнопка «Начать симуляцию»

Шаг 3: Предстартовый экран:
        → «Убедитесь, что у вас есть 90 минут без отвлечений»
        → «Режим: Экзаменационный (как на реальном НТТ)»
        → Кнопка «Я готов — начать»

Шаг 4: Симуляция начинается
        → Таймер: 90:00 → отсчёт вниз
        → Вопрос 1 из 60
        → Нельзя вернуться назад (кнопка «Назад» заблокирована)
        → Нельзя выйти без завершения (предупреждение при попытке уйти)
        → При 10 минутах до конца: «⏰ Осталось 10 минут»
        → При 0:00: автоматическое завершение с текущими ответами

Шаг 5: Экран результата НТТ-симулятора:
        → Итоговый балл: «47 из 60 (78%)»
        → Разбивка по разделам: «Алгебра: 18/20 ✅ | Геометрия: 12/20 ⚠️ | Статистика: 17/20 ✅»
        → Сравнение с предыдущей попыткой: «+6 баллов с прошлого раза»
        → Топ-3 слабые темы с рекомендацией
        → NTT Readiness Score обновляется

Шаг 6: Детальный разбор
        → Каждый вопрос: что ответил / что правильно / объяснение
        → Фильтр: «Показать только ошибки»
        → Кнопка «Создать тренировочный тест по слабым темам»
        → Тест по слабым темам генерируется автоматически (10–15 вопросов)

Постусловие:
✅ Результат симулятора сохранён в портфолио
✅ NTT Readiness Score пересчитан
✅ История попыток доступна для сравнения
✅ Слабые темы обновлены в персональной аналитике
```

### Flow 3: Оффлайн-прохождение теста (регион без интернета)

```
Актор: Ученик в районе с нестабильным интернетом
Предусловие: Учитель включил offline-режим для теста; ученик ранее открыл тест при наличии интернета

Шаг 1: Ученик открывает тест при наличии интернета
        → Service Worker автоматически кэширует тест локально
        → Уведомление: «Тест доступен офлайн ✅»

Шаг 2: Интернет пропадает
        → Браузер показывает офлайн-индикатор (полоса вверху: «Нет соединения — офлайн режим»)
        → Тест продолжает работать нормально

Шаг 3: Ученик проходит тест полностью
        → Ответы сохраняются в localStorage (Service Worker)
        → Таймер продолжает работать локально (JavaScript)

Шаг 4: Ученик нажимает «Завершить тест»
        → Система определяет: нет соединения
        → Сообщение: «Ответы сохранены локально. Результат отправится автоматически при подключении к интернету»
        → Статус: «Ожидает синхронизации»

Шаг 5: Интернет восстанавливается
        → Service Worker автоматически отправляет кэшированные ответы
        → Сервер получает ответы, рассчитывает балл
        → Ученик получает push-уведомление в Telegram: «Ваш тест "Алгебра" засчитан: 82%»

Шаг 6: Если дедлайн истёк пока не было интернета:
        → Ответы принимаются с меткой «late_submission»
        → Учитель видит в таблице: балл + значок ⏰ «Сдан офлайн после дедлайна»
        → Учитель сам решает, засчитывать или нет (кнопка «Принять» / «Отклонить»)

Постусловие:
✅ Результат засчитан несмотря на отсутствие интернета во время теста
✅ Портфолио обновлено
✅ Учитель видит пометку об офлайн-сдаче
```

### Flow 4: Получение и шеринг карточки-результата (вирусная механика)

```
Актор: Ученик, завершивший тест
Предусловие: Тест завершён, балл рассчитан

Шаг 1: Экран результата автоматически показывает карточку:
        → Карточка генерируется на лету (canvas/SVG)
        → Содержит: имя ученика, предмет, балл, дата, место в рейтинге класса (если открыт),
          логотип Zedly, QR-код на результат

Шаг 2: Ученик нажимает «Поделиться»
        → Варианты: Telegram, Instagram Stories, скачать PNG, скопировать ссылку

Шаг 3a (Telegram):
        → Открывается Telegram с предзаполненным сообщением:
          «Прошёл тест по математике на 84%! 🎯
           Готовлюсь к НТТ на Zedly»
        → Карточка прикреплена как изображение
        → Ученик выбирает чат / канал и отправляет

Шаг 3b (Instagram Stories):
        → Карточка адаптирована под формат 9:16
        → Открывается приложение Instagram с готовой историей

Шаг 4: Кто видит карточку:
        → Ссылка «zedly.uz/r/abc123» открывает публичную страницу результата
        → Страница показывает: имя, предмет, балл, дата
        → Кнопка «Попробовать Zedly» → заявка на подключение/активацию в школе

Постусловие:
✅ Карточка опубликована
✅ Каждый переход по ссылке отслеживается (utm_source=share_card)
✅ Новые подключения через карточки атрибутируются к вирусному каналу
✅ Вирусный коэффициент: ~2–3 новых пользователя на 10 поделившихся
```

### Flow 5: Просмотр и экспорт портфолио

```
Актор: Ученик (или выпускник)
Предусловие: Ученик прошёл минимум 1 тест в Zedly

Шаг 1: Ученик открывает раздел «Моё портфолио»
        → Хронологическая лента всех тестов (от первого до последнего)
        → Фильтры: предмет, класс (год обучения), тип теста, период

Шаг 2: Просмотр конкретного теста из истории:
        → Дата, предмет, название теста, балл
        → Средний балл класса в тот момент (анонимно)
        → Разбор ошибок (сохраняется навсегда)
        → Сертификат (если был выдан) → кнопка «Показать QR»

Шаг 3: Ученик нажимает «Экспортировать портфолио»
        → Генерируется PDF (background job, готов через 30–60 секунд)
        → PDF содержит:
          - ФИО ученика, школа, период
          - Таблица: дата / предмет / тест / балл
          - График прогресса по предметам
          - Список сертификатов с QR-кодами
          - Подпись: «Данные верифицированы платформой Zedly»

Шаг 4: Ученик скачивает PDF или получает ссылку
        → Ссылка действительна 30 дней
        → QR-коды в PDF ведут на страницу верификации zedly.uz/verify/{cert_id}

Шаг 5: Вуз или работодатель сканирует QR:
        → Открывается страница: «Сертификат верифицирован ✅»
        → Показывает: имя, предмет, балл, дата, школа (без лишних персональных данных)

Постусловие:
✅ Портфолио экспортировано и готово к предъявлению
✅ QR-верификация работает без логина
✅ Данные защищены: третья сторона видит только верифицированные факты
```


## 8. Граничные случаи (Edge Cases)

### 8.1 Технические граничные случаи

| Ситуация | Поведение системы | Что видит ученик |
|----------|-------------------|-----------------|
| Закрыл браузер в середине теста | Сессия активна 24 часа; ответы на пройденные вопросы сохранены | При открытии ссылки снова: «Продолжить с вопроса 7» |
| Таймер истёк, ученик ещё отвечал | Автоматический сабмит текущих ответов; пропущенные = неправильные | «Время вышло. Ваши ответы засчитаны» |
| Интернет пропал в середине теста (offline НЕ включён) | Ответы на прошедшие вопросы сохранены в памяти браузера; при восстановлении — продолжить | «Нет соединения. Не закрывайте страницу — тест сохранится» |
| Ученик пытается пройти тест дважды (повтор запрещён) | 409 Conflict; второй запуск заблокирован | «Вы уже прошли этот тест. Результат: 74%» |
| Два устройства одновременно | Новая сессия на втором устройстве блокируется | «Тест уже открыт на другом устройстве» |
| НТТ-симулятор: попытка вернуться назад | Кнопка «Назад» заблокирована в экзаменационном режиме | Кнопка неактивна, tooltip: «В экзаменационном режиме нельзя вернуться» |
| Изображение к вопросу не загрузилось | Placeholder с текстом «Изображение недоступно»; вопрос можно ответить | «Изображение временно недоступно» |
| PDF-портфолио генерируется долго (> 30 сек) | Background job; уведомление в Telegram когда готово | «Портфолио готовится... Мы уведомим вас в Telegram» |

### 8.2 Бизнес-граничные случаи

| Ситуация | Поведение системы |
|----------|-------------------|
| Ученик переходит в другую школу | Новый профиль в новой школе (новый school_id). Портфолио из старой школы: ученик сохраняет доступ к своим историческим данным через личный аккаунт, но они не видны новой школе |
| Школа закрылась / лицензия истекла | Ученики сохраняют доступ к своему портфолио и сертификатам в read-only режиме. Новые тесты назначить нельзя |
| Ученик меняет Telegram аккаунт | Старый аккаунт деактивируется как источник входа; нужно привязать новый в настройках. Данные и портфолио сохраняются |
| Учитель удалил тест после прохождения | Результаты теста остаются в портфолио ученика. Название теста: «[Тест удалён учителем]». Балл сохранён |
| Ученик жалуется на неправильный вопрос в тесте | Форма «Сообщить об ошибке» на экране разбора → идёт учителю → учитель может исправить вопрос → результаты пересчитываются |
| Ученик не прошёл ни одного теста (новый) | Пустое портфолио с онбординговым сообщением: «Как только учитель назначит тест — он появится здесь» |
| Дедлайн теста изменён учителем после начала | Если дедлайн продлён — ученики, начавшие тест, получают больше времени. Если сокращён — предупреждение ученикам в Telegram за 1 час |

### 8.3 Граничные случаи offline-режима

| Ситуация | Поведение |
|----------|-----------|
| Тест скачан, но учитель изменил вопросы пока не было интернета | При синхронизации система сравнивает версию теста. Если версия изменилась — ученику предлагается пройти обновлённую версию |
| Офлайн-ответы пришли после того, как учитель закрыл тест вручную | Сервер принимает ответы с меткой «late_submission»; учитель получает уведомление и решает сам |
| Service Worker не поддерживается (старый браузер) | Офлайн-режим недоступен. Предупреждение при открытии: «Для офлайн-режима обновите браузер» |
| Конфликт: ученик успел ответить на том же устройстве онлайн и офлайн | Timestamp wins: принимается более поздняя версия ответа с пометкой в логе |

### 8.4 Граничные случаи НТТ-симулятора

| Ситуация | Поведение |
|----------|-----------|
| Ученик запустил симулятор, но вынужден остановиться (форс-мажор) | Сессия сохраняется 2 часа; при возвращении: «Продолжить с вопроса 34, осталось 38 мин» (только в тренировочном режиме). В экзаменационном режиме — сессия автозавершается через 90 мин с момента старта |
| Ученик прошёл все доступные варианты симулятора | Показывает: «Все варианты пройдены. Новые будут добавлены к [дата]». Кнопка: «Пройти повторно» |
| Балл НТТ-симулятора резко упал (подозрение на читерство) | Система не блокирует, но фиксирует аномалию в аналитике учителя; учитель решает |

## 9. Acceptance Criteria

### Сценарий 1: Первый вход по ссылке-приглашению

```gherkin
Given ученик получил ссылку zedly.uz/join/XYZ123 от учителя через Telegram Bot
  And у ученика нет аккаунта Zedly

When ученик нажимает «Войти через Telegram»

Then аккаунт создаётся автоматически на основе данных Telegram
  And ученик добавляется в класс учителя (school_id совпадает)
  And ученик видит список назначенных тестов
  And весь процесс занимает не более 30 секунд
  And НЕ требуется вводить пароль или проходить верификацию email
```

### Сценарий 2: Сабмит теста и получение результата

```gherkin
Given ученик прошёл тест (ответил на все вопросы)

When ученик нажимает «Завершить тест» и подтверждает

Then ответы сохраняются в таблице answers с session_id и timestamp
  And результат рассчитывается немедленно (для MCQ — автоматически, < 2 секунд)
  And ученик видит экран результата с баллом
  And карточка-результат доступна для шеринга
  And родитель получает Telegram-уведомление в течение 60 секунд (если подключён)
  And analytics_snapshots для класса обновится в течение 15 минут
  And повторный сабмит того же теста заблокирован (409 Conflict)
```

### Сценарий 3: НТТ-симулятор — автозавершение по таймеру

```gherkin
Given ученик запустил НТТ-симулятор в экзаменационном режиме
  And ответил на 45 из 60 вопросов

When таймер достигает 0:00

Then система автоматически завершает сессию
  And засчитываются ответы на 45 вопросов, 15 без ответа = 0 баллов
  And сессия получает статус «completed» (НЕ «expired» или «timeout»)
  And ученик видит результат: «47 из 60» с разбором ошибок
  And NTT Readiness Score пересчитывается
  And попытка сохраняется в портфолио
```

### Сценарий 4: Офлайн-синхронизация

```gherkin
Given ученик скачал тест при наличии интернета (offline-режим включён учителем)
  And интернет пропал

When ученик полностью проходит тест офлайн и нажимает «Завершить»

Then ответы сохраняются локально (Service Worker / localStorage)
  And ученик видит: «Ответы сохранены. Отправятся при подключении»

When интернет восстанавливается (в любое время)

Then ответы автоматически отправляются на сервер без действий ученика
  And ученик получает Telegram-уведомление: «Тест "Алгебра" засчитан: 82%»
  And результат появляется в портфолио
  And учитель видит результат с меткой «offline sync»
```

### Сценарий 5: Шеринг карточки-результата

```gherkin
Given ученик завершил тест с баллом > 0%

When ученик нажимает «Поделиться результатом»

Then система генерирует карточку-изображение (PNG, 1080x1080px) за < 3 секунды
  And карточка содержит: имя ученика, предмет, балл, дату, логотип Zedly, QR-код
  And ученик может поделиться в Telegram, Instagram или скачать PNG
  And публичная ссылка zedly.uz/r/{uuid} работает без авторизации
  And страница по ссылке показывает верифицированный результат
  And содержит кнопку «Попробовать Zedly» (с utm_source=share_card)
```

### Сценарий 6: Верификация сертификата третьей стороной

```gherkin
Given ученик получил сертификат за прохождение курса с баллом >= порога

When вуз или работодатель сканирует QR-код на сертификате

Then открывается страница zedly.uz/verify/{cert_id} без авторизации
  And страница показывает: имя ученика, предмет/курс, балл, дата, статус «Верифицировано ✅»
  And страница НЕ показывает: паспортные данные, школу, другие тесты
  And время загрузки страницы < 1 секунды
  And QR работает через 5 лет после выдачи сертификата (без срока годности)
```

### Сценарий 7: Изоляция данных — ученик не видит чужое

```gherkin
Given ученик школы A авторизован
  And JWT содержит school_id = «school_A», student_id = «student_123»

When ученик любым способом запрашивает результаты другого ученика (student_456)
  Or запрашивает данные школы B

Then система возвращает 403 Forbidden
  And ответ НЕ содержит никаких данных другого ученика или школы
  And попытка записывается в audit_log
```

### Сценарий 8: Попытка повторного прохождения заблокированного теста

```gherkin
Given ученик уже прошёл тест (статус сессии: «completed»)
  And учитель НЕ включил режим «Разрешить повтор»

When ученик открывает ссылку на тот же тест

Then тест НЕ открывается для прохождения
  And ученик видит: «Вы уже прошли этот тест. Ваш результат: 74%»
  And кнопка «Смотреть разбор» — активна (ученик может пересмотреть свои ответы)
  And новая test_session НЕ создаётся
```

## 10. Бизнес-обоснование

### Почему ученик — носитель вирусной механики

Ученик не платит за платформу, но именно он создаёт вирусный рост:

1. **Карточка-результат (Spotify Wrapped-эффект).** Ученик получает красивую, шейрабельную карточку после каждого теста. Психология: «Я в топ-10% класса» → хочется поделиться. Каждые 10 поделившихся приводят 2–3 новых пользователя из других школ. Их учителя получают вопрос от учеников: «Почему у нас нет Zedly?»

2. **Родительский вирус.** После сабмита теста родитель получает Telegram-уведомление: *«Ваш ребёнок в топ-10% класса по математике!»* Родители пересылают друзьям, другие родители спрашивают директора: *«Почему у нас этого нет?»* → давление на школу снизу.

3. **НТТ как точка входа.** НТТ — главная боль каждого 9–11 классника и их семьи. Бесплатный НТТ-симулятор привлекает учеников даже без учителя. Ученик приходит сам, потом учитель видит активность и подключается.

4. **Портфолио как lock-in.** После года использования у ученика накоплено портфолио: 40+ тестов, 5+ сертификатов. Уйти = потерять историю. Это удерживает ученика, а через него — и школу.

### Экономика ученика на платформе

| Метрика | Целевое значение |
|---------|-----------------|
| Время до первого результата (Time-to-Value) | ≤ 5 минут (вход по ссылке → результат теста) |
| Тестов в месяц (активный ученик) | ≥ 6 (1.5 в неделю) |
| % учеников, поделившихся карточкой-результатом | ≥ 30% |
| Retention D30 (ученики, вернувшиеся через месяц) | ≥ 50% |
| НТТ-симулятор: попыток на ученика (9–11 класс) | ≥ 5 до НТТ |
| Конверсия «шеринг карточки → новый пользователь» | ≥ 20% переходов по ссылке |

### Ученик и госконтракт

При презентации РОНО и Министерству просвещения именно данные учеников — главный аргумент:
- Улучшение среднего балла по НТТ-симулятору у классов, использующих Zedly
- Снижение процента провальных результатов (< 50%) на реальном НТТ
- Рост вовлечённости: % учеников, завершающих назначенные тесты
