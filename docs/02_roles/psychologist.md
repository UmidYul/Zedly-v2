# Роль: Школьный психолог (Psychologist)

**Платформа:** Zedly — онлайн-тестирование и аналитика для школ Узбекистана  
**Версия документа:** 1.0  
**Статус:** Production-ready spec

---

## 1. Описание роли

Psychologist — школьная роль внутри конкретного `school_id`, отвечающая за психологические и профориентационные тестирования учеников.

Ключевой принцип: психолог работает **только в своём tenant (школе)** и **не имеет доступа к академическим оценкам**.

---

## 2. Scope доступа

- Область данных: только `school_id` из JWT.
- Допустимые типы тестов:
  - `psychological`
  - `career_guidance`
- Недопустимые данные:
  - `journal_entries` с академическими оценками;
  - школьная академическая аналитика по предметам;
  - финансовые/подписочные операции школы.

---

## 3. Разрешения (RBAC)

### 3.1 Разрешено

- `create_psychological_test`
- `edit_own_psychological_test`
- `assign_psychological_test_to_class`
- `view_psychological_results`
- `view_career_guidance_report`
- `export_psychological_pdf_report`

### 3.2 Запрещено

- `view_academic_grades`
- `view_journal_entries`
- `manage_school_users`
- `manage_school_structure`
- `manage_subscriptions_global`

---

## 4. Бизнес-логика

1. Психолог создаёт тест типа `psychological` или `career_guidance`.
2. Назначает тест классу/группе внутри своей школы.
3. После завершения получает профиль и рекомендации (rules + AI enrichment).
4. Формирует PDF-отчёт для ученика/родителя/классного руководителя.
5. Не может открыть экраны с академической успеваемостью.

---

## 5. Доступ к результатам

Результаты профориентации и психологических тестов доступны:
- Psychologist,
- Director,
- Class Teacher (классный руководитель),
- Parent (только по своему ребёнку),
- Student (собственный отчёт, если это разрешено политикой школы/плана).

---

## 6. Ограничения безопасности

- Все запросы проходят `enforce_school_scope()`.
- Доступ к тестовым результатам проходит `enforce_role(permission)` + `enforce_feature(feature_code)`.
- Экспорт PDF логируется в `audit_logs` с `request_id`, `school_id`, `user_id`.

---

## 7. KPI роли

- % учеников, прошедших профориентацию за период.
- Распределение карьерных профилей по параллелям.
- Доля учеников с низкой уверенностью профиля (`confidence_score < threshold`) для повторной диагностики.

---

## 8. Edge cases

- Психолог пытается открыть академический журнал → `403 FORBIDDEN`.
- Психолог запрашивает данные другой школы → `404`/`403` (по политике сокрытия).
- Истёкшая подписка ученика на `CAREER_GUIDANCE` → отчёт только в базовом формате или блок функционала (по feature-gate).
