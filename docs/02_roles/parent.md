# Роль: Родитель (Parent)

**Платформа:** Zedly — онлайн-тестирование и аналитика для школ Узбекистана  
**Версия документа:** 1.0  
**Статус:** Production-ready spec

---

## 1. Описание роли

Родитель — пассивный наблюдатель за успеваемостью своего ребёнка в Zedly. Он не создаёт тесты, не управляет классами, но получает уведомления о результатах ребёнка и видит его прогресс.

Родитель — ключевой вирусный агент второго уровня: получив уведомление *«Ваш ребёнок в топ-10% класса по математике»*, он пересылает его друзьям и знакомым. Другие родители спрашивают учителей и директоров: *«Почему у нас нет Zedly?»* — это создаёт органическое давление снизу и ускоряет продажи школьных лицензий.

**Язык интерфейса:** узбекский или русский.  
**Основной канал:** Telegram-уведомления (90% взаимодействия). Веб-интерфейс — для детального просмотра.  
**Устройство:** смартфон.  
**Технический уровень:** минимальный. Родитель не должен ничего настраивать — достаточно подписаться через бота.

## 2. Цели

| # | Цель | Как Zedly помогает |
|---|------|--------------------|
| 1 | Знать успеваемость ребёнка без родительских собраний | Telegram-уведомления после каждого теста |
| 2 | Видеть динамику, а не только последнюю оценку | График прогресса ребёнка по предметам |
| 3 | Понять, по каким темам ребёнок отстаёт | Персональные слабые темы ребёнка |
| 4 | Знать уровень готовности к НТТ | NTT Readiness Score ребёнка |
| 5 | Реагировать на проблемы своевременно | Уведомление при резком падении балла |

## 3. Боли

| Боль | Конкретика | Цена |
|------|------------|------|
| Узнаёт об успеваемости раз в квартал | Родительское собрание — единственный источник информации | Проблемы накапливаются 3 месяца |
| Ребёнок скрывает плохие оценки | «Нормально, всё хорошо» — а в журнале двойки | Родитель реагирует поздно |
| Нет понимания динамики | Оценка «4» — это прогресс или регресс? Непонятно без истории | Невозможно помочь ребёнку целенаправленно |
| НТТ — тревога без данных | Родитель не знает, насколько ребёнок готов к нацтесту | Стресс и необоснованные траты на репетиторов |

## 4. Разрешения (Permissions)

> Примечание по RBAC: названия прав в этом документе допускают человекочитаемые алиасы. Канонический словарь permission-key определяется в `docs/09_permissions/permissions_matrix.md`.

### 4.1 Что родитель МОЖЕТ делать

```
ДАННЫЕ РЕБЁНКА
✅ view_child_results              — результаты всех тестов ребёнка
✅ view_child_progress_chart       — график динамики баллов по предметам
✅ view_child_weak_topics          — персональные слабые темы ребёнка
✅ view_child_ntt_readiness        — NTT Readiness Score ребёнка (9–11 класс)
✅ view_child_certificates         — сертификаты ребёнка

УВЕДОМЛЕНИЯ
✅ receive_test_result_notification — уведомление после каждого теста
✅ receive_top_rank_notification    — уведомление «ребёнок в топ-X% класса»
✅ receive_low_score_alert         — уведомление при балле ниже 50%
✅ receive_ntt_progress_update     — еженедельный дайджест о НТТ-готовности
✅ configure_notification_settings — настроить какие уведомления получать

ПРОФИЛЬ
✅ link_child_account             — привязать аккаунт ребёнка к своему профилю
✅ link_multiple_children         — привязать нескольких детей (если > 1 ребёнок в Zedly)
```

### 4.2 Что родитель НЕ МОЖЕТ делать

```
❌ view_other_children_data       — данные других учеников
❌ view_class_aggregate_analytics — агрегированная аналитика класса
❌ view_child_answers_in_detail   — конкретные ответы на конкретные вопросы (только учитель)
❌ create_or_assign_tests         — создание/назначение тестов
❌ contact_other_parents          — платформа не является мессенджером
❌ access_data_of_other_schools   — изоляция по school_id (RLS)
❌ change_child_profile           — данные ученика редактирует только сам ученик или учитель
```

### 4.3 Привязка к ребёнку

Родитель привязывается к аккаунту ребёнка одним из способов:
- **Код приглашения:** учитель генерирует код для родителей класса → родитель вводит код в боте
- **QR-код:** ребёнок показывает родителю QR из своего профиля → родитель сканирует
- **Ссылка от учителя:** учитель отправляет родительскую ссылку при онбординге класса

Один родительский аккаунт может быть привязан к нескольким детям (братья/сёстры в одной или разных школах).

## 5. Видимая аналитика

### 5.1 Что приходит в Telegram

**Уведомление 1: Результат теста**
```
📊 Zedly | Школа №47

Алишер прошёл тест:
«Алгебра: квадратные уравнения»

Результат: 84% ✅
Место в классе: топ 15%
Время: 18 из 20 минут

[Посмотреть детали]
```

**Уведомление 2: Топ рейтинга (вирусное)**
```
🏆 Zedly | Школа №47

Алишер в топ-10% класса
по математике за этот месяц!

Средний балл за месяц: 81%
Прогресс: +9% за последние 4 теста

[Поделиться достижением]  [Посмотреть прогресс]
```

**Уведомление 3: Низкий балл (алерт)**
```
⚠️ Zedly | Школа №47

Алишер прошёл тест по физике
с результатом 38%.

Рекомендуем обратить внимание
на тему: «Законы Ньютона».

[Посмотреть детали]
```

**Уведомление 4: Еженедельный НТТ-дайджест**
```
📈 Zedly | НТТ-готовность

Алишер (11А класс)
Готовность к НТТ: 74% 📈 (+3% за неделю)

Сильные стороны: Математика 81%, Узбекский 78%
Требует внимания: Физика 54%

[Полный отчёт]
```

### 5.2 Веб-интерфейс родителя

Открывается по кнопке «Посмотреть детали» из уведомления:

- **Вкладка «Прогресс»:** график среднего балла по предметам за последние 3 месяца
- **Вкладка «Тесты»:** хронологический список тестов с баллами (без ответов — только итог)
- **Вкладка «Слабые темы»:** топ-5 тем с наибольшим % ошибок
- **Вкладка «НТТ»:** NTT Readiness Score с разбивкой по разделам (только 9–11 класс)
- **Вкладка «Сертификаты»:** список полученных сертификатов с QR

### 5.3 Что родитель НЕ видит

- Конкретные вопросы и ответы теста
- Имена и результаты других учеников
- Аналитику класса / школы / района

## 6. Пользовательские флоу

### Flow 1: Подключение к аккаунту ребёнка

```
Актор: Родитель, чей ребёнок уже в Zedly
Предусловие: Ребёнок зарегистрирован, учитель отправил родительские коды

Шаг 1: Родитель получает сообщение от учителя в родительском чате:
        «Для подключения к Zedly-отчётам вашего ребёнка:
         1. Напишите боту @ZedlyBot
         2. Введите код: PARENT-4F9K2»

Шаг 2: Родитель открывает @ZedlyBot → отправляет /start
        → Бот: «Привет! Введите код для подключения к ребёнку»

Шаг 3: Родитель вводит код PARENT-4F9K2
        → Бот: «Подтверждение: вы хотите подключиться к профилю Алишера Каримова, 9А класс?»
        → Кнопки: [Да, это мой ребёнок] [Нет, другой код]

Шаг 4: Родитель подтверждает
        → Привязка выполнена
        → Бот: «Готово! Теперь вы будете получать уведомления об успехах Алишера»
        → Кнопка «Настроить уведомления»

Шаг 5: Родитель настраивает уведомления:
        ✅ Каждый тест (вкл/выкл)
        ✅ Только низкий балл (< 50%)
        ✅ Топ рейтинга
        ✅ Еженедельный дайджест НТТ

Постусловие:
✅ Родитель привязан к аккаунту ребёнка
✅ Первое уведомление придёт после следующего теста
✅ Процесс занял < 2 минут
```

---

### Flow 2: Получение вирусного уведомления и шеринг

```
Актор: Родитель
Предусловие: Ребёнок вошёл в топ-10% класса

Шаг 1: Родитель получает уведомление в Telegram:
        «🏆 Алишер в топ-10% класса по математике!»

Шаг 2: Родитель нажимает «Поделиться достижением»
        → Генерируется карточка: имя, предмет, достижение, логотип Zedly

Шаг 3: Родитель пересылает в родительский чат школы / семейный чат
        → Другие родители видят достижение

Шаг 4: Другие родители, чьи дети не на Zedly, спрашивают:
        «Что это? Как подключиться?»
        → Ссылка на карточке ведёт на страницу входа/подключения

Постусловие:
✅ Вирусный охват: 1 уведомление → 1 шеринг → 2–5 новых потенциальных пользователей
✅ Каждый переход атрибутируется (utm_source=parent_share)
```

---

### Flow 3: Реакция на тревожный алерт

```
Актор: Родитель
Предусловие: Ребёнок получил балл < 50% на тесте

Шаг 1: Родитель получает алерт:
        «⚠️ Алишер прошёл тест по физике с результатом 38%»

Шаг 2: Родитель нажимает «Посмотреть детали»
        → Открывается страница: предмет, дата, балл, слабые темы теста
        → «3 из 5 ошибок — в теме "Законы Ньютона"»

Шаг 3: Родитель разговаривает с ребёнком о результате
        (платформа не даёт советов по воспитанию — только данные)

Шаг 4 (опционально): Родитель может написать учителю через обычный мессенджер
        (Zedly не является каналом коммуникации между родителем и учителем — MVP)

Постусловие:
✅ Родитель осведомлён о проблеме своевременно (не через 3 месяца на собрании)
✅ Конкретная тема проблемы известна
✅ Родитель может принять меры (дополнительные занятия, репетитор)
```

## 7. Граничные случаи (Edge Cases)

| Ситуация | Поведение системы | Что видит родитель |
|----------|-------------------|-------------------|
| Ребёнок заблокировал уведомления Zedly в Telegram | Уведомления перестают доходить. Система фиксирует «Bot blocked» | Ничего (уведомления не доходят) |
| Родитель привязал чужого ребёнка по ошибке (неправильный код) | Возможность отвязаться в настройках бота: /unlink | «Отвязать ребёнка» в меню бота |
| У ребёнка нет результатов ещё (новый ученик) | Пустой профиль с сообщением: «Как только учитель назначит тест, здесь появятся результаты» | Пустой экран с объяснением |
| Родитель хочет видеть ответы ребёнка на каждый вопрос | Система не показывает (только итог и слабые темы). Это защита: ребёнок сам разбирает с учителем | «Детальный разбор доступен ученику в его профиле» |
| Два родителя одного ребёнка (мать и отец) | Оба могут быть привязаны к одному профилю ребёнка — независимо | Каждый получает уведомления отдельно |
| Ребёнок переходит в другую школу | Родительская привязка сохраняется. После перехода ребёнка в новую школу — данные новой школы начинают поступать автоматически | Без изменений в настройках |
| Родитель хочет отключить все уведомления | /settings в боте → «Отключить все уведомления» | Уведомления прекращаются немедленно |
| Уведомление о «топ-10%» приходит слишком часто (ребёнок всегда в топе) | Алгоритм: не чаще 1 раза в 7 дней отправляет «топ»-уведомление | Максимум 1 позитивное уведомление в неделю |

## 8. Acceptance Criteria

### Сценарий 1: Уведомление после теста

```gherkin
Given родитель привязан к аккаунту ребёнка
  And ребёнок завершил тест

When результат рассчитан и сохранён в test_sessions

Then родитель получает Telegram-уведомление в течение 60 секунд
  And уведомление содержит: имя ребёнка, название теста, балл, место в классе
  And уведомление содержит кнопку «Посмотреть детали»
  And переход по кнопке открывает страницу без авторизации (токен в ссылке, TTL 7 дней)
```

### Сценарий 2: Вирусный шеринг

```gherkin
Given ребёнок вошёл в топ-10% класса по итогам месяца

When система рассчитывает ежемесячный рейтинг

Then родитель получает «топ»-уведомление не чаще 1 раза в 7 дней
  And уведомление содержит кнопку «Поделиться достижением»
  And нажатие генерирует карточку-изображение за < 3 секунды
  And карточка содержит utm_source=parent_share для атрибуции
```

### Сценарий 3: Изоляция данных

```gherkin
Given родитель привязан к ребёнку A

When родитель запрашивает данные ребёнка B (любым способом)

Then система возвращает 403 Forbidden
  And никакие данные ребёнка B не передаются
  And попытка записывается в audit_log
```

## 9. Бизнес-обоснование

Родитель не платит, но создаёт два ключевых эффекта:

1. **Вирусный рост.** «Топ-10%»-уведомление шерится в родительских чатах. Каждые 10 пересылок приводят 2–3 новых запроса в другие школы. Это органический канал без затрат на маркетинг.

2. **Давление на директора.** Когда родители нескольких классов знают о Zedly, но школа не подключена — они давят на директора: *«Почему у нас нет этой системы?»* Это ускоряет принятие решения о лицензии.

| Метрика | Целевое значение |
|---------|-----------------|
| % родителей, подключённых к платформе (от числа учеников) | ≥ 40% |
| % родителей, поделившихся «топ»-уведомлением | ≥ 25% |
| Конверсия «шеринг родителя → новое подключение» | ≥ 15% |
