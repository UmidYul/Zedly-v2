# Роль: Учитель (Teacher)

**Платформа:** Zedly — онлайн-тестирование и аналитика для школ Узбекистана  
**Версия документа:** 1.0  
**Статус:** Production-ready spec

---

## 1. Описание роли

Учитель — основной создатель контента на платформе Zedly и ключевой вирусный агент роста. Это педагог общеобразовательной школы Узбекистана (государственной или частной), ведущий один или несколько предметов в одном или нескольких классах.

Учитель — первый пользователь, который приходит на платформу (через Telegram или реферальную ссылку), и именно его активность запускает цепочку: 5–7 активных учителей в школе → директор сам приходит за школьной лицензией.

**Язык интерфейса:** узбекский или русский (переключение в настройках профиля).  
**Основной канал входа:** Telegram Login Widget (учителя уже в Telegram — нулевое переобучение).  
**Устройство:** преимущественно смартфон (Android), реже десктоп.

## 2. Цели

| # | Цель | Как Zedly помогает |
|---|------|--------------------|
| 1 | Быстро создавать качественные тесты | AI-генерация из учебника за 2–3 минуты вместо 1–2 часов ручного набора |
| 2 | Видеть реальную картину знаний класса | Аналитический дашборд: средний балл, топ ошибок, прогресс по времени |
| 3 | Готовить учеников к НТТ | НТТ-симулятор в формате реального экзамена |
| 4 | Тратить минимум времени на организацию тестирования | Назначение теста одной кнопкой → ученики получают ссылку в Telegram |
| 5 | Иметь доказательную базу своей работы | История тестов, результаты классов, публичный рейтинг «Учитель месяца» |
| 6 | Использовать готовые тесты коллег | Marketplace тестов, отфильтрованный по предмету и классу |

## 3. Боли

### 3.1 Боли до Zedly

| Боль | Конкретика | Цена |
|------|------------|------|
| Ручная проверка тестов | Учитель проверяет 30 работ × 10 вопросов = 300 ответов вручную | 3–5 часов/неделю на один предмет |
| Создание тестов с нуля | Набор вопросов в Word, распечатка, раздача — каждый раз | 1–2 часа на один тест |
| Нет аналитики | «Класс плохо сдал контрольную» — но непонятно, какие именно темы проблемные | Учитель идёт по программе вслепую |
| Нет НТТ-инструментов | Учителя готовят к НТТ по распечаткам прошлых лет, без симуляции реального теста | Ученики не знают формата, паникуют на экзамене |
| Хаос с результатами | Оценки в бумажном журнале, в Excel, в телефоне — нет единой истории | При проверке РОНО нечего показать |
| Нет связи с родителями | Родители узнают об успеваемости только на родительском собрании (раз в квартал) | Проблемы выявляются поздно |

### 3.2 Боли, которые Zedly НЕ решает (важно для честности документа)

- Zedly не заменяет живой урок и объяснение материала
- Zedly не решает проблему слабой мотивации учеников — только делает прогресс видимым
- Zedly не автоматически проверяет развёрнутые письменные ответы (только MCQ и короткие ответы)

### 4.2 Что учитель НЕ МОЖЕТ делать

```
❌ view_other_class_results       — результаты классов других учителей своей школы
❌ view_school_aggregate_analytics — общая аналитика школы (только директор, платный план)
❌ view_students_of_other_teachers — список учеников других классов
❌ access_data_of_other_schools   — любые данные других school_id (RLS на уровне БД)
❌ delete_marketplace_test        — удалять тесты других учителей из marketplace
❌ edit_other_teacher_test        — редактировать тесты других учителей
❌ manage_school_license          — управление лицензией (только директор)
❌ access_roono_panel             — панель РОНО
❌ view_audit_logs                — журнал безопасности
```

### 4.3 Freemium vs Платный план

| Функция | Freemium (бесплатно) | Платный (школьная лицензия) |
|---------|---------------------|----------------------------|
| Количество учеников | до 30 на класс | без ограничений |
| Количество тестов | до 10 активных | без ограничений |
| AI-генерация | 5 тестов/месяц | без ограничений |
| Аналитика | базовая (средний балл, топ ошибок) | расширенная (тренды, прогнозы, сравнение с прошлым периодом) |
| Экспорт результатов | только CSV | CSV + Excel + PDF |
| Marketplace | просмотр и копирование | + публикация собственных тестов |
| НТТ-симулятор | полный доступ | полный доступ |
| Telegram Bot | полный доступ | полный доступ |
| Offline-режим | полный доступ | полный доступ |

> **Триггер перехода на платный план:** когда в школе 5–7 учителей активны на freemium, директору автоматически показывается уведомление: *«92% ваших учителей уже на Zedly — время для школьной лицензии»*. Учитель сам не покупает лицензию — это задача директора.

## 5. Ограничения

| Ограничение | Значение | Причина |
|-------------|----------|---------|
| Изоляция по school_id | Учитель видит и создаёт данные только внутри своей школы | Мультитенантность, RLS в PostgreSQL |
| Редактирование теста | Нельзя редактировать тест после того, как хотя бы один ученик начал его проходить | Целостность данных |
| Удаление теста | Нельзя удалить тест с завершёнными сессиями | Сохранность истории учеников |
| Открытые ответы | Тест с открытыми ответами не получает финального балла до ручной проверки учителем | Точность оценки |
| Изображения к вопросам | Максимум 5 MB, форматы: jpg, png, webp | Оптимизация трафика для регионов |
| Freemium лимит | До 30 учеников на класс, до 10 активных тестов | Монетизационная модель |
| Marketplace публикация | Тест публикуется только после модерации командой Zedly | Контроль качества контента |

## 6. Видимая аналитика

### 6.1 Дашборд учителя (главный экран после входа)

**Виджет 1: Обзор классов**
- Список классов учителя (например: 9А Математика, 10Б Физика)
- Для каждого класса: средний балл последнего теста, дата проведения, % учеников завершивших тест
- Индикатор: 🔴 если средний балл < 50%, 🟡 50–70%, 🟢 > 70%

**Виджет 2: Слабые темы класса**
- Топ-5 тем с наибольшим % ошибок по последним 3 тестам
- Пример: «Квадратные уравнения — 67% ошибок», «Тригонометрия — 54% ошибок»
- Кнопка: «Создать тест по этой теме» → конструктор с предзаполненной темой

**Виджет 3: Прогресс класса по времени**
- Sparkline-график: средний балл класса по последним 10 тестам
- Сравнение с предыдущим месяцем: «+8% по сравнению с прошлым месяцем»

**Виджет 4: Активность учеников**
- Сколько учеников прошли последний тест (например: 24/30)
- Кто ещё не прошёл → кнопка «Напомнить в Telegram»

**Виджет 5: НТТ Readiness (если учитель ведёт выпускные классы)**
- Средний балл класса по НТТ-симулятору
- Топ-3 слабых раздела НТТ

### 6.2 Детальная аналитика теста

Открывается после выбора конкретного теста:

- **Распределение баллов:** гистограмма (сколько учеников в диапазонах 0–20%, 20–40%, 40–60%, 60–80%, 80–100%)
- **Вопрос-за-вопросом:** по каждому вопросу — % правильных ответов, какой неправильный вариант выбирали чаще всего
- **Таблица учеников:** имя, балл, время прохождения, статус (завершён / не начат / в процессе)
- **Сравнение с предыдущим тестом по той же теме:** стало лучше или хуже

### 6.3 Что учитель НЕ видит

- Агрегированную аналитику всей школы (только директор)
- Результаты учеников других классов
- Аналитику других учителей
- Данные любых других школ

## 7. Пользовательские флоу

### Flow 1: Создание теста и назначение классу (основной сценарий)

```
Актор: Учитель
Предусловие: Учитель авторизован, класс добавлен, есть минимум 1 ученик

Шаг 1: Учитель нажимает «Создать тест» на дашборде
        → Открывается конструктор тестов

Шаг 2: Учитель вводит название, выбирает предмет, класс, тип теста
        (обычный / НТТ-режим / челлендж), устанавливает лимит времени

Шаг 3a (ручной режим): Учитель добавляет вопросы вручную
        → Выбирает тип вопроса (MCQ / открытый / соответствие)
        → Вводит текст вопроса, варианты ответов, правильный ответ
        → Опционально прикрепляет изображение (drag & drop, max 5MB)

Шаг 3b (AI-режим): Учитель нажимает «Сгенерировать с AI»
        → Загружает текст параграфа или PDF главы учебника
        → AI предлагает 10–15 вопросов за 15–30 секунд
        → Учитель принимает / отклоняет / редактирует каждый вопрос
        → Сохраняет выбранные вопросы в тест

Шаг 3c (из marketplace): Учитель находит готовый тест в marketplace
        → Нажимает «Скопировать в мою библиотеку»
        → Редактирует при необходимости

Шаг 4: Учитель нажимает «Назначить классу»
        → Выбирает класс (или несколько классов)
        → Устанавливает дедлайн (опционально)
        → Устанавливает дату начала (опционально: сразу или запланировать)

Шаг 5: Система генерирует уникальную ссылку на тест
        → Учитель нажимает «Отправить в Telegram»
        → Telegram Bot отправляет ученикам ссылку с кнопкой «Пройти тест»

Шаг 6: По мере прохождения учитель видит в реальном времени:
        → Прогресс-бар: «18 из 30 учеников завершили»
        → WebSocket обновление без перезагрузки страницы

Шаг 7: После дедлайна или ручного закрытия теста:
        → Аналитика теста доступна в разделе «Результаты»
        → Если есть открытые ответы — статус «Требует проверки»

Постусловие:
✅ Тест сохранён с school_id учителя
✅ Ученики получили уведомление в Telegram
✅ Результаты доступны в аналитике
✅ analytics_snapshots обновятся в течение 15 минут после последнего завершения
```

### Flow 2: НТТ-подготовка класса

```
Актор: Учитель выпускного класса (9, 10, 11 класс)
Предусловие: Учитель авторизован, класс добавлен

Шаг 1: Учитель открывает раздел «НТТ-подготовка»
        → Видит список доступных НТТ-симуляторов по предмету
        → (Симуляторы созданы командой Zedly и верифицированы)

Шаг 2: Учитель выбирает симулятор (например: «Математика НТТ 2025, вариант 3»)
        → Просматривает состав: 60 вопросов, 90 минут, разделы по темам

Шаг 3: Учитель назначает симулятор классу
        → Выбирает режим: «Экзаменационный» (строгий таймер, нельзя вернуться)
          или «Тренировочный» (можно возвращаться к вопросам)
        → Устанавливает дату (например: «Воскресенье в 10:00»)

Шаг 4: Ученики проходят симулятор в назначенное время
        → Таймер 90 минут, автозавершение
        → После завершения каждый ученик видит свой балл и разбор ошибок

Шаг 5: Учитель открывает результаты НТТ-симулятора:
        → Средний балл класса по каждому разделу НТТ
        → Сравнение с предыдущим симулятором (прогресс)
        → Топ слабых тем → кнопка «Создать тренировочный тест по этой теме»

Шаг 6: Учитель создаёт целевые тренировочные тесты по слабым темам
        → Цикл повторяется до НТТ

Постусловие:
✅ История НТТ-симуляторов хранится для каждого ученика
✅ NTT Readiness Score рассчитывается автоматически
✅ Учитель видит динамику подготовки класса
```

### Flow 3: Назначение теста через Telegram Bot (mobile-first сценарий)

```
Актор: Учитель в мобильном Telegram
Предусловие: Telegram аккаунт привязан к профилю Zedly

Шаг 1: Учитель пишет боту /assign или нажимает кнопку «Назначить тест»
        → Бот показывает список последних тестов учителя (3–5 штук)

Шаг 2: Учитель выбирает тест из списка
        → Бот спрашивает: «Какому классу назначить?»
        → Учитель выбирает класс из своего списка

Шаг 3: Бот спрашивает: «Дедлайн? (нажми "Без дедлайна" или введи дату)»
        → Учитель вводит «15 марта 23:59» или нажимает «Без дедлайна»

Шаг 4: Бот подтверждает: «Тест "Алгебра: квадратные уравнения" назначен 9А до 15 марта»
        → Ученики класса автоматически получают сообщение от бота с кнопкой «Пройти тест»

Шаг 5: Учитель получает уведомления по мере прохождения:
        → «18 из 30 учеников завершили тест»
        → «Дедлайн через 2 часа. Не прошли: Алишер, Дилноза, Санжар»

Шаг 6: После дедлайна:
        → Бот присылает краткую сводку: средний балл, % завершения
        → Ссылка на полную аналитику в веб-версии

Постусловие:
✅ Весь флоу выполнен в Telegram без открытия браузера
✅ Результаты синхронизированы с веб-платформой
✅ Учитель не тратит время на ручную рассылку ссылок
```

### Flow 4: Разбор ошибок после теста

```
Актор: Учитель
Предусловие: Тест завершён, есть результаты

Шаг 1: Учитель открывает раздел «Результаты» → выбирает завершённый тест
        → Видит общую статистику теста

Шаг 2: Учитель переходит в «Анализ вопросов»
        → Видит каждый вопрос: % правильных ответов, топ неправильных вариантов

Шаг 3: Учитель находит вопрос с % ошибок > 60%
        → Кликает «Почему ошибаются?»
        → Видит: какой вариант выбирали чаще (типичное заблуждение)

Шаг 4: Учитель нажимает «Создать тренировочный тест по этой теме»
        → Конструктор открывается с предзаполненной темой
        → AI предлагает похожие вопросы с усиленным акцентом на типичную ошибку

Шаг 5: Учитель публикует разбор теста для учеников
        → Ученики получают в Telegram: свой результат + правильные ответы + объяснения

Постусловие:
✅ Слабые темы зафиксированы в аналитике класса
✅ Создан follow-up тест для доработки
✅ Ученики получили обратную связь
```

### Flow 5: Публикация теста в Marketplace

```
Актор: Учитель (только платный план или школьная лицензия)
Предусловие: Тест создан и использован минимум 1 раз в своём классе

Шаг 1: Учитель открывает свою библиотеку тестов
        → Выбирает тест → нажимает «Опубликовать в Marketplace»

Шаг 2: Заполняет метаданные:
        → Предмет (из стандартного списка узбекской программы)
        → Класс (1–11)
        → Тема (из дерева тем учебной программы)
        → Краткое описание (необязательно)
        → Язык теста (узбекский / русский)

Шаг 3: Тест уходит на модерацию команде Zedly
        → Статус: «На проверке»
        → SLA модерации: 48 часов

Шаг 4: После одобрения:
        → Тест появляется в marketplace
        → Учитель получает уведомление в Telegram: «Ваш тест опубликован»
        → Учитель видит счётчик: сколько раз тест скопировали другие учителя

Шаг 5 (отклонение): Если модератор отклонил:
        → Учитель получает причину отклонения с конкретными замечаниями
        → Может исправить и подать повторно

Постусловие:
✅ Тест доступен всем учителям платформы (без привязки к школе)
✅ Автор сохраняет авторство (имя учителя видно, школа — нет)
✅ Сетевой эффект: чем больше учителей публикуют, тем ценнее marketplace
```

## 8. Граничные случаи (Edge Cases)

### 8.1 Технические граничные случаи

| Ситуация | Поведение системы | Что видит учитель |
|----------|-------------------|-------------------|
| Ученик начал тест, но закрыл браузер | Сессия остаётся активной 24 часа, ответы сохраняются; после 24 часов — автозавершение с текущими ответами | Статус ученика: «В процессе (прервал)» |
| Ученик пытается пройти тест дважды (если повтор запрещён) | 409 Conflict, тест не открывается | Ученик видит: «Вы уже прошли этот тест. Ваш результат: X%» |
| Учитель редактирует тест после того, как ученик начал проходить | Изменения запрещены пока есть активные сессии (статус теста: locked) | Сообщение: «Нельзя редактировать: 5 учеников сейчас проходят тест» |
| Нет интернета у ученика (offline-режим) | Ответы кэшируются в Service Worker; синхронизируются при восстановлении | Учитель видит статус: «Ожидает синхронизации» |
| Дедлайн истёк, но ученик ещё проходит тест (начал до дедлайна) | Ученик может завершить тест; его результат принимается с пометкой «сдан в срок» | Нормальный статус; дедлайн применяется к моменту начала, не завершения |
| Ученик начал тест после дедлайна | Тест не открывается | Ученик видит: «Дедлайн истёк». Учитель может вручную продлить дедлайн для конкретного ученика |
| AI-генерация вернула некорректный вопрос (фактическая ошибка) | Вопрос предлагается с флагом «Требует проверки» | Учитель видит флаг и должен вручную подтвердить или отредактировать |
| Загруженный PDF для AI — не учебник, а посторонний документ | AI генерирует вопросы по тексту документа; предупреждение не выводится | Учитель видит сгенерированные вопросы и сам решает, использовать или нет |

### 8.2 Бизнес-граничные случаи

| Ситуация | Поведение системы |
|----------|-------------------|
| Учитель уволился / покинул школу | Его тесты остаются в системе школы; назначаются директором другому учителю. Личный аккаунт деактивируется, данные не удаляются |
| Школа отменила лицензию → downgrade до freemium | Тесты сохраняются. Если учеников > 30 — классы переходят в «только чтение» до возобновления. Уведомление учителю за 7 дней до даунгрейда |
| Учитель ведёт классы в двух разных школах | Два отдельных профиля (два разных school_id). Нельзя использовать тесты из одной школы в другой — полная изоляция |
| Учитель пытается назначить тест классу другой школы | 403 Forbidden. Попытка записывается в audit_log |
| Freemium учитель добавляет 31-го ученика | Система блокирует добавление: «Лимит 30 учеников. Попросите директора школы активировать школьную лицензию» |
| Тест с открытыми ответами: учитель долго не проверяет (> 14 дней) | Автоматическое напоминание в Telegram на 7-й и 14-й день. На 30-й день — тест закрывается с пометкой «не проверен», итоговый балл не выставляется |

### 8.3 Граничные случаи Telegram-интеграции

| Ситуация | Поведение |
|----------|-----------|
| Ученик не использует Telegram | Учитель получает альтернативную ссылку для отправки вручную (email, мессенджер) |
| Telegram Bot заблокирован учеником | Уведомления не доходят; учитель видит статус «Bot blocked» у ученика в списке класса |
| Учитель случайно отключил Telegram-аккаунт от Zedly | Функции назначения через бота отключаются. Напоминание в веб-интерфейсе: «Подключите Telegram для полного функционала» |
| Telegram не работает (технический сбой) | Тест доступен по прямой ссылке в браузере; функционал не теряется |

## 9. Acceptance Criteria

### Сценарий 1: Создание и публикация теста

```gherkin
Given учитель авторизован в системе Zedly
  And у учителя есть хотя бы один класс с минимум 1 учеником

When учитель создаёт тест с минимум 3 вопросами
  And нажимает «Назначить классу»
  And выбирает класс и подтверждает

Then тест сохраняется в БД с school_id учителя
  And статус теста меняется на «active»
  And ученики класса получают уведомление в Telegram в течение 60 секунд
  And тест НЕ виден учителям и ученикам других школ (проверка через RLS)
  And в таблице test_sessions создаётся запись для каждого ученика со статусом «not_started»
```

### Сценарий 2: AI-генерация вопросов

```gherkin
Given учитель нажал «Сгенерировать с AI»
  And загрузил текст параграфа (минимум 200 символов)

When система обрабатывает запрос

Then в течение 30 секунд учитель видит минимум 5 предложенных вопросов
  And каждый вопрос имеет: текст, 4 варианта ответа, помеченный правильный ответ, тему
  And вопросы с низкой уверенностью AI помечены флагом «Требует проверки»
  And учитель может принять, отклонить или отредактировать каждый вопрос индивидуально
  And принятые вопросы добавляются в тест
```

### Сценарий 3: Просмотр аналитики после теста

```gherkin
Given тест завершён (все ученики прошли или дедлайн истёк)

When учитель открывает раздел «Аналитика» этого теста

Then учитель видит средний балл класса
  And видит топ-5 вопросов с наибольшим % ошибок
  And видит для каждого вопроса: какой неправильный вариант выбирали чаще всего
  And видит таблицу учеников с индивидуальными баллами
  And аналитика загружается менее чем за 1 секунду (читается из analytics_snapshots)
  And данные учеников других классов НЕ отображаются
```

### Сценарий 4: Напоминание ученикам через Telegram

```gherkin
Given тест назначен классу
  And 5 учеников из 30 ещё не прошли тест
  And до дедлайна осталось менее 2 часов

When учитель нажимает «Напомнить не прошедшим»

Then только 5 непрошедших учеников получают сообщение от Telegram Bot
  And сообщение содержит название теста, кнопку «Пройти» и дедлайн
  And учителю показывается подтверждение: «Напоминание отправлено 5 ученикам»
  And уже прошедшие тест ученики уведомление НЕ получают
```

### Сценарий 5: Freemium-ограничение

```gherkin
Given учитель на freemium плане
  And в его классе уже 30 учеников

When учитель пытается добавить 31-го ученика

Then система блокирует добавление
  And показывает сообщение: «Достигнут лимит freemium (30 учеников). Попросите директора активировать школьную лицензию»
  And 31-й ученик НЕ добавляется в класс
  And в системе фиксируется событие «freemium_limit_hit» для последующего триггера директору
```

### Сценарий 6: Попытка доступа к чужой школе

```gherkin
Given учитель школы A авторизован
  And JWT содержит school_id = «school_A»

When учитель делает GET /api/v1/tests?school_id=school_B (любым способом)

Then система возвращает 403 Forbidden
  And тело ответа НЕ содержит информацию о существовании ресурсов школы B
  And попытка записывается в таблицу audit_logs с полями: user_id, school_id, target_school_id, endpoint, timestamp
  And учитель не получает никаких данных школы B
```

### Сценарий 7: Offline-прохождение теста учеником

```gherkin
Given учитель назначил тест с включённым offline-режимом
  And ученик загрузил тест при наличии интернета

When ученик проходит тест без интернета
  And при подключении интернета нажимает «Отправить результаты»

Then ответы синхронизируются с сервером в течение 10 секунд
  And результат появляется в аналитике учителя
  And если дедлайн ещё не истёк — результат принимается как обычный
  And если дедлайн истёк — результат принимается с пометкой «late_submission» (учитель видит это в таблице)
```

## 10. Бизнес-обоснование

### Почему учитель — главный вирусный агент Zedly

Учитель — это точка входа в школу, а не директор. Логика вирусного роста:

1. **Freemium снижает барьер до нуля.** Учитель регистрируется бесплатно, без согласования с директором. Нет бюджета → нет сделки → нет проволочек.

2. **Aha-moment за 15 минут.** Учитель создаёт первый тест с помощью AI за 2–3 минуты, назначает классу через Telegram Bot, видит результаты. За 15 минут он видит ценность. Это ключевой метрик онбординга.

3. **Органическое давление снизу.** Когда 5–7 учителей активно используют Zedly, директор видит это сам — он замечает, что коллеги обсуждают платформу, ученики проходят тесты. Zedly автоматически отправляет директору триггер-уведомление: *«92% ваших учителей уже на Zedly — перейдите на школьную лицензию»*.

4. **Социальное доказательство внутри школы.** Публичный рейтинг «Учитель месяца» (по вовлечённости учеников) создаёт конкуренцию между педагогами — коллеги видят рейтинг и регистрируются.

5. **Вирус за пределы школы.** Карточка-результат (Spotify Wrapped) шерится учениками в Telegram и Instagram → другие школьники видят → их учителя получают вопросы «почему у нас нет Zedly?».

### Экономика учителя на платформе

| Метрика | Целевое значение |
|---------|-----------------|
| Время до первого теста (Time-to-Value) | ≤ 15 минут после регистрации |
| Тестов в месяц (активный учитель) | ≥ 4 (1 в неделю) |
| % учителей, перешедших на Telegram Bot | ≥ 70% в первые 2 недели |
| Retention D30 (учителя, вернувшиеся через месяц) | ≥ 60% |
| NPS учителей | ≥ 50 |
| Конверсия «учитель → школьная лицензия» | 5–7 активных учителей = 1 лицензия директора |

### Роль учителя в госконтракте

При пилоте с РОНО именно учитель является исполнителем на месте. РОНО оценивает: снизилась ли нагрузка на учителей (в часах), улучшились ли результаты учеников. Поэтому UX учителя — это одновременно продуктовый KPI и государственный KPI.
