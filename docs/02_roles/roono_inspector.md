# Роль: РОНО-инспектор (District Education Inspector)

**Платформа:** Zedly — онлайн-тестирование и аналитика для школ Узбекистана  
**Версия документа:** 1.0  
**Статус:** Production-ready spec

---

## 1. Описание роли

РОНО (Районный отдел народного образования) — инспектор районного управления образования, который курирует все школы своего района. В Узбекистане один РОНО охватывает в среднем 30–80 школ.

РОНО-инспектор — ключевой партнёр Zedly на пути к государственным контрактам. Именно через РОНО реализуется стратегия «сверху вниз»: РОНО подписывает пилотный договор на 5–10 школ → кейс с цифрами → масштабирование на область → Минпрос.

**Язык интерфейса:** узбекский или русский.  
**Основной канал входа:** email (корпоративный адрес РОНО) или ссылка из пилотного договора.  
**Устройство:** преимущественно десктоп (рабочее место в РОНО).  
**Технический уровень:** базовый — инспектор работает с Excel и email, Zedly должен быть проще.

## 2. Цели

| # | Цель | Как Zedly помогает |
|---|------|--------------------|
| 1 | Видеть реальную картину успеваемости по всем школам района | Агрегированный дашборд: все школы района в одном окне |
| 2 | Формировать квартальные отчёты без ручного сбора | Автоматическая генерация районного отчёта за 1 клик |
| 3 | Выявлять проблемные школы до проверки | Рейтинг школ по среднему баллу, динамика, красные флажки |
| 4 | Сравнивать школы по предметам | Матрица: школа × предмет с цветовой индикацией |
| 5 | Отчитываться перед областью / Министерством | Экспорт агрегированных данных в стандартных форматах |
| 6 | Контролировать охват цифровизации | Сколько школ района подключено, сколько учителей активны |

## 3. Боли

| Боль | Конкретика | Цена |
|------|------------|------|
| Ручной сбор отчётности | РОНО объезжает 40–60 школ или собирает Excel-файлы по email | 40–60 часов в месяц только на сбор данных |
| Данные несопоставимы | Каждая школа сдаёт отчёт в своём формате — таблицы несовместимы | Ещё 10–15 часов на приведение к единому виду |
| Нет оперативной картины | Инспектор узнаёт о проблемах в школе при плановой проверке | Реагирует на кризис, а не предотвращает его |
| Нет доказательной базы | «Школа №12 хуже сдаёт» — но нет цифр для официальных выводов | Невозможно принять административное решение с обоснованием |
| Тяжело готовиться к проверкам области | Сводный отчёт по всем школам — многодневная работа | Инспектор работает сверхурочно перед каждой областной проверкой |

## 4. Разрешения (Permissions)

> Примечание по RBAC: названия прав в этом документе допускают человекочитаемые алиасы. Канонический словарь permission-key определяется в `docs/09_permissions/permissions_matrix.md`.

### 4.1 Что РОНО-инспектор МОЖЕТ делать

```
АНАЛИТИКА РАЙОНА
✅ view_district_aggregate_analytics  — агрегированная аналитика всех школ района
✅ view_school_rankings               — рейтинг школ района по среднему баллу
✅ view_school_details                — детальная аналитика конкретной школы (агрегат, не персональные данные)
✅ view_subject_performance_district  — успеваемость по предметам в разрезе всех школ
✅ view_ntt_readiness_district        — НТТ-готовность по всем школам района
✅ view_coverage_stats                — сколько школ подключено к Zedly, % охвата

ОТЧЁТЫ
✅ generate_district_report           — квартальный/годовой отчёт по всему району
✅ generate_school_report             — отчёт по конкретной школе
✅ schedule_auto_reports              — настроить автоматическую отправку отчётов
✅ export_district_data               — выгрузить данные района в Excel/CSV
✅ compare_schools                    — сравнительный анализ 2+ школ

УПРАВЛЕНИЕ ПИЛОТОМ
✅ invite_school_to_pilot             — пригласить школу подключиться к Zedly (в рамках пилота)
✅ view_pilot_progress                — прогресс по школам пилотной программы
```

### 4.2 Что РОНО-инспектор НЕ МОЖЕТ делать

```
❌ view_individual_student_data       — персональные данные конкретных учеников
❌ view_individual_teacher_tests      — конкретные тесты конкретных учителей
❌ access_schools_from_other_district — данные школ другого района
❌ manage_school_licenses             — управление лицензиями (это директор школы)
❌ create_or_assign_tests             — создание тестов
❌ edit_school_data                   — редактирование профилей школ
❌ view_national_analytics            — данные других районов / национальный уровень (только Министерство)
```

### 4.3 Уровни доступа к данным школы

РОНО видит по каждой школе:
- ✅ Средний балл по предметам (агрегат по школе)
- ✅ Динамика за квартал/год
- ✅ Количество активных учителей и учеников
- ✅ NTT Readiness Score (агрегат по выпускным классам)
- ✅ Процент охвата (сколько учеников прошло хотя бы один тест)
- ❌ Имена учеников и их индивидуальные результаты
- ❌ Конкретные тесты и вопросы

## 5. Видимая аналитика

### 5.1 Дашборд РОНО

**Виджет 1: Карта района**
- Список всех школ района с цветовой индикацией:
  - 🟢 Средний балл > 70%, активно используют Zedly
  - 🟡 Средний балл 50–70% или низкая активность
  - 🔴 Средний балл < 50% или не подключены к Zedly
- Клик на школу → детальная карточка школы

**Виджет 2: Рейтинг школ**
- Таблица: Место | Школа | Средний балл | Динамика | Охват учеников | Статус Zedly
- Сортировка по любому столбцу
- Фильтр: только подключённые к Zedly / все школы района

**Виджет 3: Успеваемость по предметам**
- Матрица: Предмет (строки) × Школа (столбцы)
- Ячейка = средний балл по предмету в школе
- Цветовая индикация: видно, в каких школах «провальные» предметы
- Фильтр по классам (5–9 классы, 10–11 классы)

**Виджет 4: Охват цифровизации**
- % школ района, подключённых к Zedly
- % учителей, создавших минимум 1 тест за месяц
- % учеников, прошедших минимум 1 тест за месяц
- Динамика: рост охвата за квартал

**Виджет 5: НТТ-готовность района**
- Средний NTT Readiness Score по всем выпускным классам района
- Топ-3 школы по НТТ-готовности
- Топ-3 школы с наименьшей НТТ-готовностью → требуют внимания

### 5.2 Квартальный отчёт района

Структура автоматически генерируемого отчёта:

1. **Общая сводка:** количество школ, учителей, учеников в системе, период
2. **Рейтинг школ:** топ-5 и антирейтинг-5 по среднему баллу
3. **Успеваемость по предметам:** сводная таблица по всему району
4. **Динамика:** сравнение с прошлым кварталом
5. **НТТ-готовность:** по каждой школе с выпускными классами
6. **Охват цифровизации:** % школ и учителей на платформе
7. **Рекомендации:** автоматически сформированные на основе данных
   - Пример: «Школа №23 показала снижение среднего балла по математике на 15%. Рекомендуется проверка»

Форматы: PDF (официальный бланк) + Excel (для передачи в область).

## 6. Пользовательские флоу

### Flow 1: Пилотное подключение района (ключевой сценарий роста)

```
Актор: РОНО-инспектор, ведущий переговоры с Zedly
Предусловие: Zedly провёл презентацию для РОНО, подписан пилотный договор на 3 месяца

Шаг 1: Zedly-менеджер создаёт аккаунт РОНО в системе:
        → district_id привязывается к списку school_id пилотных школ
        → РОНО-инспектор получает email с логином и паролем

Шаг 2: РОНО-инспектор входит в панель:
        → Видит список пилотных школ (5–10 школ)
        → Статус каждой: «Ожидает подключения» / «Активна»

Шаг 3: РОНО отправляет каждой пилотной школе официальное письмо/приказ:
        «Школа включена в пилотную программу Zedly. Директору необходимо зарегистрироваться по ссылке до [дата]»
        → Zedly генерирует уникальные ссылки-приглашения для каждого директора

Шаг 4: По мере подключения школ — РОНО видит прогресс:
        «Подключено 6 из 10 школ. Последняя активность: Школа №7 — сегодня в 09:14»

Шаг 5: Через 1 месяц РОНО формирует промежуточный отчёт пилота:
        → Данные 6 подключённых школ
        → Сравнение с 4 неподключёнными (если есть данные для сравнения)
        → Кейс для масштабирования: «охват вырос на X%, отчётность сократилась на Y часов»

Шаг 6: После 3 месяцев — финальный отчёт пилота
        → РОНО передаёт в Министерство просвещения области
        → Основание для районного/областного контракта

Постусловие:
✅ Пилот задокументирован с цифрами
✅ РОНО имеет доказательную базу для масштабирования
✅ Zedly получает кейс и референс для других районов
```

---

### Flow 2: Формирование квартального отчёта

```
Актор: РОНО-инспектор
Предусловие: Конец квартала, нужен отчёт для области

Шаг 1: РОНО открывает раздел «Отчёты»
        → Выбирает «Квартальный отчёт по району»
        → Выбирает период: I квартал 2025/2026

Шаг 2: Preview отчёта: какие школы включены, какие данные есть
        → Если школа не подключена к Zedly — она не входит в отчёт
        → Предупреждение: «4 школы не подключены к Zedly — их данных нет в отчёте»

Шаг 3: РОНО нажимает «Сформировать отчёт»
        → Background job, статус: «Формируется (~60 секунд для данных 30+ школ)»

Шаг 4: Отчёт готов
        → PDF: официальный бланк с шапкой РОНО, печать
        → Excel: структурированные данные для областной системы

Шаг 5: РОНО скачивает, подписывает ЭЦП и отправляет в область

Постусловие:
✅ Отчёт сформирован за 2 минуты вместо 40–60 часов
✅ Данные верифицированы Zedly (нет ручного ввода — нет ошибок)
✅ История отчётов хранится в системе 2 года
```

---

### Flow 3: Выявление проблемной школы

```
Актор: РОНО-инспектор
Предусловие: Система зафиксировала аномалию

Шаг 1: РОНО получает автоматический Telegram/email-алерт:
        «⚠️ Школа №23 (Алмазарский район): средний балл по математике
         упал на 18% за последние 30 дней. Требует внимания»

Шаг 2: РОНО открывает детальную карточку школы №23:
        → Динамика по предметам за 3 месяца
        → Активность учителей математики: «Последний тест: 3 недели назад»
        → NTT Readiness Score выпускных классов: снижение

Шаг 3: РОНО понимает причину: учитель математики не использовал платформу 3 недели
        → Нажимает «Скачать детальный отчёт по школе»

Шаг 4: РОНО связывается с директором школы №23 напрямую
        (вне платформы — телефон, email; Zedly не является коммуникационным каналом РОНО→директор в MVP)

Шаг 5: После разговора директор активирует учителей
        → РОНО видит в дашборде: «Активность школы №23 восстановлена»

Постусловие:
✅ Проблема выявлена проактивно, до плановой проверки
✅ Реакция заняла дни, а не месяцы
✅ Данные для официального разговора с директором — на руках
```

## 7. Граничные случаи (Edge Cases)

| Ситуация | Поведение системы |
|----------|-------------------|
| РОНО видит школу, которая не подключена к Zedly | В дашборде отображается как «Не в системе». Кнопка «Пригласить директора» |
| Директор школы сменился, данные рассинхронизированы | РОНО видит флаг «Смена директора, требует верификации». Школа остаётся в системе, данные сохранены |
| РОНО пытается открыть персональные данные ученика | 403 Forbidden. Сообщение: «Персональные данные учеников доступны только учителю и директору школы» |
| Отчёт включает данные школ с низкой активностью (< 5 тестов) | Данные включаются с пометкой «Низкая статистическая значимость». Предупреждение в отчёте |
| РОНО запрашивает данные школ другого района | 403 Forbidden. Доступ строго ограничен своим district_id |
| Конец учебного года — нужен годовой отчёт | Система поддерживает периоды: квартал, полугодие, учебный год. Годовой отчёт агрегирует 4 квартала |
| Инспектор увольняется, нужно передать доступ | Через поддержку Zedly: старый аккаунт деактивируется, новый создаётся с тем же district_id. Данные сохранены |

## 8. Acceptance Criteria

### Сценарий 1: Изоляция по district_id

```gherkin
Given РОНО-инспектор авторизован с district_id = «tashkent_almazar»

When инспектор открывает дашборд

Then видит только школы с district_id = «tashkent_almazar»
  And школы других районов недоступны (403 на любой запрос)
  And все запросы фильтруются по district_id на уровне PostgreSQL RLS
```

### Сценарий 2: Генерация районного отчёта

```gherkin
Given РОНО выбрал квартал и нажал «Сформировать отчёт»

When background job завершается

Then PDF-отчёт доступен в течение 90 секунд (для 30+ школ)
  And отчёт содержит данные только школ своего района
  And персональные данные учеников в отчёт НЕ включены
  And данные совпадают с тем, что РОНО видит в дашборде
```

### Сценарий 3: Автоматический алерт о проблемной школе

```gherkin
Given школа в районе показала падение среднего балла > 15% за 30 дней

When система пересчитывает analytics_snapshots

Then РОНО-инспектор получает Telegram/email-уведомление в течение 24 часов
  And уведомление содержит: название школы, предмет, % падения, рекомендацию
  And алерт НЕ содержит персональных данных учеников
```

## 9. Бизнес-обоснование

### РОНО — шлюз к государственным контрактам

Один договор с РОНО = доступ к 30–80 школам одновременно. Это в 30–80 раз эффективнее поштучной продажи директорам.

Стратегия: бесплатный 3-месячный пилот для РОНО → кейс с цифрами:
- «Время на квартальный отчёт: было 50 часов → стало 2 минуты»
- «Охват тестированием: вырос с 15% до 68% за 3 месяца»
- «Средний балл НТТ в пилотных школах: +8% по сравнению с непилотными»

Этот кейс РОНО передаёт в Министерство → областной контракт → национальный контракт.

| Метрика | Целевое значение |
|---------|-----------------|
| Конверсия «пилот РОНО → районный контракт» | ≥ 60% |
| Время формирования квартального отчёта (до/после) | 50 часов → 2 минуты |
| % школ района, подключённых после 3 месяцев пилота | ≥ 70% |
| Стоимость одного районного контракта | $5,000–$50,000/год |
