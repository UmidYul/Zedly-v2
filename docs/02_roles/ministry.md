# Роль: Министерство просвещения (Ministry of Education)

**Платформа:** Zedly — онлайн-тестирование и аналитика для школ Узбекистана  
**Версия документа:** 1.0  
**Статус:** Production-ready spec

---

## 1. Описание роли

Представитель Министерства просвещения Республики Узбекистан — пользователь с самым высоким уровнем агрегации данных на платформе. Это аналитик, руководитель департамента или советник, которому нужна национальная картина качества образования для принятия политических и управленческих решений.

Министерство — целевой клиент для крупнейшего государственного контракта Zedly ($50,000–$500,000+). Работа с Министерством начинается после успешных пилотов на уровне РОНО и области.

**Важно:** Министерство видит только агрегированные, анонимизированные данные. Никаких персональных данных учеников на национальном уровне.

**Язык интерфейса:** узбекский или русский (официальный).  
**Основной канал входа:** корпоративный email Министерства, выдаётся персонально.  
**Устройство:** десктоп (официальный рабочий компьютер).  
**Технический уровень:** базовый — пользователь работает с таблицами и презентациями, не с кодом.

## 2. Цели

| # | Цель | Как Zedly помогает |
|---|------|--------------------|
| 1 | Видеть национальную картину качества образования | Региональный дашборд: все области страны в одном окне |
| 2 | Выявлять системные проблемы (слабые предметы по стране) | Национальная карта слабых тем — какие темы не понимают школьники по всей стране |
| 3 | Оценивать эффективность образовательных реформ | Динамика: до/после изменения программы или учебника |
| 4 | Готовить данные для парламента и президентских отчётов | Экспорт в официальных форматах, пригодных для государственных презентаций |
| 5 | Сравнивать регионы между собой | Рейтинг областей по среднему баллу, динамика, аутсайдеры |
| 6 | Обосновывать необходимость методической поддержки школ | «Тема X вызывает затруднения в 78% школ» — конкретные данные для принятия решений |

## 3. Боли

| Боль | Конкретика | Цена |
|------|------------|------|
| Нет единой системы данных | Каждый регион сдаёт отчёт в своём формате — несопоставимые данные | Невозможно делать выводы на национальном уровне |
| Данные несвежие | Минпрос получает данные за прошлый год — решения принимаются на устаревшей информации | Реформы запаздывают на 1–2 года |
| Нет данных о конкретных темах | Известен общий балл, но непонятно, какие темы программы слабее всего усвоены | Нельзя адресно корректировать программу |
| Нет сравнения регионов | «Самаркандская область слабее Ташкентской» — но нет объективных данных | Политические решения без доказательной базы |
| Трудно оценить НТТ-готовность | Реальные результаты НТТ известны только после экзамена | Нет возможности заранее направить ресурсы в отстающие регионы |

## 4. Разрешения (Permissions)

### 4.1 Что Министерство МОЖЕТ делать

```
НАЦИОНАЛЬНАЯ АНАЛИТИКА
✅ view_national_dashboard            — агрегированная аналитика по всей стране
✅ view_regional_comparison           — сравнение регионов (областей) между собой
✅ view_district_comparison           — сравнение районов внутри области
✅ view_subject_performance_national  — успеваемость по предметам на национальном уровне
✅ view_weak_topics_national          — топ слабых тем по всей стране
✅ view_ntt_readiness_national        — НТТ-готовность по регионам
✅ view_trend_analysis                — многолетние тренды (год к году, квартал к кварталу)
✅ view_coverage_stats_national       — охват цифровизации: % школ и учителей на платформе

ОТЧЁТЫ
✅ generate_national_report           — сводный национальный отчёт (PDF/Excel)
✅ generate_regional_report           — отчёт по конкретному региону
✅ export_anonymized_data             — экспорт анонимизированных данных для внешнего анализа
✅ schedule_auto_reports              — автоматические отчёты по расписанию

СПЕЦИАЛЬНЫЕ ФУНКЦИИ
✅ view_reform_impact_analysis        — анализ влияния изменений программы на успеваемость
✅ access_predictive_analytics        — предиктивная аналитика (V2: прогноз результатов НТТ)
```

### 4.2 Что Министерство НЕ МОЖЕТ делать

```
❌ view_individual_school_details_below_district — данные конкретной школы (только агрегат по региону)
❌ view_individual_student_data        — персональные данные учеников
❌ view_individual_teacher_data        — данные конкретных учителей
❌ manage_school_licenses              — управление лицензиями
❌ create_or_assign_tests              — создание тестов
❌ access_raw_database                 — прямой доступ к базе данных
❌ modify_platform_settings            — изменение настроек платформы
```

### 4.3 Принцип анонимизации для национального уровня

На уровне Министерства все данные агрегированы до уровня **региона (области)**:
- Данные конкретных школ — недоступны
- Данные конкретных учителей — недоступны
- Данные учеников — недоступны полностью
- Минимальная единица агрегации: **район (РОНО)**

Исключение: отчёт по запросу Министерства на конкретный регион — доступен до уровня РОНО, но не ниже.

## 5. Видимая аналитика

### 5.1 Национальный дашборд

**Виджет 1: Карта Узбекистана**
- Тепловая карта регионов: цвет = средний балл по стране
- 🟢 Регион выше национального среднего
- 🟡 Регион на уровне среднего ±5%
- 🔴 Регион ниже среднего на >5%
- Клик на регион → детальная аналитика региона

**Виджет 2: Национальные KPI**
- Средний балл по стране за текущий квартал
- Динамика: «▲ +3.2% по сравнению с прошлым кварталом»
- Охват: «68% школ страны используют Zedly»
- Количество проведённых тестов за квартал: «1,240,000 тестов»

**Виджет 3: Топ слабых предметов по стране**
- Таблица: Предмет | Средний % ошибок | Количество школ с проблемой | Динамика
- Пример: «Физика — 58% ошибок | 2,340 школ | ▼ ухудшение»
- Кнопка «Детальный анализ» → темы внутри предмета

**Виджет 4: Топ слабых тем по стране**
- Темы учебной программы с наибольшим % ошибок в масштабе страны
- Пример: «Квадратные уравнения (математика, 9 класс) — 62% ошибок в 89% школ»
- Это напрямую показывает: где нужна методическая поддержка или пересмотр программы

**Виджет 5: НТТ-готовность по регионам**
- Таблица регионов по среднему NTT Readiness Score
- Цветовая индикация: красные регионы требуют приоритетного внимания
- Прогноз: «Если текущая динамика сохранится — средний балл НТТ по стране составит X%»

**Виджет 6: Рейтинг регионов**
- Полная таблица: Регион | Средний балл | Динамика за год | Охват Zedly | НТТ Readiness
- Экспорт в Excel одним кликом

### 5.2 Национальный отчёт (для парламента / президентских структур)

Структура официального отчёта:

1. **Введение:** период, охват данных, методология
2. **Общая картина:** средний балл по стране, динамика за год
3. **Региональный срез:** рейтинг всех 14 областей + г. Ташкент
4. **Анализ по предметам:** топ-5 слабых предметов по стране
5. **Анализ по темам:** 20 тем программы с наибольшим % ошибок
6. **НТТ-готовность:** прогноз по регионам
7. **Эффект цифровизации:** корреляция «охват Zedly → успеваемость»
8. **Рекомендации:** автоматически сформированные на основе данных
9. **Приложения:** таблицы с детальными данными по регионам

Форматы: PDF (официальный бланк Министерства) + Excel + PowerPoint (слайды для презентации).

### 5.3 Что Министерство НЕ видит

- Данные конкретных школ (только агрегат по РОНО и выше)
- Персональные данные учеников и учителей
- Финансовые данные (лицензии, платежи школ)

## 6. Пользовательские флоу

### Flow 1: Мониторинг НТТ-готовности накануне экзамена

```
Актор: Аналитик Министерства
Предусловие: НТТ через 2 месяца, нужна приоритизация поддержки

Шаг 1: Аналитик открывает виджет «НТТ-готовность по регионам»
        → Видит таблицу всех регионов с NTT Readiness Score
        → Три региона подсвечены красным: Сурхандарья (48%), Кашкадарья (51%), Джизак (53%)

Шаг 2: Аналитик кликает на Сурхандарьинскую область
        → Детальный срез: NTT Readiness по районам области
        → Топ слабых предметов в регионе: «Математика 44%, Физика 39%»

Шаг 3: Аналитик формирует рабочую записку:
        → Нажимает «Экспортировать анализ по региону»
        → PDF с данными по Сурхандарье для служебного использования

Шаг 4: На основе данных Министерство направляет методистов
        в три региона с фокусом на математику и физику

Шаг 5: Через месяц аналитик проверяет динамику:
        → Сурхандарья: NTT Readiness Score +6% за месяц
        → Методическое вмешательство подтверждается данными

Постусловие:
✅ Решение о методической поддержке принято на основе данных, а не интуиции
✅ Эффект вмешательства измерим и задокументирован
✅ Данные можно предъявить в парламентском отчёте
```

---

### Flow 2: Оценка влияния изменения учебной программы

```
Актор: Руководитель департамента Министерства
Предусловие: В 2024 году была изменена программа по математике 7 класса

Шаг 1: Руководитель открывает «Анализ трендов»
        → Выбирает: предмет «Математика», класс «7», период «2023–2026»

Шаг 2: График показывает:
        → 2023 (до реформы): средний балл 64%
        → 2024 (год изменения): средний балл 58% (падение — ожидаемо при переходе)
        → 2025: средний балл 61%
        → 2026 (текущий год): средний балл 67%

Шаг 3: Вывод: программа дала результат через 2 года после введения
        → Руководитель формирует отчёт «Эффект реформы программы математики 7 класса»

Шаг 4: Дополнительный анализ: топ слабых тем в 7 классе после реформы
        → «Тема "Функции" вызывает затруднения в 71% школ»
        → Рекомендация: усилить методическое обеспечение по теме «Функции»

Постусловие:
✅ Реформа оценена объективно с данными
✅ Выявлена конкретная точка улучшения
✅ Данные готовы для отчёта перед руководством
```

---

### Flow 3: Подготовка ежегодного национального доклада

```
Актор: Аналитик Министерства
Предусловие: Конец учебного года, нужен годовой национальный доклад

Шаг 1: Аналитик открывает раздел «Отчёты»
        → Выбирает «Национальный годовой отчёт»
        → Выбирает период: 2025/2026 учебный год

Шаг 2: Настройка отчёта:
        → Включить разделы: региональный рейтинг ✅, слабые темы ✅, НТТ-готовность ✅, охват ✅
        → Формат: PDF + Excel + PowerPoint
        → Язык: узбекский

Шаг 3: Нажимает «Сформировать»
        → Background job (для национальных данных: ~3–5 минут)
        → Уведомление на email: «Отчёт готов. Скачать: [ссылка]»

Шаг 4: Аналитик проверяет отчёт, добавляет комментарии (в Word/PDF-редакторе вне платформы)
        → Финальная версия передаётся руководству

Постусловие:
✅ Годовой доклад сформирован за 5 минут вместо недель ручного труда
✅ Данные из единого источника — нет расхождений между разделами
✅ Формат PowerPoint готов к презентации в парламенте без дополнительной работы
```

## 7. Граничные случаи (Edge Cases)

| Ситуация | Поведение системы |
|----------|-------------------|
| Министерство запрашивает данные конкретной школы | 403. Сообщение: «Данные на уровне школ доступны только директору и РОНО соответствующего района» |
| Регион недавно подключился к Zedly (мало данных) | Данные включаются с предупреждением: «Низкая статистическая значимость: < 30 дней данных, < 10% школ региона» |
| Национальный отчёт генерируется для миллионов записей | Асинхронная генерация (background job), уведомление на email. Отчёт готов за 3–10 минут |
| Аналитик Министерства пытается скачать персональные данные | Экспорт данных доступен только на уровне районов и выше. Персональные данные исключены на уровне запроса к БД |
| Данные одного региона аномально высокие (подозрение на вброс) | Система фиксирует статистические выбросы и помечает их в отчёте: «Данные требуют верификации» |
| Пользователь Министерства уходит в другой департамент | Доступ деактивируется через заявку в поддержку Zedly. Новый пользователь получает новый аккаунт |
| Министерство хочет данные по конкретному учебнику | Данные по учебникам не хранятся отдельно в MVP. В V2: маппинг тем тестов к учебникам |

## 8. Acceptance Criteria

### Сценарий 1: Изоляция на национальном уровне

```gherkin
Given пользователь Министерства авторизован

When открывает национальный дашборд

Then видит данные агрегированные до уровня регионов
  And данные конкретных школ недоступны (минимум агрегации — РОНО)
  And персональные данные учеников отсутствуют полностью
  And любой запрос на более детальные данные возвращает 403
```

### Сценарий 2: Генерация национального отчёта

```gherkin
Given пользователь выбрал годовой период и нажал «Сформировать»

When background job завершается

Then PDF-отчёт доступен в течение 10 минут
  And отчёт содержит: рейтинг всех регионов, топ слабых предметов, НТТ-готовность, охват
  And данные персонализированы до уровня регионов, не ниже
  And PowerPoint-версия содержит готовые слайды с графиками
```

### Сценарий 3: Анализ трендов

```gherkin
Given пользователь выбрал предмет «Математика», класс «9», период «2023–2026»

When запрашивает данные

Then система возвращает агрегированный тренд по годам
  And данные представлены на уровне регионов
  And поддерживается сравнение «до/после» изменения программы (по дате изменения)
  And время загрузки графика < 3 секунды (данные из analytics_snapshots)
```

## 9. Бизнес-обоснование

### Министерство — крупнейший контракт Zedly

Один национальный контракт с Министерством просвещения = $500,000+/год и де-факто статус государственной образовательной инфраструктуры. После этого уровня конкурентам практически невозможно вытеснить Zedly.

**Путь к контракту:**
1. Пилот РОНО (3 месяца, бесплатно) → кейс
2. Районный контракт → следующие кейсы
3. Областной уровень → Министерство слышит о Zedly от нескольких регионов
4. Министерство инициирует переговоры или тендер

**Ключевые аргументы для Министерства:**
- Экономия: замена ручного сбора отчётности на автоматизацию (национальный масштаб: сотни тысяч человеко-часов)
- Принятие решений на данных: первая в стране система с национальной картой слабых тем
- Данные хранятся в Узбекистане (требование законодательства — Zedly соответствует)
- Стоимость разработки государственного аналога: $2–5 миллионов + 3 года + риск провала

| Метрика | Целевое значение |
|---------|-----------------|
| Стоимость национального контракта | $200,000–500,000/год |
| Срок от первого пилота РОНО до переговоров с Минпросом | 12–18 месяцев |
| % отчётности РОНО, переведённой на Zedly | ≥ 80% при национальном контракте |
