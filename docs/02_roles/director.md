# Роль: Директор (Director)

**Платформа:** Zedly — онлайн-тестирование и аналитика для школ Узбекистана  
**Версия документа:** 1.0  
**Статус:** Production-ready spec

---

## 1. Описание роли

Директор — административный руководитель школы, который управляет лицензией Zedly, видит агрегированную аналитику всей школы и является ключевым лицом, принимающим решение о покупке школьной лицензии.

Директор приходит на платформу не первым — он приходит тогда, когда 5–7 учителей его школы уже активно используют Zedly на freemium. Zedly автоматически отправляет директору триггер-уведомление, и директор сам приходит за лицензией. Это ключевая конверсионная механика платформы.

**Язык интерфейса:** узбекский или русский.  
**Основной канал входа:** email или Telegram (директора чаще используют email).  
**Устройство:** преимущественно десктоп или планшет (директор работает в кабинете).  
**Технический уровень:** средний — директор не технарь, интерфейс должен быть понятен без инструкций.

## 2. Цели

| # | Цель | Как Zedly помогает |
|---|------|--------------------|
| 1 | Видеть реальную картину успеваемости школы | Дашборд: средний балл по классам, предметам, учителям — в реальном времени |
| 2 | Готовить отчёты для РОНО без ручного сбора данных | Автоматическая генерация отчётов в нужном формате за 1 клик |
| 3 | Контролировать активность учителей | Видит, какие учителя используют платформу, а какие — нет |
| 4 | Управлять лицензиями и доступами | Добавить/удалить учителя, управлять тарифом |
| 5 | Обосновать результаты перед РОНО и родителями | Данные с динамикой: «за квартал средний балл вырос на 12%» |
| 6 | Подготовить школу к НТТ | Видит NTT Readiness Score по всем выпускным классам |

## 3. Боли

### 3.1 Боли до Zedly

| Боль | Конкретика | Цена |
|------|------------|------|
| Сбор отчётности для РОНО вручную | Директор обходит учителей, собирает Excel-таблицы, сводит вручную | 8–12 часов на квартальный отчёт |
| Нет картины успеваемости в реальном времени | Директор узнаёт о проблемах на педсовете раз в квартал | Поздно реагирует — проблемы накапливаются |
| Непонятно, какие учителя работают эффективно | Субъективные оценки, нет данных | Кадровые решения принимаются вслепую |
| Нет инструмента давления на неактивных учителей | «Используйте цифровые инструменты» — пустые слова без платформы | Низкая цифровая вовлечённость педколлектива |
| Сложно доказать результаты РОНО | Бумажные журналы, нет динамики, нет сравнения | Слабая позиция на проверках |

### 3.2 Чего Zedly НЕ делает для директора

- Не заменяет систему управления школой (расписание, кадровый учёт)
- Не даёт доступ к персональным данным учеников без необходимости (только агрегаты)
- Не автоматически повышает успеваемость — только делает проблемы видимыми

## 4. Разрешения (Permissions)

### 4.1 Что директор МОЖЕТ делать

```
АНАЛИТИКА ШКОЛЫ
🔒 view_school_aggregate_analytics  — агрегированная аналитика всей школы (только платный план)
✅ view_class_analytics_all         — аналитика любого класса в своей школе
🔒 view_teacher_activity            — активность учителей: тестов создано, учеников охвачено (только платный план)
✅ view_subject_performance         — успеваемость по предметам в разрезе классов
✅ view_ntt_readiness_school        — NTT Readiness Score по всем выпускным классам
✅ view_trends_by_quarter           — динамика по четвертям/учебному году

ОТЧЁТЫ ДЛЯ РОНО
✅ generate_roono_report            — сформировать отчёт в формате РОНО (PDF/Excel)
✅ schedule_auto_reports            — настроить автоматическую отправку отчётов по расписанию
✅ export_school_data               — выгрузить данные школы в CSV/Excel

УПРАВЛЕНИЕ ПОЛЬЗОВАТЕЛЯМИ
✅ invite_teacher                   — пригласить учителя в школу (email или Telegram)
✅ deactivate_teacher               — деактивировать учителя (уволился)
✅ view_teacher_list                — список всех учителей школы и их статус
✅ reassign_classes                 — переназначить класс другому учителю
✅ manage_student_accounts          — массовая загрузка учеников (CSV-импорт)

ЛИЦЕНЗИЯ
✅ purchase_school_license          — купить/продлить школьную лицензию (Payme/Click)
✅ view_license_status              — текущий тариф, дата истечения, количество мест
✅ view_usage_stats                 — сколько учителей/учеников активно используют платформу

ПРОФИЛЬ ШКОЛЫ
✅ edit_school_profile              — название школы, регион, контакты
✅ set_school_logo                  — загрузить логотип (появляется на сертификатах)
```

> Примечание: для консистентности RBAC `director` использует канонические права `view_school_analytics` и `view_school_teacher_activity` из `permissions_matrix.md`; ключи выше — человекочитаемые алиасы.

### 4.2 Что директор НЕ МОЖЕТ делать

```
❌ view_individual_student_answers  — ответы конкретного ученика (только учитель и сам ученик)
❌ create_test                      — создавать тесты (это роль учителя)
❌ assign_test_to_class             — назначать тесты классу напрямую
❌ access_data_of_other_schools     — данные других school_id (RLS на уровне БД)
❌ view_district_analytics          — агрегированная аналитика района (только РОНО)
❌ edit_teacher_tests               — редактировать тесты учителей
❌ view_audit_logs                  — системный журнал безопасности (только admin)
❌ change_platform_settings         — глобальные настройки платформы
```

### 4.3 Freemium vs Платная лицензия директора

| Функция | До покупки лицензии | После покупки лицензии |
|---------|--------------------|-----------------------|
| Доступ к дашборду директора | ❌ Недоступен | ✅ Полный доступ |
| Аналитика по учителям | ❌ | ✅ |
| Генерация отчётов РОНО | ❌ | ✅ |
| Управление пользователями | Базовое (только просмотр списка) | ✅ Полное |
| Количество учителей freemium | До 30 учеников на класс у каждого | Без ограничений для всей школы |
| Логотип школы на сертификатах | ❌ | ✅ |
| Приоритетная поддержка | ❌ | ✅ |

> **Триггер покупки:** когда ≥ 5 учителей школы активны на freemium, директор получает Telegram/email-уведомление: *«92% ваших учителей уже используют Zedly. Школьная лицензия снимет все ограничения — от $X/месяц»*. Кнопка «Купить лицензию» ведёт напрямую на оплату через Payme или Click.

## 5. Ограничения

| Ограничение | Значение | Причина |
|-------------|----------|---------|
| Изоляция по school_id | Директор видит только свою школу | Мультитенантность, RLS |
| Агрегация без персонализации | Аналитика по классам и предметам, но не по конкретным ученикам | Приватность учеников |
| Только читатель тестов | Не может создавать или редактировать тесты | Разделение ролей |
| Отчёты РОНО — только генерация | Директор генерирует отчёт, но отправляет его вручную (пока нет интеграции с порталом РОНО) | MVP ограничение; в V2 — автоотправка |

## 6. Видимая аналитика

### 6.1 Дашборд директора

**Виджет 1: Обзор школы (KPI-панель)**
- Средний балл по школе за текущий месяц
- Динамика: «▲ +5% по сравнению с прошлым месяцем»
- Количество активных учителей / всего учителей
- Количество тестов проведено за месяц
- Процент учеников, охваченных тестированием

**Виджет 2: Успеваемость по классам (Heatmap)**
- Матрица: Класс (строки) × Предмет (столбцы)
- Ячейка = средний балл; цвет: 🔴 < 50%, 🟡 50–70%, 🟢 > 70%
- Клик на ячейку → детальная аналитика класса по предмету

**Виджет 3: Активность учителей**
- Таблица: Учитель | Предмет | Тестов за месяц | Учеников охвачено | Последняя активность
- Учителя без активности > 2 недель помечены 🔴
- Кнопка «Напомнить» → Telegram-уведомление учителю

**Виджет 4: НТТ Readiness (выпускные классы)**
- Только для 9, 10, 11 классов
- По каждому выпускному классу: средний NTT Readiness Score и динамика
- Топ-3 слабых предмета по школе в контексте НТТ

**Виджет 5: Топ слабых тем по школе**
- Темы с наибольшим % ошибок по всем классам
- Пример: «Квадратные уравнения — 58% ошибок в 4 классах»
- Рекомендация: «Рассмотрите методическую поддержку по этой теме»

### 6.2 Отчёт для РОНО

Содержание автоматически генерируемого отчёта:
- Шапка: школа, район, РОНО, период
- Средний балл по предметам в разрезе классов (таблица)
- Динамика за квартал / год
- Процент охвата учеников тестированием
- Топ-5 слабых тем по школе
- Сравнение с предыдущим периодом
- Подпись: «Данные сформированы автоматически платформой Zedly»

Форматы: PDF (для печати и подписи) + Excel (для РОНО-систем).

### 6.3 Что директор НЕ видит

- Ответы конкретных учеников на конкретные вопросы
- Аналитику других школ
- Данные района и области (только РОНО)
- Системные логи и технические данные

## 7. Пользовательские флоу

### Flow 1: Первый вход и покупка школьной лицензии

```
Актор: Директор, который получил триггер-уведомление
Предусловие: ≥ 5 учителей школы активны на freemium

Шаг 1: Директор получает Telegram/email:
        «В вашей школе [Школа №47, Ташкент] 7 учителей активно используют Zedly.
         Школьная лицензия снимет лимиты для всех.
         [Посмотреть дашборд директора]»

Шаг 2: Директор переходит по ссылке → страница регистрации/входа
        → Регистрируется (email + пароль или Telegram)
        → Система автоматически привязывает к школе по домену email или коду

Шаг 3: Директор видит preview дашборда (blur-эффект на данных):
        «Ваши учителя уже создали 47 тестов и охватили 320 учеников.
         Откройте полный дашборд — купите школьную лицензию»
        → Кнопка «Купить лицензию» (выделена, основное CTA)

Шаг 4: Страница тарифов:
        → Базовый: [цена]/месяц — до 50 учителей
        → Стандарт: [цена]/месяц — до 200 учителей
        → Корпоративный: договорная цена — 200+ учителей
        → Годовая оплата = скидка 20%

Шаг 5: Оплата через Payme или Click
        → Ввод реквизитов → подтверждение
        → Лицензия активируется немедленно

Шаг 6: Дашборд директора открывается полностью
        → Все учителя школы автоматически переведены на платный план
        → Директор видит полную аналитику школы

Постусловие:
✅ Лицензия активна, записана в таблицу school_licenses
✅ Все учителя школы получают Telegram-уведомление: «Директор активировал школьную лицензию»
✅ Ограничение 30 учеников снято для всех классов
✅ Директор видит полный дашборд
```

### Flow 2: Формирование квартального отчёта для РОНО

```
Актор: Директор (платный план)
Предусловие: Прошёл минимум 1 месяц с активным использованием

Шаг 1: Директор открывает раздел «Отчёты»
        → Видит список доступных шаблонов отчётов:
          - «Квартальный отчёт РОНО» (стандартная форма)
          - «Отчёт по НТТ-готовности»
          - «Отчёт по активности учителей»
          - «Произвольный экспорт данных»

Шаг 2: Директор выбирает «Квартальный отчёт РОНО»
        → Выбирает период: I квартал 2025/2026 учебного года
        → Система показывает preview: какие данные войдут в отчёт

Шаг 3: Директор нажимает «Сформировать отчёт»
        → Background job запускается
        → Статус: «Формируется... (~30 секунд)»

Шаг 4: Отчёт готов
        → Кнопки: «Скачать PDF» | «Скачать Excel» | «Отправить на email»
        → PDF содержит официальную шапку с названием школы и логотипом
        → Excel содержит все данные в структурированном виде

Шаг 5: Директор скачивает PDF, подписывает (ЭЦП или вручную) и отправляет в РОНО

Постусловие:
✅ Отчёт сформирован без ручного сбора данных
✅ Данные актуальны на момент формирования
✅ История отчётов сохраняется (директор может скачать повторно)
✅ Экономия: 8–12 часов ручного труда → 2 минуты
```

### Flow 3: Работа с неактивными учителями

```
Актор: Директор
Предусловие: Несколько учителей не используют Zedly > 2 недель

Шаг 1: Директор открывает виджет «Активность учителей»
        → Видит таблицу с сортировкой по активности

Шаг 3: Директор нажимает «Отправить напоминание»
        → Выбирает: стандартный текст или свой
        → Стандартный: «[Иванова М.А.], директор школы просит активнее использовать Zedly для тестирования учеников»
        → Отправляется через Telegram Bot или email

Шаг 4 (альтернативный): Директор видит паттерн — 5 учителей неактивны
        → Нажимает «Напомнить всем неактивным» → массовая рассылка

Постусловие:
✅ Уведомления отправлены
✅ Факт отправки зафиксирован в истории (директор видит: «Напоминание отправлено 15.02.2026»)
✅ Если учитель активируется после напоминания — директор видит это в дашборде
```

```
Шаг 3: Подтверждение → учитель деактивирован немедленно

Постусловие:
✅ Деактивированный учитель теряет доступ к платформе
✅ Его тесты и результаты учеников сохранены
✅ Классы переданы новому учителю
✅ История изменений зафиксирована
```

## 8. Граничные случаи (Edge Cases)

### 8.1 Технические

| Ситуация | Поведение системы | Что видит директор |
|----------|-------------------|-------------------|
| Директор открывает дашборд до покупки лицензии | Данные размыты (blur), CTA «Купить лицензию» | Preview с призывом к действию |
| Директор не получил триггер-уведомление (Telegram заблокирован) | Резервная отправка на email; если нет email — уведомление при следующем входе | Баннер на странице входа |
| Отчёт РОНО генерируется дольше 60 секунд | Уведомление: «Отчёт готовится. Пришлём ссылку на email» | Email со ссылкой на скачивание |
| Директор пытается увидеть аналитику другой школы | 403 Forbidden, запись в audit_log | Страница ошибки доступа |
| Школа превысила лимит учителей тарифного плана | Новых учителей нельзя добавить; уведомление об апгрейде | «Лимит тарифа достигнут. Обновите план» |

### 8.2 Бизнес

| Ситуация | Поведение системы |
|----------|-------------------|
| Директор не продлил лицензию (истекла) | Graceful downgrade: учителя переходят на freemium с лимитом 30 учеников. Данные сохранены. Директор теряет доступ к своему дашборду. Предупреждение за 14 и 7 дней до истечения |
| Два директора в одной школе (смена директора) | Система поддерживает роль «директор» только для одного пользователя. Старый директор деактивируется через обращение в поддержку |
| Директор хочет видеть ответы конкретного ученика | Система не разрешает: директор видит только агрегаты. Если нужна детальная информация — учитель формирует отчёт |
| РОНО требует иной формат отчёта | Директор может запросить кастомный шаблон через поддержку Zedly; срок разработки — 5 рабочих дней |
| Школа объединилась с другой | Требует ручной миграции через поддержку Zedly; автоматически не поддерживается (MVP) |

## 9. Acceptance Criteria

### Сценарий 1: Триггер покупки лицензии

```gherkin
Given в школе активны ≥ 5 учителей на freemium
  And директор ещё не купил лицензию

When система фиксирует 5-го активного учителя (создал тест + назначил классу)

Then директор получает Telegram/email-уведомление в течение 60 минут
  And уведомление содержит: количество активных учителей, кнопку «Посмотреть дашборд»
  And событие «license_trigger_sent» записывается в analytics
  And если директор не реагирует 3 дня — повторное уведомление
```

### Сценарий 2: Дашборд директора — изоляция данных

```gherkin
Given директор авторизован в школе A

When директор открывает дашборд

Then видит аналитику только своей школы (school_id = school_A)
  And данные других школ недоступны (RLS)
  And все запросы к БД фильтруются по school_id на уровне PostgreSQL RLS
  And попытка получить данные school_B возвращает 403 + запись в audit_log
```

### Сценарий 3: Генерация отчёта РОНО

```gherkin
Given директор выбрал период «I квартал 2025/2026»
  And нажал «Сформировать квартальный отчёт РОНО»

When background job завершает работу

Then PDF-отчёт доступен для скачивания в течение 60 секунд
  And отчёт содержит: средний балл по предметам, динамику, охват учеников
  And данные соответствуют тем, что директор видит в дашборде (нет расхождений)
  And отчёт хранится в системе 12 месяцев
  And директор может скачать его повторно в любое время
```

### Сценарий 4: Деактивация учителя

```gherkin
Given директор деактивировал учителя Иванову М.А.

Then Иванова М.А. немедленно теряет доступ к платформе
  And её текущие JWT-токены инвалидируются в Redis
  And все её тесты остаются в системе с пометкой «учитель деактивирован»
  And классы Ивановой отображаются как «без учителя» до назначения нового
  And ученики Ивановой сохраняют доступ к своим результатам и портфолио
```

## 10. Бизнес-обоснование

### Директор как монетизационный шлюз

Директор — единственная роль, которая платит за платформу. Вся воронка ведёт к нему:

- Учителя создают ценность на freemium → директор видит активность
- Zedly отправляет триггер → директор приходит сам
- Preview дашборда с blur-эффектом → директор хочет видеть данные
- Оплата через Payme/Click → 1 клик, без счётов и договоров

**Целевая конверсия:** 5–7 активных учителей в школе = 1 школьная лицензия. При средней школе в 30–50 учителей — это достижимо за 2–3 месяца органического роста.

### Экономика директора

| Метрика | Целевое значение |
|---------|-----------------|
| Время от триггера до покупки лицензии | ≤ 7 дней |
| Конверсия «триггер → покупка» | ≥ 25% |
| Retention лицензии (продление через год) | ≥ 80% |
| Среднее время формирования отчёта РОНО (до/после) | 10 часов → 2 минуты |
| NPS директоров | ≥ 55 |
