# Роль: SuperAdmin

**Платформа:** Zedly  
**Версия документа:** 1.0  
**Статус:** Production-ready spec

---

## 1. Назначение роли

SuperAdmin — глобальная служебная роль оператора платформы Zedly.

Роль не принадлежит школе (`school_id IS NULL`) и используется для:
- управления тенантами (школами),
- управления тарифами/планами,
- мониторинга глобальной статистики без PII.

---

## 2. Политика доступа

### Разрешено
- `access_admin_panel`
- `manage_all_schools`
- `suspend_school`
- `manage_subscriptions_global`
- `manage_plans`
- `view_global_analytics_aggregated`

### Ограничено
- Нет прямого доступа к PII учеников/родителей в operational таблицах.
- Доступ к cross-school данным только через агрегированные `admin_*` views.

---

## 3. RLS-модель

- School data остаются под RLS.
- SuperAdmin операции выполняются:
  - через отдельный backend namespace `/admin/*`;
  - через отдельные read-model/view без персональных данных;
  - через сервисные функции с явным allowlist.

---

## 4. Типовые операции

1. Создать новую школу (`tenant bootstrap`).
2. Заблокировать школу при нарушении политики.
3. Назначить/изменить план для user-based подписок.
4. Смотреть глобальные KPI по регионам без персонализации.

---

## 5. Аудит

- Все действия SuperAdmin обязаны попадать в неизменяемый audit-log.
- Для критических действий (блокировка школы, смена плана) — reason + actor + timestamp обязательны.
